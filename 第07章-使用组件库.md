# 第07章 使用组件库

## 7.1 为什么使用组件库

### 原生开发的痛点

在不使用组件库的情况下，所有UI都需要手写WXML+WXSS，存在以下问题：

| 痛点 | 说明 |
|------|------|
| 重复造轮子 | 按钮、弹窗、下拉刷新等基础组件反复手写 |
| 样式不统一 | 不同开发者写出的样式风格不一致 |
| 维护成本高 | 修改一个通用样式需要改动多个文件 |
| 交互体验差 | 动画、过渡效果难以手写完善 |
| 开发效率低 | 大量时间花在UI还原上 |

### 组件库的优势

```
使用组件库前：
├── 手写按钮样式（50行WXSS）
├── 手写弹窗逻辑（100行JS）
├── 手写表单验证（80行JS）
└── 手写加载动画（30行WXSS）

使用组件库后：
├── <van-button type="primary">按钮</van-button>
├── <van-dialog id="dialog" />
├── <van-field rules="{{rules}}" />
└── <van-loading type="spinner" />
```

### 小程序主流组件库

| 组件库 | 特点 | 适用场景 |
|--------|------|---------|
| Vant Weapp | 有赞出品，组件丰富，文档完善 | 电商、通用业务 |
| TDesign | 腾讯出品，设计规范统一 | 中后台、企业应用 |
| ColorUI | 酷炫UI，轻量 | 个人项目、展示类 |
| WeUI | 微信官方风格 | 微信原生风格应用 |
| iview-weapp | iView的小程序版 | 后台管理类 |

> **💡 本章选择Vant Weapp**：它是目前社区最活跃、组件覆盖最全的小程序组件库，适合教学和实战。

## 7.2 何谓自定义组件库

### 从原生组件到自定义组件

小程序从基础库1.6.3开始支持自定义组件。开发者可以将一段UI封装成独立组件，实现复用。

### 自定义组件的结构

```
components/
└── my-button/
    ├── my-button.js      # 组件逻辑
    ├── my-button.json    # 组件配置
    ├── my-button.wxml    # 组件模板
    └── my-button.wxss    # 组件样式
```

```javascript
// components/my-button/my-button.js
Component({
  // 组件属性（外部传入）
  properties: {
    type: {
      type: String,
      value: 'default'
    },
    disabled: {
      type: Boolean,
      value: false
    }
  },

  // 组件内部数据
  data: {
    isActive: false
  },

  // 组件方法
  methods: {
    onTap() {
      if (this.data.disabled) return;
      this.triggerEvent('tap');  // 触发自定义事件
    }
  }
});
```

```xml
<!-- components/my-button/my-button.wxml -->
<view 
  class="my-btn my-btn--{{type}} {{disabled ? 'my-btn--disabled' : ''}}"
  bindtap="onTap">
  <slot></slot>
</view>
```

```json
// components/my-button/my-button.json
{
  "component": true,
  "usingComponents": {}
}
```

### 组件库 vs 页面

| 特性 | 组件（Component） | 页面（Page） |
|------|------------------|-------------|
| 注册方式 | `Component({...})` | `Page({...})` |
| 数据 | `properties` + `data` | `data` |
| 事件 | `triggerEvent` | `bindTap`等 |
| 生命周期 | `attached`/`detached` | `onLoad`/`onUnload` |
| 复用性 | 高（可跨页面使用） | 低（一个路由一个页面） |

### 什么是"组件库"

组件库是一套预先开发好的、可复用的组件集合，通常包含：

```
组件库
├── 基础组件：Button、Icon、Cell、Popup
├── 表单组件：Input、Switch、Picker、Uploader
├── 反馈组件：Dialog、Toast、Loading、ActionSheet
├── 展示组件：Tag、Badge、Card、Steps
├── 导航组件：Tab、NavBar、Sidebar
└── 业务组件：AddressList、Sku、Coupon
```

> **💡 理解**：Vant Weapp本质上就是别人写好的一套自定义组件集合，我们通过npm安装后直接使用，省去自己造轮子的时间。

## 7.3 导入Vant-Weapp组件库

### 第一步：初始化npm

在项目根目录（与`app.json`同级）执行：

```bash
# 如果没有package.json，先初始化
npm init -y

# 安装Vant Weapp
npm i @vant/weapp -S --production
```

安装后项目结构：

```
项目根目录/
├── miniprogram_npm/       # 构建后生成
├── node_modules/          # npm包
│   └── @vant/weapp/
├── package.json
├── app.json
└── pages/
```

### 第二步：修改 project.config.json

确保`project.config.json`中的`packNpmManually`和`packNpmRelationList`配置正确：

```json
{
  "setting": {
    "packNpmManually": true,
    "packNpmRelationList": [
      {
        "packageJsonPath": "./package.json",
        "miniprogramNpmDistDir": "./miniprogram_npm/"
      }
    ]
  }
}
```

> **⚠️ 注意**：如果项目使用了自定义目录结构（如代码在`miniprogram/`子目录下），`packageJsonPath`和`miniprogramNpmDistDir`需要相应调整。

### 第三步：构建npm

在微信开发者工具中：

1. 点击菜单栏 **工具 → 构建 npm**
2. 等待构建完成，会生成`miniprogram_npm`目录
3. 构建成功后，`miniprogram_npm/@vant/weapp/`下会出现组件文件

```
构建前：
├── node_modules/@vant/weapp/   ← npm安装的原始包

构建后：
├── miniprogram_npm/@vant/weapp/  ← 小程序可用的构建产物
```

> **⚠️ 常见问题**：
> - **"找不到可以构建的npm包"**：确保`package.json`在项目根目录，且`node_modules`中已安装依赖。
> - **构建后组件不生效**：重启开发者工具或清除缓存后重新编译。
> - **旧版基础库不支持**：Vant Weapp需要基础库2.6.5+。

### 第四步：在页面json中引入组件

### 按需引入 vs 全局引入

```json
// 按需引入（推荐）：在需要的页面json中配置
// pages/xxx/xxx.json
{
  "usingComponents": {
    "van-button": "@vant/weapp/button/index"
  }
}

// 全局引入：在app.json中配置，所有页面可用
// app.json
{
  "usingComponents": {
    "van-button": "@vant/weapp/button/index",
    "van-icon": "@vant/weapp/icon/index"
  }
}
```

> **💡 建议**：常用组件（如`van-button`、`van-icon`）可以在`app.json`中全局引入；不常用的按需在页面引入，减小包体积。

## 7.4 实现录音与语音消息的发送（wx.getRecorderManager）

### RecorderManager概述

`wx.getRecorderManager()`返回全局唯一的录音管理器，支持录制mp3、aac等格式：

```javascript
const recorderManager = wx.getRecorderManager();

Page({
  data: {
    isRecording: false,
    recordDuration: 0
  },

  onLoad() {
    this.initRecorder();
  },

  initRecorder() {
    // 录音开始回调
    recorderManager.onStart(() => {
      console.log('录音开始');
      this.setData({ isRecording: true });
    });

    // 录音结束回调
    recorderManager.onStop((res) => {
      console.log('录音结束', res);
      this.setData({ isRecording: false });

      if (res.duration < 1000) {
        wx.showToast({ title: '录音太短', icon: 'none' });
        return;
      }

      // res.tempFilePath: 录音文件临时路径
      // res.duration: 录音时长（毫秒）
      this.sendVoiceMessage(res.tempFilePath, res.duration);
    });

    // 录音错误回调
    recorderManager.onError((err) => {
      console.error('录音错误：', err);
      this.setData({ isRecording: false });
      wx.showToast({ title: '录音失败', icon: 'none' });
    });
  },

  startRecord() {
    recorderManager.start({
      duration: 60000,           // 最长录音时长（毫秒）
      sampleRate: 44100,         // 采样率
      numberOfChannels: 1,       // 录音通道数
      encodeBitRate: 192000,     // 编码码率
      format: 'mp3'              // 音频格式
    });
  },

  stopRecord() {
    recorderManager.stop();
  },

  sendVoiceMessage(filePath, duration) {
    // 上传语音文件到服务器
    wx.uploadFile({
      url: 'https://api.example.com/upload/voice',
      filePath: filePath,
      name: 'voice',
      formData: {
        duration: Math.floor(duration / 1000)
      },
      success: (res) => {
        const data = JSON.parse(res.data);
        console.log('上传成功：', data.url);

        // 添加语音评论
        const voiceComment = {
          id: Date.now(),
          type: 'voice',
          voice: data.url,
          voiceDuration: Math.floor(duration / 1000),
          username: '当前用户',
          avatar: '/images/avatar/default.png',
          createTime: this.formatTime(new Date())
        };

        const commentList = [voiceComment, ...this.data.commentList];
        this.setData({ commentList });
      }
    });
  },

  formatTime(date) {
    const h = String(date.getHours()).padStart(2, '0');
    const m = String(date.getMinutes()).padStart(2, '0');
    return `${h}:${m}`;
  }
});
```

### 录音参数详解

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `duration` | 最长录音时长（ms） | 60000（60秒） |
| `sampleRate` | 采样率 | 44100 |
| `numberOfChannels` | 声道数 | 1（单声道） |
| `encodeBitRate` | 编码码率 | 192000 |
| `format` | 音频格式 | mp3 |

> **⚠️ 注意**：
> - 采样率和码率越高，音质越好但文件越大。
> - `format`推荐用`mp3`，兼容性最好。
> - 录音需要用户授权`scope.record`权限。

## 7.5 微信隐私接口调用指南

### 隐私接口概述

涉及用户隐私的接口（录音、位置、相册等）需要用户明确授权才能使用。微信对隐私接口的管理分为两层：

1. **权限授权**：首次调用时弹出授权弹窗
2. **隐私协议**：2023年9月起，需要用户同意隐私协议后才能调用

### 隐私相关接口列表

| 接口 | 所需权限scope | 说明 |
|------|-------------|------|
| `wx.getRecorderManager` | `scope.record` | 录音 |
| `wx.chooseMedia` | `scope.writePhotosAlbum` | 保存图片到相册 |
| `wx.getLocation` | `scope.userLocation` | 获取位置 |
| `wx.chooseAddress` | — | 获取地址 |
| `wx.startBluetoothDiscover` | `scope.bluetooth` | 蓝牙 |
| `wx.saveImageToPhotosAlbum` | `scope.writePhotosAlbum` | 保存图片 |

### 配置隐私协议

在`app.json`中配置`__usePrivacyCheck__`：

```json
{
  "__usePrivacyCheck__": true
}
```

在小程序管理后台「设置 → 服务内容声明 → 用户隐私保护指引」中填写隐私协议。

### 代码中处理隐私授权

```javascript
Page({
  onRecordStart() {
    // 检查隐私授权
    wx.getPrivacySetting({
      success(res) {
        if (res.needAuthorization) {
          // 需要用户同意隐私协议
          wx.requirePrivacyAuthorize({
            success() {
              console.log('用户同意隐私协议');
              this.startRecord();
            },
            fail() {
              console.log('用户拒绝隐私协议');
              wx.showToast({ title: '需要同意隐私协议才能录音', icon: 'none' });
            }
          });
        } else {
          // 已同意，直接录音
          this.startRecord();
        }
      }
    });
  },

  startRecord() {
    // 检查录音权限
    wx.authorize({
      scope: 'scope.record',
      success: () => {
        recorderManager.start({ format: 'mp3' });
      },
      fail: () => {
        wx.showToast({ title: '需要录音权限', icon: 'none' });
      }
    });
  }
});
```

> **⚠️ 注意**：从2023年9月15日起，所有涉及用户隐私的接口都需要先处理隐私协议。如果未配置隐私协议或用户未同意，相关接口将无法调用。

## 7.6 用户拒绝授权后如何再次拉起授权（wx.openSetting）

### 授权流程问题

当用户首次拒绝授权后，再次调用`wx.authorize`不会弹出授权弹窗。需要引导用户去设置页手动开启权限。

```javascript
Page({
  startRecord() {
    wx.authorize({
      scope: 'scope.record',
      success: () => {
        // 授权成功，开始录音
        recorderManager.start({ format: 'mp3' });
      },
      fail: () => {
        // 授权失败（用户曾拒绝过）
        this.showAuthDialog();
      }
    });
  },

  showAuthDialog() {
    wx.showModal({
      title: '提示',
      content: '需要您授权录音权限才能发送语音消息，是否前往设置开启？',
      confirmText: '去设置',
      cancelText: '不了',
      success: (res) => {
        if (res.confirm) {
          this.openSetting();
        }
      }
    });
  },

  openSetting() {
    wx.openSetting({
      success: (res) => {
        if (res.authSetting['scope.record']) {
          // 用户在设置页开启了权限
          console.log('录音权限已开启');
          recorderManager.start({ format: 'mp3' });
        } else {
          // 用户仍然拒绝
          wx.showToast({ title: '未开启录音权限', icon: 'none' });
        }
      }
    });
  }
});
```

### wx.getSetting 检查当前授权状态

```javascript
// 在调用接口前检查权限状态
wx.getSetting({
  success(res) {
    const authSetting = res.authSetting;

    if (authSetting['scope.record'] === false) {
      // 用户曾拒绝过
      console.log('用户曾拒绝录音权限，需引导去设置');
    } else if (authSetting['scope.record'] === true) {
      // 已授权
      console.log('已有录音权限');
    } else {
      // 未请求过授权（首次）
      console.log('首次请求录音权限');
    }
  }
});
```

### 封装权限检查工具

```javascript
// utils/auth.js
const auth = {
  /**
   * 检查并请求权限
   * @param {string} scope - 权限名，如 scope.record
   * @returns {Promise<boolean>}
   */
  ensure(scope) {
    return new Promise((resolve, reject) => {
      wx.getSetting({
        success(res) {
          if (res.authSetting[scope] === true) {
            // 已授权
            resolve(true);
          } else if (res.authSetting[scope] === false) {
            // 曾拒绝，引导去设置
            wx.showModal({
              title: '权限申请',
              content: '需要您开启相应权限才能使用此功能，是否前往设置？',
              confirmText: '去设置',
              success(modalRes) {
                if (modalRes.confirm) {
                  wx.openSetting({
                    success(settingRes) {
                      if (settingRes.authSetting[scope]) {
                        resolve(true);
                      } else {
                        reject(new Error('用户未开启权限'));
                      }
                    }
                  });
                } else {
                  reject(new Error('用户拒绝去设置'));
                }
              }
            });
          } else {
            // 首次请求
            wx.authorize({
              scope: scope,
              success() { resolve(true); },
              fail() { reject(new Error('用户拒绝授权')); }
            });
          }
        }
      });
    });
  }
};

module.exports = auth;
```

```javascript
// 使用示例
const auth = require('../../utils/auth.js');

Page({
  async onRecordStart() {
    try {
      await auth.ensure('scope.record');
      recorderManager.start({ format: 'mp3' });
    } catch (err) {
      wx.showToast({ title: '需要录音权限', icon: 'none' });
    }
  }
});
```

## 7.7 播放语音消息（wx.createInnerAudioContext）

### 基本用法

`wx.createInnerAudioContext`创建音频上下文，用于播放网络或本地音频：

```javascript
Page({
  data: {
    playingId: null  // 当前播放的评论id
  },

  onPlayVoice(event) {
    const voiceUrl = event.currentTarget.dataset.src;
    const commentId = event.currentTarget.dataset.id;

    // 如果正在播放同一条语音，停止播放
    if (this.data.playingId === commentId) {
      this.audioContext.stop();
      this.setData({ playingId: null });
      return;
    }

    // 停止之前的播放
    if (this.audioContext) {
      this.audioContext.stop();
    }

    // 创建新的音频上下文
    this.audioContext = wx.createInnerAudioContext();
    this.audioContext.src = voiceUrl;

    // 播放开始
    this.audioContext.onPlay(() => {
      console.log('开始播放');
      this.setData({ playingId: commentId });
    });

    // 播放结束
    this.audioContext.onEnded(() => {
      console.log('播放结束');
      this.setData({ playingId: null });
    });

    // 播放错误
    this.audioContext.onError((err) => {
      console.error('播放错误：', err);
      this.setData({ playingId: null });
      wx.showToast({ title: '播放失败', icon: 'none' });
    });

    this.audioContext.play();
  },

  onUnload() {
    // 页面卸载时销毁音频上下文
    if (this.audioContext) {
      this.audioContext.destroy();
      this.audioContext = null;
    }
  }
});
```

### InnerAudioContext 完整API

```javascript
const audio = wx.createInnerAudioContext();

audio.src = 'https://example.com/voice.mp3';  // 音频地址
audio.autoplay = false;     // 是否自动播放
audio.loop = false;          // 是否循环播放
audio.volume = 1.0;          // 音量 0~1
audio.obeyMuteSwitch = true; // 是否遵循系统静音开关
audio.playbackRate = 1.0;    // 播放倍速

// 方法

audio.pause();     // 暂停
audio.stop();      // 停止
audio.seek(10);    // 跳转到指定位置（秒）
audio.destroy();   // 销毁实例

// 事件
audio.onCanplay();      // 可以播放
audio.onPlay();         // 开始播放
audio.onPause();        // 暂停
audio.onStop();         // 停止
audio.onEnded();        // 自然结束
audio.onTimeUpdate();   // 播放进度变化
audio.onError();        // 错误
audio.onWaiting();      // 加载中
audio.onSeeking();      // 开始跳转
audio.onSeeked();       // 跳转完成
```

> **⚠️ 注意**：
> - `InnerAudioContext`使用后必须调用`destroy()`销毁，否则会内存泄漏。
> - 同一时间最好只创建一个音频上下文实例。
> - 网络音频需要配置合法域名（`downloadFile`合法域名）。
> - `obeyMuteSwitch`设为`false`时，即使手机开了静音也能播放（适用于语音消息场景）。

## 7.8 文章计数功能

### 计数功能设计

文章计数功能包括：阅读量、在读人数、收藏数等。

```javascript
// pages/post-detail/post-detail.js
const postData = require('../../data/data.js');

Page({
  data: {
    postId: '',
    postDetail: null,
    reading: 0,        // 在读人数
    isReading: false
  },

  onLoad(options) {
    const postId = options.id;
    this.loadPostDetail(postId);
    this.incrementViews(postId);
    this.startReading(postId);
  },

  onUnload() {
    // 页面卸载时减少在读人数
    this.stopReading(this.data.postId);
  },

  // 增加阅读量
  incrementViews(postId) {
    // 本地模拟：从缓存读取并+1
    const viewsCache = wx.getStorageSync('post_views') || {};
    viewsCache[postId] = (viewsCache[postId] || 0) + 1;
    wx.setStorageSync('post_views', viewsCache);

    // 实际项目：请求服务器增加阅读量
    // wx.request({ url: '.../posts/' + postId + '/views', method: 'POST' });
  },

  // 开始阅读（在读人数+1）
  startReading(postId) {
    const readingCache = wx.getStorageSync('post_reading') || {};
    if (!readingCache[postId]) {
      readingCache[postId] = true;
      wx.setStorageSync('post_reading', readingCache);
      this.setData({
        reading: this.data.reading + 1,
        isReading: true
      });
    }
  },

  // 停止阅读（在读人数-1）
  stopReading(postId) {
    if (this.data.isReading) {
      const readingCache = wx.getStorageSync('post_reading') || {};
      delete readingCache[postId];
      wx.setStorageSync('post_reading', readingCache);
      this.setData({
        reading: Math.max(0, this.data.reading - 1),
        isReading: false
      });
    }
  }
});
```

## 7.9 本章小结

### 知识点回顾

| 知识点 | 核心内容 |
|-------|---------|
| 组件库 | Vant Weapp通过npm安装→构建npm→json引入 |
| 自定义组件 | `Component({properties, data, methods})` |
| 录音 | `wx.getRecorderManager()`，支持mp3格式 |
| 隐私接口 | 需配置隐私协议，`wx.requirePrivacyAuthorize` |
| 再次授权 | `wx.openSetting`引导用户去设置页开启权限 |
| 语音播放 | `wx.createInnerAudioContext`，用完需destroy |
| 文章计数 | `onLoad`增加计数，`onUnload`减少在读人数 |

### Vant Weapp使用流程

```
npm init -y → npm i @vant/weapp → 工具构建npm → 页面json引入 → WXML使用
```

### 录音→上传→播放完整链路

```
录音：getRecorderManager.start() → onStop获取tempFilePath
     ↓
上传：wx.uploadFile上传到服务器 → 获取URL
     ↓
展示：评论列表展示语音条
     ↓
播放：onPlayVoice → createInnerAudioContext.play() → onEnded
     ↓
销毁：onUnload → audioContext.destroy()
```

### 权限处理完整流程

```
调用接口 → wx.getSetting检查
  ├── 已授权 → 直接使用
  ├── 首次请求 → wx.authorize弹窗
  └── 曾拒绝 → wx.showModal引导 → wx.openSetting去设置
```

### 常见坑点

1. **构建npm失败**：确保`package.json`在正确位置，开发者工具版本足够新
2. **录音权限被拒**：首次拒绝后`wx.authorize`不再弹窗，需`wx.openSetting`
3. **音频内存泄漏**：`InnerAudioContext`必须`destroy()`，否则多次创建会内存溢出
4. **隐私协议未配置**：2023年9月起未配置隐私协议会导致录音等接口失效
5. **Vant组件样式覆盖**：使用`custom-class`属性传入自定义类名覆盖样式
