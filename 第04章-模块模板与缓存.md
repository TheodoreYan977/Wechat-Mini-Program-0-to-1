# 第04章 模块模板与缓存

## 4.1 将文章数据从业务中分离

随着项目规模增长，把数据直接写在页面JS中会导致代码臃肿、难以维护。最佳实践是将数据抽离成独立的模块文件。

### 之前的问题

```javascript
// ❌ 不好的做法：数据与逻辑混在一起
Page({
  data: {
    postList: [
      { id: 1, title: '文章1', content: '内容1...', img: '/images/1.png' },
      { id: 2, title: '文章2', content: '内容2...', img: '/images/2.png' },
      // ...几十条数据堆积在这里
    ]
  }
});
```

### 分离后的结构

将数据独立到`data.js`文件中：

```
项目结构：
├── data/
│   └── data.js          # 独立的文章数据模块
├── pages/
│   └── posts/
│       └── posts.js     # 页面逻辑，从data模块导入数据
```

```javascript
// data/data.js - 独立数据文件
const postList = [
  {
    id: 1,
    title: '深入理解小程序双线程模型',
    content: '小程序采用视图层和逻辑层分离的双线程架构...',
    img: '/images/post/s-1.png',
    author: '张三',
    date: '2025-01-15',
    views: 1024,
    detail: '这是文章的详细内容...'
  },
  {
    id: 2,
    title: 'Flex布局实战指南',
    content: 'Flex布局是小程序中最常用的布局方式...',
    img: '/images/post/s-2.png',
    author: '李四',
    date: '2025-01-20',
    views: 768,
    detail: '这是文章的详细内容...'
  }
];

module.exports = {
  postList: postList
};
```

```javascript
// pages/posts/posts.js - 页面逻辑
const postData = require('../../data/data.js');

Page({
  data: {
    postList: []
  },

  onLoad() {
    this.setData({
      postList: postData.postList
    });
  }
});
```

> **💡 好处**：数据与逻辑分离后，修改数据内容不需要动页面逻辑，更换数据源（如改为接口请求）也更方便。

## 4.2 小程序中模块的导入与导出

小程序支持CommonJS和ES Module两种模块规范。

### 4.2.1 module.exports / exports（CommonJS规范）

```javascript
// utils/format.js - 导出模块
function formatDate(date, fmt = 'YYYY-MM-DD') {
  const d = new Date(date);
  const year = d.getFullYear();
  const month = String(d.getMonth() + 1).padStart(2, '0');
  const day = String(d.getDate()).padStart(2, '0');
  return fmt.replace('YYYY', year).replace('MM', month).replace('DD', day);
}

function formatNumber(num) {
  if (num >= 10000) {
    return (num / 10000).toFixed(1) + '万';
  }
  return num.toString();
}

// 方式1：module.exports（推荐）
module.exports = {
  formatDate: formatDate,
  formatNumber: formatNumber
};

// 方式2：exports（注意不能直接赋值exports = {...}）
// exports.formatDate = formatDate;
// exports.formatNumber = formatNumber;
```

```javascript
// 页面中导入模块
const { formatDate, formatNumber } = require('../../utils/format.js');

// 或整体导入
const format = require('../../utils/format.js');
format.formatDate(new Date());
```

> **⚠️ 关键区别**：
> - `module.exports = {...}`：直接替换导出对象，推荐使用
> - `exports.xxx = ...`：在默认对象上添加属性
> - **不能写**`exports = {...}`，因为`exports`只是`module.exports`的引用，重新赋值会断开引用关系

### 4.2.2 require 与 import

小程序同时支持`require`（CommonJS）和`import`（ES Module）：

```javascript
// CommonJS方式（同步加载）
const postData = require('../../data/data.js');

// ES Module方式（需要工具链支持，如开启ES6转ES5）
import { postList } from '../../data/data.js';
import postData from '../../data/data.js';
```

| 特性 | require（CommonJS） | import（ES Module） |
|------|-------------------|-------------------|
| 加载时机 | 运行时加载 | 编译时确定 |
| 语法 | `const x = require('...')` | `import x from '...'` |
| 导出 | `module.exports` | `export` / `export default` |
| 动态加载 | 支持`require(variable)` | 不支持（需静态路径） |
| 小程序支持 | 原生支持 | 需开启"ES6转ES5" |

```javascript
// ES Module导出
// data/data.js
export const postList = [...];
export default { postList };

// 导入
import postData, { postList } from '../../data/data.js';
```

> **⚠️ 注意**：小程序中`require`只支持相对路径和绝对路径（`/`开头），不支持`node_modules`路径（除非使用构建npm）。`import`同理。

## 4.3 小程序的模板化（template的定义与使用）

当多个页面有相同的UI结构时，使用`template`可以复用WXML片段，减少重复代码。

### 定义模板

使用`<template name="模板名">`定义模板：

```xml
<!-- templates/post/post-template.wxml -->
<template name="postItem">
  <view class="post-item-template">
    <image class="post-img" src="{{img}}" mode="aspectFill"></image>
    <view class="post-content">
      <text class="post-title">{{title}}</text>
      <text class="post-desc">{{content}}</text>
      <view class="post-info">
        <text>{{author}}</text>
        <text>{{date}}</text>
        <text>阅读 {{views}}</text>
      </view>
    </view>
  </view>
</template>
```

### 使用模板

使用`<template is="模板名" data="{{...数据对象}}">`引用模板：

```xml
<!-- pages/posts/posts.wxml -->
<import src="../../templates/post/post-template.wxml" />

<view class="post-list">
  <block wx:for="{{postList}}" wx:key="id">
    <template is="postItem" data="{{...item}}"></template>
  </block>
</view>
```

### data的展开传递

`data="{{...item}}"`使用展开运算符将`item`对象的属性展开为模板内的独立变量：

```javascript
// item对象
{
  id: 1,
  title: '文章标题',
  content: '摘要内容',
  img: '/images/1.png',
  author: '张三',
  date: '2025-01-01',
  views: 100
}
```

展开后，模板内可以直接使用`{{title}}`、`{{content}}`等，而无需写`{{item.title}}`。

> **⚠️ 注意**：`import`只能引入模板定义，不能引入普通WXML代码。如果需要引入整个WXML文件（包含非模板内容），使用`include`。

```xml
<!-- include：引入整个文件的WXML（除template和wxs） -->
<include src="../../templates/header.wxml" />
```

## 4.4 消除模板对外部变量名的依赖

模板的一个问题是：模板内部使用的变量名必须与传入数据的属性名一致。如果数据源的属性名与模板变量名不匹配，需要手动映射。

### 问题场景

```javascript
// 数据源的属性名是 name 和 description
{
  name: '文章标题',
  description: '文章摘要'
}

// 但模板中期望的是 title 和 content
<template name="postItem">
  <text>{{title}}</text>      <!-- 期望title -->
  <text>{{content}}</text>    <!-- 期望content -->
</template>
```

### 解决方案：在data属性中映射

```xml
<!-- 在传入data时进行变量名映射 -->
<template is="postItem" data="{{title: item.name, content: item.description}}">
</template>
```

### 更优雅的方案：使用ES6展开与重命名

```javascript
// 在JS中预处理数据，统一字段名
const rawData = [
  { name: '文章1', description: '摘要1' }
];

const normalizedData = rawData.map(item => ({
  title: item.name,
  content: item.description,
  img: item.image,
  author: item.writer,
  date: item.publishDate,
  views: item.readCount
}));
```

> **💡 最佳实践**：在设计模板时，应定义一套统一的字段命名规范（如`title`/`content`/`img`），数据源不一致时在JS层做转换，保持模板的通用性。

## 4.6 使用缓存在本地模拟服务器数据库

在实际开发中，后端API可能还没准备好。我们可以利用小程序的本地缓存（Storage）来模拟服务器数据库，实现数据的持久化存储。

### 4.6.1 应用程序生命周期

小程序的生命周期与页面生命周期不同，它作用于整个App：

```javascript
// app.js
App({
  // 小程序初始化完成时触发（全局只触发一次）
  onLaunch(options) {
    console.log('App onLaunch', options);
    // 适合做：初始化本地数据库、检查更新、获取系统信息
    this.initStorage();
  },

  // 小程序从后台切到前台
  onShow(options) {
    console.log('App onShow');
  },

  // 小程序从前台切到后台
  onHide() {
    console.log('App onHide');
  },

  // 小程序发生脚本错误或API调用失败
  onError(msg) {
    console.error('App onError', msg);
  },

  // 页面不存在时触发
  onPageNotFound(res) {
    wx.redirectTo({
      url: '/pages/404/404'
    });
  },

  globalData: {
    userInfo: null
  },

  // 初始化本地数据库
  initStorage() {
    const db = wx.getStorageSync('mini_db');
    if (!db) {
      wx.setStorageSync('mini_db', {
        posts: [],
        version: '1.0.0'
      });
    }
  }
});
```

### 4.6.2 Storage缓存初始化本地数据库

小程序提供同步和异步两种Storage API：

```javascript
// ===== 同步API =====
wx.setStorageSync('key', { data: 'value' });  // 写入
const data = wx.getStorageSync('key');          // 读取
wx.removeStorageSync('key');                    // 删除指定key
wx.clearStorageSync();                          // 清空所有缓存
const info = wx.getStorageInfoSync();           // 获取缓存信息
// info.keys → 所有key数组
// info.currentSize → 当前占用空间(KB)
// info.limitSize → 限制空间(KB)，通常10MB

// ===== 异步API =====
wx.setStorage({
  key: 'key',
  data: { data: 'value' },
  success() { console.log('保存成功'); },
  fail(err) { console.error('保存失败', err); }
});

wx.getStorage({
  key: 'key',
  success(res) { console.log('读取成功', res.data); },
  fail(err) { console.error('读取失败', err); }
});
```

### 缓存操作封装类

```javascript
// utils/storage.js
class DB {
  constructor() {
    this.storageKey = 'mini_db';
  }

  // 初始化数据库
  init(data) {
    if (!wx.getStorageSync(this.storageKey)) {
      wx.setStorageSync(this.storageKey, data);
    }
  }

  // 读取整个数据库
  getDB() {
    return wx.getStorageSync(this.storageKey) || {};
  }

  // 保存整个数据库
  saveDB(data) {
    wx.setStorageSync(this.storageKey, data);
  }

  // 读取某个集合（表）
  getCollection(collectionName) {
    const db = this.getDB();
    return db[collectionName] || [];
  }

  // 保存某个集合
  saveCollection(collectionName, data) {
    const db = this.getDB();
    db[collectionName] = data;
    this.saveDB(db);
  }

  // 向集合中添加一条记录
  insert(collectionName, item) {
    const list = this.getCollection(collectionName);
    item.id = item.id || Date.now();
    list.push(item);
    this.saveCollection(collectionName, list);
    return item;
  }

  // 根据ID查询单条记录
  findById(collectionName, id) {
    const list = this.getCollection(collectionName);
    return list.find(item => item.id === id);
  }

  // 清空缓存
  clear() {
    wx.removeStorageSync(this.storageKey);
  }
}

module.exports = new DB();
```

### 4.6.3 Babel与SWC编译

小程序开发者工具内置了JavaScript编译器，支持将ES6+语法编译为ES5：

- **Babel编译器**：传统方案，兼容性好，但编译速度较慢
- **SWC编译器**：新一代Rust编写的编译器，速度比Babel快20倍以上

在开发者工具中配置：

```
设置 → 项目设置 → 本地设置
  ☑ ES6转ES5
  ☑ 增强编译（使用SWC）
```

> **💡 实战建议**：开启"增强编译"（SWC）可以显著提升编译速度。但某些极端语法可能兼容性不如Babel，遇到问题可切换回Babel。

### 4.6.4 缓存强制清理及注意事项

开发过程中经常需要清理缓存：

| 清理方式 | 说明 |
|---------|------|
| 工具栏"清缓存" → 全部清除 | 清除编译缓存、数据缓存、文件缓存等 |
| 工具栏"清缓存" → 数据缓存 | 仅清除Storage数据 |
| 代码中`wx.clearStorageSync()` | 运行时清除所有Storage |

> **⚠️ 注意事项**：
> 1. **缓存不会自动清除**：用户删除小程序或更新小程序版本时，缓存可能保留。不要依赖缓存存储重要数据。
> 2. **缓存大小限制**：单个key上限1MB，总上限10MB。大文件不要存Storage，用文件系统API。
> 3. **敏感数据不要明文存储**：Storage未加密，可通过调试工具查看。敏感信息需加密后存储。
> 4. **清除缓存后数据丢失**：开发时改了数据结构，可能需要清缓存后重新初始化。
> 5. **缓存是同步还是异步**：推荐使用同步API（`xxxSync`），代码更清晰，数据量小时性能差异可忽略。

## 4.7 使用ES6语法编写缓存操作类

小程序全面支持ES6+语法，让我们用更现代的方式编写缓存操作类：

```javascript
// utils/db.js - ES6版本的缓存数据库操作类
const STORAGE_KEY = 'mini_db';

// 数据库初始化默认数据
const defaultDB = {
  posts: [],
  banners: [],
  version: '1.0.0',
  createdAt: new Date().toISOString()
};

class MiniDB {
  constructor(key = STORAGE_KEY) {
    this.storageKey = key;
    this.init();
  }

  /**
   * 初始化数据库
   */
  init() {
    const db = this.getDB();
    if (!db || Object.keys(db).length === 0) {
      this.saveDB(defaultDB);
      console.log('数据库初始化完成');
    }
  }

  /**
   * 读取整个数据库对象
   * @returns {Object} 数据库对象
   */
  getDB() {
    return wx.getStorageSync(this.storageKey) || {};
  }

  /**
   * 保存整个数据库对象
   * @param {Object} db - 数据库对象
   */
  saveDB(db) {
    wx.setStorageSync(this.storageKey, db);
  }

  /**
   * 获取某个集合（表）的全部数据
   * @param {string} collection - 集合名称
   * @returns {Array} 集合数据数组
   */
  findAll(collection) {
    const db = this.getDB();
    return db[collection] || [];
  }

  /**
   * 根据ID查询单条记录
   * @param {string} collection - 集合名称
   * @param {number|string} id - 记录ID
   * @returns {Object|null} 查询到的记录
   */
  findById(collection, id) {
    const list = this.findAll(collection);
    return list.find(item => item.id === id) || null;
  }

  /**
   * 按条件查询
   * @param {string} collection - 集合名称
   * @param {Function} predicate - 查询条件函数
   * @returns {Array} 符合条件的记录数组
   */
  where(collection, predicate) {
    return this.findAll(collection).filter(predicate);
  }

  /**
   * 插入一条记录
   * @param {string} collection - 集合名称
   * @param {Object} item - 要插入的记录
   * @returns {Object} 插入后的记录（含自动生成的ID）
   */
  insert(collection, item) {
    const list = this.findAll(collection);
    item.id = item.id || Date.now();
    item.createdAt = item.createdAt || new Date().toISOString();
    list.push(item);
    this._updateCollection(collection, list);
    return item;
  }

  /**
   * 批量插入
   * @param {string} collection - 集合名称
   * @param {Array} items - 记录数组
   */
  insertMany(collection, items) {
    const list = this.findAll(collection);
    items.forEach(item => {
      item.id = item.id || Date.now() + Math.random();
      item.createdAt = new Date().toISOString();
      list.push(item);
    });
    this._updateCollection(collection, list);
  }

  /**
   * 更新记录
   * @param {string} collection - 集合名称
   * @param {number|string} id - 记录ID
   * @param {Object} updates - 更新字段
   * @returns {Object|null} 更新后的记录
   */
  update(collection, id, updates) {
    const list = this.findAll(collection);
    const index = list.findIndex(item => item.id === id);
    if (index === -1) return null;
    list[index] = { ...list[index], ...updates, updatedAt: new Date().toISOString() };
    this._updateCollection(collection, list);
    return list[index];
  }

  /**
   * 删除记录
   * @param {string} collection - 集合名称
   * @param {number|string} id - 记录ID
   * @returns {boolean} 是否删除成功
   */
  delete(collection, id) {
    const list = this.findAll(collection);
    const filtered = list.filter(item => item.id !== id);
    if (filtered.length === list.length) return false;
    this._updateCollection(collection, filtered);
    return true;
  }

  /**
   * 清空指定集合
   * @param {string} collection - 集合名称
   */
  clearCollection(collection) {
    this._updateCollection(collection, []);
  }

  /**
   * 销毁整个数据库
   */
  destroy() {
    wx.removeStorageSync(this.storageKey);
  }

  /**
   * 获取缓存使用情况
   * @returns {Object} { currentSize, limitSize, keys }
   */
  getStorageInfo() {
    return wx.getStorageInfoSync();
  }

  /**
   * 私有方法：更新集合数据
   * @private
   */
  _updateCollection(collection, data) {
    const db = this.getDB();
    db[collection] = data;
    this.saveDB(db);
  }
}

// 导出单例实例
module.exports = new MiniDB();
```

### 在页面中使用DB类

```javascript
// pages/posts/posts.js
const db = require('../../utils/db.js');
const postData = require('../../data/data.js');

Page({
  data: {
    postList: []
  },

  onLoad() {
    this.initData();
    this.loadPosts();
  },

  // 初始化数据到缓存
  initData() {
    let posts = db.findAll('posts');
    if (posts.length === 0) {
      // 首次启动，将静态数据写入缓存
      db.insertMany('posts', postData.postList);
      posts = db.findAll('posts');
    }
  },

  // 从缓存加载数据
  loadPosts() {
    const posts = db.findAll('posts');
    this.setData({
      postList: posts
    });
  },

  // 阅读量+1
  increaseViews(e) {
    const id = e.currentTarget.dataset.id;
    const post = db.findById('posts', id);
    if (post) {
      db.update('posts', id, { views: (post.views || 0) + 1 });
      this.loadPosts();
```

## 4.10 本章小结

本章围绕模块化、模板化和缓存三个主题，大幅提升了项目的代码组织能力：

1. **数据分离**：将文章数据从页面逻辑中抽离为独立模块，实现数据与视图解耦
2. **模块导入导出**：掌握了`module.exports`/`exports`/`require`/`import`的使用方法和区别
3. **模板化**：学会了使用`template`定义和复用WXML片段，通过`import`引入，`data="{{...item}}"`展开传参
4. **CSS模板化**：通过`@import`复用WXSS样式，实现样式层面的模块化
5. **消除变量名依赖**：掌握了模板数据映射技巧，保持模板通用性
6. **本地缓存**：理解了Storage API的使用，构建了完整的本地数据库操作类
7. **ES6语法**：使用Class、箭头函数、展开运算符、模板字符串等现代语法编写了高效的DB类
8. **完整数据结构**：完善了文章列表、轮播图、文章详情的数据模块

关键要点回顾：
- `module.exports = {...}`是推荐的导出方式，不要直接给`exports`赋值
- template的`data`属性使用`{{...item}}`展开对象，使模板内变量名独立于外部
- `@import`必须在WXSS文件顶部，导入的样式可被当前文件覆盖
- Storage同步API更易用，单key限制1MB，总量限制10MB
- 开发阶段善用"清缓存"功能，修改数据结构后需重新初始化
- ES6 Class语法让缓存操作类更清晰、更易扩展

至此，我们已经完成了小程序基础开发的四个核心章节，从环境搭建到页面开发，从数据绑定到模块化缓存，掌握了微信小程序开发的核心技能。后续章节将进入更高级的主题，包括组件化开发、网络请求、云开发等内容。
