# 第03章 数据绑定与文章列表

## 3.1 文章列表页面结构分析及准备工作

本章我们将构建一个文章列表页面，它由顶部轮播图和下方文章列表两部分组成。最终效果类似一个简易的资讯类小程序首页。

### 页面结构分析

```
文章列表页面
├── 顶部轮播图（swiper）
│   └── 多张轮播图片
├── 文章列表（wx:for 列表渲染）
│   └── 文章卡片
│       ├── 封面图（image）
│       ├── 标题（text）
│       ├── 摘要（text）
│       └── 底部信息（作者、日期、阅读量）
```

### 准备工作

在`app.json`中注册两个页面——欢迎页和文章页：

```json
{
  "pages": [
```

## 3.2 swiper组件（轮播图）

`swiper`是小程序内置的滑块容器组件，专门用于实现轮播图效果。

### 基本用法

```xml
<!-- posts.wxml -->
<swiper class="banner-swiper" 
        indicator-dots="{{true}}" 
        autoplay="{{true}}" 
        interval="3000" 
        circular="{{true}}">
  <swiper-item>
    <image src="/images/banner1.png" mode="aspectFill"></image>
  </swiper-item>
  <swiper-item>
    <image src="/images/banner2.png" mode="aspectFill"></image>
  </swiper-item>
  <swiper-item>
    <image src="/images/banner3.png" mode="aspectFill"></image>
  </swiper-item>
</swiper>
```

```css
/* posts.wxss */
.banner-swiper {
  width: 100%;
  height: 400rpx;
}
.banner-swiper image {
  width: 100%;
  height: 100%;
}
```

### swiper核心属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `indicator-dots` | boolean | false | 是否显示面板指示点 |
| `indicator-color` | color | rgba(0,0,0,.3) | 指示点颜色 |
| `indicator-active-color` | color | #000000 | 当前选中的指示点颜色 |
| `autoplay` | boolean | false | 是否自动切换 |
| `interval` | number | 5000 | 自动切换时间间隔（ms） |
| `circular` | boolean | false | 是否采用衔接式滑动（首尾循环） |
| `vertical` | boolean | false | 滑动方向是否为纵向 |
| `current` | number | 0 | 当前所在滑块索引 |

## 3.3 Boolean值陷阱

在WXML中设置组件属性时，有一个容易踩的**布尔值陷阱**：

```xml
<!-- ❌ 错误写法：这样写autoplay的值是字符串"false"，而非布尔值false -->
<!-- 非空字符串在布尔上下文中被视为true，所以轮播依然会自动播放！ -->
<swiper autoplay="false">

<!-- ❌ 错误写法：同理，"{{false}}"外层引号内整体被视为字符串 -->
<swiper autoplay="{{false}}">  <!-- 这个其实是正确的 -->

<!-- 实际上上面第二个写法是正确的，关键区别在于： -->

<!-- ❌ 真正的错误： -->
<swiper autoplay="false">  <!-- 字符串 "false" → truthy → 自动播放 -->

<!-- ✅ 正确写法1：使用数据绑定表达式 -->
<swiper autoplay="{{false}}">  <!-- 布尔值 false → 不自动播放 -->

<!-- ✅ 正确写法2：直接写属性名（等同于true） -->
<swiper autoplay>  <!-- 布尔值 true → 自动播放 -->
```

**核心原理**：WXML中属性值如果不加`{{}}`，会被当作**字符串**处理。字符串`"false"`在布尔判断中是`truthy`（因为非空字符串），所以被当作`true`。

> **⚠️ 重要规则**：所有需要传递布尔值的属性，务必使用`{{true}}`或`{{false}}`语法，而非直接写`true`/`false`字符串。

```xml
<!-- ✅ 推荐写法：统一使用数据绑定 -->
<swiper 
  indicator-dots="{{true}}" 
  autoplay="{{true}}" 
  circular="{{true}}">
</swiper>
```

## 3.5 image组件的5种缩放模式与9种裁剪模式

`image`组件的`mode`属性决定了图片如何适配容器，共14种模式。

### 5种缩放模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| `scaleToFill` | **默认值**，拉伸图片填满容器（不保持比例） | 不推荐，图片会变形 |
| `aspectFit` | 保持纵横比缩放，完整显示图片（可能有留白） | 需要完整展示图片的场景 |
| `aspectFill` | 保持纵横比缩放，填满容器（可能裁剪） | 轮播图、封面图（推荐） |
| `widthFix` | 宽度不变，高度自动变化 | 瀑布流、长图 |
| `heightFix` | 高度不变，宽度自动变化 | 固定高度的横向列表 |

```xml
<!-- 不同缩放模式对比 -->
<image mode="scaleToFill" src="..." class="demo-img"></image>
<image mode="aspectFit" src="..." class="demo-img"></image>
<image mode="aspectFill" src="..." class="demo-img"></image>
<image mode="widthFix" src="..." style="width:100%"></image>
```

### 9种裁剪模式

裁剪模式不缩放图片，只裁剪显示区域：

| 模式 | 说明 |
|------|------|
| `top` | 裁剪顶部 |
| `bottom` | 裁剪底部 |
| `center` | 裁剪中间 |
| `left` | 裁剪左侧 |
| `right` | 裁剪右侧 |
| `top left` | 裁剪左上 |
| `top right` | 裁剪右上 |
| `bottom left` | 裁剪左下 |
| `bottom right` | 裁剪右下 |

### 懒加载

对于长列表中的图片，开启懒加载可以提升初始渲染性能：

```xml
<image 
  src="{{item.img}}" 
  mode="aspectFill" 
  lazy-load="{{true}}"
  class="post-img">
</image>
```

> **⚠️ 注意**：
> - `lazy-load`只对`page`与`scroll-view`下的图片有效
> - `image`组件默认宽度300px、高度225px，必须通过样式设置宽高
> - 图片路径支持本地路径和网络路径（HTTPS）

## 3.7 JS文件的代码结构与Page页面的生命周期

每个页面通过`Page()`函数注册，其参数对象包含数据、生命周期函数和自定义方法。

```javascript
// posts.js
Page({

  /**
   * 页面的初始数据
   */
  data: {
    postList: [],
    bannerList: []
  },

  /**
   * 生命周期：页面加载（只执行一次）
   * 可获取路由参数 options
   */
  onLoad(options) {
    console.log('页面加载', options);
    this.loadPosts();
  },

  /**
   * 生命周期：页面显示（每次切换到该页面都触发）
   */
  onShow() {
    console.log('页面显示');
  },

  /**
   * 生命周期：页面初次渲染完成（可获取节点信息）
   */
  onReady() {
    console.log('页面渲染完成');
  },

  /**
   * 生命周期：页面隐藏
   */
  onHide() {
    console.log('页面隐藏');
  },

  /**
   * 生命周期：页面卸载
   */
  onUnload() {
    console.log('页面卸载');
  },

  /**
   * 页面相关事件：下拉刷新
   */
  onPullDownRefresh() {
    console.log('用户下拉刷新');
    this.loadPosts(() => {
      wx.stopPullDownRefresh();
    });
  },

  /**
   * 页面上拉触底事件
   */
  onReachBottom() {
    console.log('页面上拉触底，加载更多');
    this.loadMorePosts();
  },

  /**
   * 自定义方法：加载文章数据
   */
  loadPosts(callback) {
    // 模拟异步请求
    setTimeout(() => {
      this.setData({
        postList: this.generateMockData()
      });
      if (callback) callback();
    }, 500);
  },

  generateMockData() {
    return [
      {
        id: 1,
        title: '文章标题1',
        content: '这是文章摘要内容...',
        img: '/images/post/s-1.png',
        author: '作者',
        date: '2025-01-01',
        views: 100
      }
    ];
  }
});
```

### 页面生命周期执行顺序

```
页面创建 → onLoad → onShow → onReady
                            ↓
页面隐藏 → onHide → （返回）→ onShow
页面卸载 → onUnload
```

| 生命周期 | 触发时机 | 典型用途 |
|---------|---------|---------|
| `onLoad` | 页面加载，只触发一次 | 初始化数据、获取路由参数 |
| `onShow` | 页面显示，可多次触发 | 刷新数据、恢复状态 |
| `onReady` | 首次渲染完成，只触发一次 | 获取节点信息（如尺寸） |
| `onHide` | 页面隐藏 | 暂停定时器、保存草稿 |
| `onUnload` | 页面卸载 | 清理资源、取消请求 |

## 3.8 数据绑定

数据绑定是小程序的核心机制——将JS中的`data`数据自动渲染到WXML视图层。

使用`{{}}`双花括号语法将变量绑定到WXML：

```xml
<!-- 简单绑定 -->
<text>{{message}}</text>

<!-- 属性绑定 -->
<image src="{{imageUrl}}"></image>

<!-- 表达式绑定 -->
<text>{{price + '元'}}</text>
<text>{{isVip ? '会员价' : '原价'}}</text>
<text>{{list.length}}条数据</text>

<!-- 条件渲染 -->
<view wx:if="{{isLoggedIn}}">已登录</view>
<view wx:else>未登录</view>
```

```javascript
Page({
  data: {
    message: 'Hello World',
    imageUrl: '/images/banner.png',
    price: 99,
    isVip: true,
    isLoggedIn: false,
    list: [1, 2, 3]
  }
});
```

## 3.9 用中间容器的思想理解小程序的数据绑定

理解数据绑定的关键，是认识到小程序的**逻辑层与视图层是分离的**。

可以将数据绑定过程想象为一个"中间容器"：

```
JS逻辑层(data)          中间容器(setData)           WXML视图层
┌──────────────┐        ┌──────────────┐        ┌──────────────┐
│ data: {      │        │              │        │              │
│   name: '张三'│ ──setData──→ │ {name:'张三'} │ ──渲染──→ │ {{name}}     │
│ }            │        │              │        │              │
└──────────────┘        └──────────────┘        └──────────────┘
```

当你修改`this.data.name`时，视图层并**不会**自动更新。必须通过`this.setData()`将数据"搬运"到中间容器，中间容器再通知视图层重新渲染。

```javascript
// ❌ 直接修改data，视图层不会更新
this.data.name = '李四';

// ✅ 使用setData，触发视图层更新
this.setData({
  name: '李四'
});
```

## 3.13 数据绑定更新

使用`setData`更新数据时，支持**路径表达式**进行局部更新：

```javascript
Page({
  data: {
    user: {
      name: '张三',
      age: 25
    },
    list: [
      { id: 1, name: '项目1' },
      { id: 2, name: '项目2' }
    ]
  },

  updateName() {
    // 更新对象的某个属性
    this.setData({
      'user.name': '李四'   // 使用路径表达式
    });
  },

  updateListItem() {
    // 更新数组的某一项
    this.setData({
      'list[0].name': '新项目1'
    });
  },

  addListItem() {
    // 追加数组项
    const newList = this.data.list.concat({ id: 3, name: '项目3' });
    this.setData({
      list: newList
    });
  }
});
```

> **⚠️ 注意**：不能直接通过`this.data.list[0].name = '新值'`修改数组项，必须使用`setData`。对于数组操作，推荐使用`concat`、`map`、`filter`等不可变方法生成新数组。

## 3.14 深入理解this.data属性与this.setData()函数

### this.data

`this.data`是页面数据的**读取入口**，包含当前页面的所有数据。

```javascript
// 读取数据
const currentList = this.data.list;
const userName = this.data.user.name;
```

### this.setData()

`this.setData()`是更新数据的**唯一正确方式**，它做两件事：
1. 更新`this.data`中的值
2. 将变化的数据发送到视图层，触发重新渲染

```javascript
// 基本用法
this.setData({
  key: value
});

// 多字段同时更新
this.setData({
  name: '张三',
  age: 26,
  'user.name': '李四'
});

// 回调（在视图更新完成后执行）
this.setData({
  message: '已更新'
}, () => {
  console.log('视图已更新');
  // 可以在这里获取更新后的节点信息
});
```

### 性能优化要点

```javascript
// ❌ 不好的做法：频繁调用setData
for (let i = 0; i < 100; i++) {
  this.setData({
    count: i
  });
}

// ✅ 好的做法：合并为一次调用
const data = {};
for (let i = 0; i < 100; i++) {
  data.count = i;
}
this.setData(data);

// ❌ 不好的做法：传递大量不需要的数据
this.setData({
  hugeList: this.data.hugeList  // 整个列表重新渲染
});

// ✅ 好的做法：只更新变化的部分
this.setData({
  'list[0].name': '新名称'  // 只更新这一项
});
```

> **⚠️ 性能原则**：
> 1. 尽量减少`setData`的调用频率，合并多次更新
> 2. 只传递变化的数据，避免传输整个大对象
> 3. 不要在`onPageScroll`等高频事件中频繁`setData`

## 3.15 列表渲染wx:for

`wx:for`用于渲染列表数据，是小程序中最常用的指令之一。

### 基本用法

```xml
<!-- 简单数组 -->
<view wx:for="{{postList}}" wx:key="id">
  <text>{{index}} - {{item.title}}</text>
</view>
```

```javascript
Page({
  data: {
    postList: [
      { id: 1, title: '文章1' },
      { id: 2, title: '文章2' },
      { id: 3, title: '文章3' }
    ]
  }
});
```

### wx:for的index和item

| 变量 | 说明 |
|------|------|
| `item` | 当前项的数据 |
| `index` | 当前项的索引（从0开始） |

### 自定义变量名

使用`wx:for-item`和`wx:for-index`可以自定义变量名（适合嵌套循环）：

```xml
<view wx:for="{{postList}}" wx:for-item="post" wx:for-index="idx" wx:key="id">
  <text>{{idx + 1}}. {{post.title}}</text>
</view>
```

### wx:key的作用

`wx:key`用于标识列表项的唯一性，帮助小程序高效地更新列表：

```xml
<!-- ✅ 使用数据中的唯一字段 -->
<view wx:for="{{list}}" wx:key="id">{{item.name}}</view>

<!-- ✅ 当item是基本类型（字符串/数字）时，使用*this -->
<view wx:for="{{['a','b','c']}}" wx:key="*this">{{item}}</view>
```

> **⚠️ 重要**：不写`wx:key`会报警告，且列表更新性能差。如果列表项有唯一ID字段，用该字段名；如果没有，用`*this`（要求列表项本身就是唯一的）。

### block标签

`wx:for`可以放在`<block>`标签上，`<block>`本身不渲染，只用于逻辑控制：

```xml
<block wx:for="{{postList}}" wx:key="id">
  <view class="post-item">{{item.title}}</view>
  <view class="post-divider"></view>
</block>
```

## 3.17 从欢迎页面路由到文章页面

### 事件绑定

在WXML中通过`bind:tap`（或`bindtap`）绑定点击事件：

```xml
<!-- welcome.wxml -->
<view class="container">
  <image class="avatar" src="/images/avatar.png"></image>
  <text class="username">Hello, 小程序</text>
  <button bind:tap="onTapToPosts" type="primary">进入文章列表</button>
</view>
```

```javascript
// welcome.js
Page({
  onTapToPosts() {
    wx.navigateTo({
      url: '/pages/posts/posts'
    });
  }
});
```

### 捕捉事件命名中的冒号

小程序事件绑定有两种写法：

```xml
<!-- 写法1：带冒号（推荐，更清晰） -->
<button bind:tap="onTap">点击</button>

<!-- 写法2：不带冒号（传统写法） -->
<button bindtap="onTap">点击</button>
```

两种写法等价。除了`bind`，还有`catch`（阻止冒泡）和`capture`（捕获阶段）：

```xml
<!-- bind：冒泡事件，事件会向上传递 -->
<view bind:tap="onOuterTap">
  <button bind:tap="onInnerTap">点击我</button>
  <!-- 点击按钮时，先触发onInnerTap，再触发onOuterTap -->
</view>

<!-- catch：阻止事件冒泡 -->
<view bind:tap="onOuterTap">
  <button catch:tap="onInnerTap">点击我</button>
  <!-- 点击按钮时，只触发onInnerTap，不会冒泡到onOuterTap -->
</view>

<!-- capture-bind：捕获阶段触发 -->
<view capture-bind:tap="onCaptureTap">
  <button bind:tap="onBubTap">点击我</button>
  <!-- 点击时先触发onCaptureTap（捕获），再触发onBubTap（冒泡） -->
</view>
```

### redirectTo与navigateTo

| 方法 | 行为 | 页面栈变化 |
|------|------|-----------|
| `wx.navigateTo` | 保留当前页，跳转到新页 | 页面栈push，新页面入栈 |
| `wx.redirectTo` | 关闭当前页，跳转到新页 | 当前页出栈，新页面入栈 |
| `wx.navigateBack` | 返回上一页 | 当前页出栈 |
| `wx.switchTab` | 跳转到tabBar页面 | 清空非tab页 |

```javascript
// navigateTo：可以从文章页返回欢迎页
wx.navigateTo({
  url: '/pages/posts/posts?id=123'
});

// redirectTo：跳转后无法返回欢迎页（欢迎页被关闭）
wx.redirectTo({
  url: '/pages/posts/posts'
});
```

> **⚠️ 注意**：小程序页面栈最多10层。超过10层后`navigateTo`会失效，需用`redirectTo`替代。

### 冒泡事件与非冒泡事件

小程序事件分为**冒泡事件**和**非冒泡事件**：

- **冒泡事件**：事件会向父节点传递（如`tap`、`longpress`、`touchstart`等）
- **非冒泡事件**：事件不会向父节点传递（如`submit`、`input`、`scroll`等）

```xml
<view bind:tap="onParentTap">
  <!-- tap是冒泡事件，点击子元素会冒泡到父元素 -->
  <view bind:tap="onChildTap">子元素</view>
</view>

<!-- 用catch阻止冒泡 -->
<view bind:tap="onParentTap">
  <view catch:tap="onChildTap">子元素（不会冒泡）</view>
</view>
```

### WXS函数响应事件与事件捕获阶段

WXS（WeiXin Script）函数可以直接在WXML中响应事件，优势是运行在视图层，无需跨线程通信，性能更好：

```xml
<!-- wxs函数响应事件 -->
<wxs module="utils" src="./utils.wxs"></wxs>

<view>
  <button bind:tap="{{utils.handleTap}}">WXS处理点击</button>
</view>
```

```javascript
// utils.wxs
function handleTap(event, ownerInstance) {
  console.log('WXS处理点击', event);
  // 可以直接操作组件的style和class，无需setData
  var instance = ownerInstance.selectComponent('.target');
  instance.setStyle({
    color: 'red'
  });
}

module.exports = {
  handleTap: handleTap
};
```

事件流的三个阶段：

```
捕获阶段（capture） → 目标阶段（target） → 冒泡阶段（bubbling）
```

```xml
<!-- 事件捕获阶段绑定 -->
<view capture-bind:tap="onCapture1">
  <view capture-bind:tap="onCapture2">
    <button bind:tap="onTap">按钮</button>
  </view>
</view>
<!-- 执行顺序：onCapture1 → onCapture2 → onTap -->
```
