# 第11章 从服务器获取数据

上一章我们用 mock 数据完成了电影模块的组件搭建。本章把数据源换成真实的服务端接口，学习小程序网络请求的全套知识：全局变量、`wx.request` 的使用、Promise/async-await 异步处理、HTTP/HTTPS 与可信域名配置。

## 11.1 准备从服务器获取数据

### 11.1.1 整体思路

电影模块首页需要三类数据：正在热映、即将上映、Top250。每类只取前 3 条。数据从后端 API 获取后，通过 `setData` 传给页面中的 `movie-list` 组件。

### 11.1.2 接口约定

假设后端提供以下接口（以豆瓣电影 API 风格为例）：

| 接口 | 方法 | 参数 | 说明 |
|------|------|------|------|
| `/v2/movie/in_theaters` | GET | `start`, `count` | 正在热映 |
| `/v2/movie/coming_soon` | GET | `start`, `count` | 即将上映 |
| `/v2/movie/top250` | GET | `start`, `count` | Top250 |

返回结构（简略）：

```json
{
  "count": 3,
  "start": 0,
  "total": 100,
  "subjects": [
    {
      "id": 1292052,
      "title": "肖申克的救赎",
      "year": "1994",
      "rating": { "average": 9.7, "max": 10 },
      "images": { "small": "http://...", "large": "http://..." },
      "genres": ["犯罪", "剧情"]
    }
  ]
}
```

> **注意**：真实项目中后端返回的字段名和结构可能与上面不同，需要根据实际接口调整 `movie` 组件中的字段映射。本章统一使用上面的结构。

### 11.1.3 前置准备

为了把请求逻辑集中管理，我们在项目根目录新建 `utils/` 目录，存放工具函数和请求封装：

```
项目根目录/
├── app.js
├── app.json
├── app.wxss
├── utils/
│   └── http.js       // 网络请求封装
├── components/
│   ├── stars/
│   ├── movie/
│   └── movie-list/
└── pages/
    └── movie/
```

## 11.2 小程序的全局变量（app.js中的globalData）

### 11.2.1 为什么需要全局变量

小程序中每个页面是独立的 `Page` 实例，页面之间默认无法直接共享数据。但有些数据是全局性的，比如用户登录态、基础 URL、主题色等。小程序在 `app.js` 的 `App()` 构造器中提供了 `globalData` 字段来存放全局数据。

### 11.2.2 定义 globalData

```javascript
// app.js
App({
  globalData: {
    // 服务器基础地址
    baseUrl: 'https://api.example.com',
    // 用户信息
    userInfo: null,
    // 登录 token
    token: '',
    // 是否已登录
    isLogin: false
  },

  onLaunch() {
    // 小程序启动时读取本地缓存的 token
    const token = wx.getStorageSync('token')
    if (token) {
      this.globalData.token = token
      this.globalData.isLogin = true
    }
  }
})
```

### 11.2.3 在页面中访问 globalData

任何页面或组件都可以通过 `getApp()` 获取到小程序全局唯一的应用实例，进而访问 `globalData`：

```javascript
// pages/movie/movie.js
const app = getApp()

Page({
  onLoad() {
    const baseUrl = app.globalData.baseUrl
    console.log('基础地址', baseUrl)
    this.loadMovies()
  },
  loadMovies() {
    const url = app.globalData.baseUrl + '/v2/movie/in_theaters'
    // ...
  }
})
```

### 11.2.4 修改 globalData

直接赋值即可，但要注意小程序不会因为 `globalData` 变化而自动触发页面 `setData`，需要手动同步：

```javascript
// 修改全局数据
app.globalData.token = 'new_token'

// 如果页面要用，还需手动 setData
this.setData({ isLogin: app.globalData.isLogin })
```

> **坑点提示**：
> - `globalData` 不是响应式的，修改它不会自动更新页面。需要更新视图时仍要调用 `setData`。
> - `getApp()` 必须在 `App()` 注册之后调用，在 `app.js` 自身的 `onLaunch` 中应使用 `this` 而不是 `getApp()`。
> - 不要把大对象或频繁变化的数据放 `globalData`，它更适合放配置类、登录态等少量共享数据。大量共享数据建议用缓存（`wx.setStorage`）或状态管理方案。

## 11.3 获取服务端电影分类数据

### 11.3.1 封装请求函数

把请求逻辑封装到 `utils/http.js`，便于复用和维护：

```javascript
// utils/http.js
const app = getApp()

/**
 * 获取电影列表
 * @param {string} category - 分类：in_theaters / coming_soon / top250
 * @param {number} start - 起始位置
 * @param {number} count - 每页数量
 */
function getMovieList(category, start = 0, count = 3) {
  return new Promise((resolve, reject) => {
    wx.request({
      url: app.globalData.baseUrl + '/v2/movie/' + category,
      method: 'GET',
      data: { start, count },
      header: {
        'Content-Type': 'application/json'
      },
      success(res) {
        if (res.statusCode === 200) {
          resolve(res.data)
        } else {
          reject(new Error('请求失败，状态码：' + res.statusCode))
        }
      },
      fail(err) {
        reject(err)
      }
    })
  })
}

module.exports = {
  getMovieList
}
```

> **设计要点**：把 `wx.request` 包装成 Promise，后续可以用 `async/await` 优雅地调用，避免回调地狱。

### 11.3.2 在页面中调用

```javascript
// pages/movie/movie.js
const http = require('../../utils/http.js')

Page({
  data: {
    inTheaters: [],
    comingSoon: [],
    top250: []
  },
  onLoad() {
    this.loadAllMovies()
  },
  async loadAllMovies() {
    wx.showLoading({ title: '加载中' })
    try {
      // 并行请求三个分类
      const [inTheaters, comingSoon, top250] = await Promise.all([
        http.getMovieList('in_theaters', 0, 3),
        http.getMovieList('coming_soon', 0, 3),
        http.getMovieList('top250', 0, 3)
      ])
      this.setData({
        inTheaters: inTheaters.subjects,
        comingSoon: comingSoon.subjects,
        top250: top250.subjects
      })
    } catch (err) {
      console.error('加载失败', err)
      wx.showToast({ title: '加载失败', icon: 'none' })
    } finally {
      wx.hideLoading()
    }
  }
})
```

> **注意**：`Promise.all` 适合"多个请求都成功才继续"的场景。如果某个分类失败不影响其他分类展示，可以改为分别 `catch` 或使用 `Promise.allSettled`（需基础库 2.10+ 支持或自行 polyfill）。

### 11.4.3 字段映射的坑

后端返回的字段是 `subjects` 数组，每个元素的 `images.small` 是海报地址、`rating.average` 是评分。这些字段正好与 `movie` 组件中使用的字段一致，所以可以直接把 `subjects` 赋给 `movies`。

如果后端字段不一致（例如海报字段叫 `poster`、评分叫 `score`），就需要在请求成功后做一次字段映射：

```javascript
const mapped = res.data.subjects.map(item => ({
  id: item.id,
  title: item.title,
  year: item.year,
  images: { small: item.poster },
  rating: { average: item.score }
}))
this.setData({ inTheaters: mapped })
```

> **坑点提示**：字段映射是前后端联调时最常见的坑。建议在 `http.js` 的封装层统一做映射，让页面层只关心业务逻辑，不关心数据格式转换。

## 11.5 小程序中的异步处理模式

### 11.5.1 回调函数（传统方式）

`wx.request` 原生是回调风格的：

```javascript
wx.request({
  url: '...',
  success(res) { console.log(res) },
  fail(err) { console.error(err) }
})
```

多个请求串联时会出现"回调地狱"：

```javascript
wx.request({
  url: url1,
  success(res1) {
    wx.request({
      url: url2,
      success(res2) {
        wx.request({
          url: url3,
          success(res3) {
            // 嵌套越来越深
          }
        })
      }
    })
  }
})
```

### 11.5.2 Promise 方式

把 `wx.request` 包装成 Promise（如 11.3.1 所示），就能用 `.then` 链式调用：

```javascript
http.getMovieList('in_theaters', 0, 3)
  .then(data => {
    this.setData({ inTheaters: data.subjects })
    return http.getMovieList('coming_soon', 0, 3)
  })
  .then(data => {
    this.setData({ comingSoon: data.subjects })
    return http.getMovieList('top250', 0, 3)
  })
  .then(data => {
    this.setData({ top250: data.subjects })
  })
  .catch(err => {
    console.error(err)
  })
```

### 11.5.3 async/await 方式

`async/await` 是处理异步最优雅的方式，小程序基础库 2.10+ 已全面支持：

```javascript
async loadAllMovies() {
  try {
    const data1 = await http.getMovieList('in_theaters', 0, 3)
    this.setData({ inTheaters: data1.subjects })

    const data2 = await http.getMovieList('coming_soon', 0, 3)
    this.setData({ comingSoon: data2.subjects })

    const data3 = await http.getMovieList('top250', 0, 3)
    this.setData({ top250: data3.subjects })
  } catch (err) {
    console.error('加载失败', err)
  }
}
```

### 11.5.4 并行 vs 串行

上面的 `async/await` 写法是串行的（一个接一个），三个请求总耗时 = 请求1 + 请求2 + 请求3。如果三个请求互不依赖，应该用 `Promise.all` 并行：

```javascript
async loadAllMovies() {
  try {
    const [d1, d2, d3] = await Promise.all([
      http.getMovieList('in_theaters', 0, 3),
      http.getMovieList('coming_soon', 0, 3),
      http.getMovieList('top250', 0, 3)
    ])
    this.setData({
      inTheaters: d1.subjects,
      comingSoon: d2.subjects,
      top250: d3.subjects
    })
  } catch (err) {
    console.error(err)
  }
}
```

并行时总耗时 ≈ 最慢的那个请求，性能更好。

> **注意**：小程序有**并发请求数量限制**（同时最多 10 个 `wx.request`）。超过会被排队。首页 3 个请求远没到上限，但如果列表页同时发起大量请求要注意控制。

### 11.5.5 wx API 的 Promise 化

小程序从基础库 2.10.2 起支持 `wx.request` 等 API 直接返回 Promise（只需不传 success/fail 回调）。但并非所有 API 都支持，且低版本需兼容。常见做法是用 `promisify` 工具统一转换：

```javascript
// utils/promisify.js
function promisify(fn) {
  return function (options = {}) {
    return new Promise((resolve, reject) => {
      fn(Object.assign({}, options, {
        success: resolve,
        fail: reject
      }))
    })
  }
}

// 使用
const request = promisify(wx.request)
request({ url: '...' }).then(res => console.log(res))
```

## 11.7 组件化编程意义的探讨

通过电影首页的实现，可以深刻体会到组件化带来的价值：

### 11.7.1 复用性

`movie-list` 组件在首页使用了 3 次，只需传不同的 `title`、`category`、`movies`，无需重复编写列表结构。`movie` 组件会在首页、更多页、搜索页中复用。`stars` 组件会在 `movie` 内复用。一份代码，多处使用。

### 11.7.2 解耦与可维护性

组件内部的数据和逻辑是封闭的，修改 `stars` 的星级算法不会影响 `movie` 和 `movie-list`。页面只关心"拿数据 + 传给组件"，不关心组件内部如何渲染。这种关注点分离让大型项目的维护成本大幅降低。

### 11.7.3 团队协作

组件化让团队可以按组件分工：A 负责 `stars`，B 负责 `movie`，C 负责 `movie-list`，只要约定好属性和事件接口，就能并行开发互不阻塞。

### 11.7.4 可测试性

独立的组件更容易单独测试。可以写一个测试页面，传入不同的 `score` 验证 `stars` 的渲染是否正确，而不需要启动整个业务流程。

## 11.8 关于wx.request()的设置

`wx.request` 是小程序网络请求的核心 API，它的完整参数如下。

### 11.8.1 url

请求地址，必须是 HTTPS（开发阶段可关闭域名校验，见 11.9）。不能包含端口号（除非是开发环境）。

```javascript
wx.request({
  url: 'https://api.example.com/v2/movie/in_theaters'
})
```

### 11.8.2 method

请求方法，默认 `GET`。支持 `OPTIONS`、`GET`、`HEAD`、`POST`、`PUT`、`DELETE`、`TRACE`、`CONNECT`。最常用的是 `GET` 和 `POST`。

```javascript
wx.request({
  url: '...',
  method: 'POST'
})
```

### 11.8.3 data

请求参数。
- `GET` 请求：`data` 会作为 query string 拼接到 url 后面。
- `POST` 请求：`data` 作为请求体发送，格式由 `header.Content-Type` 决定。

```javascript
wx.request({
  url: '...',
  method: 'POST',
  data: {
    username: 'admin',
    password: '123456'
  }
})
```

### 11.8.4 header

请求头。常见用法：

```javascript
wx.request({
  url: '...',
  header: {
    'Content-Type': 'application/json',          // 默认值
    'Authorization': 'Bearer ' + token           // 携带鉴权 token
  }
})
```

> **注意**：`Content-Type` 默认是 `application/json`，会对 data 做 JSON 序列化。如果后端期望 `application/x-www-form-urlencoded` 格式，需要显式设置，否则后端解析不到参数。这是联调最常见的问题之一。

### 11.8.5 超时 timeout

```javascript
wx.request({
  url: '...',
  timeout: 10000   // 10秒超时
})
```

也可以在 `app.json` 中全局配置：

```json
{
  "networkTimeout": {
    "request": 10000,
    "connectSocket": 10000,
    "uploadFile": 10000,
    "downloadFile": 10000
  }
}
```

超时后会触发 `fail` 回调，`err.errMsg` 包含 `timeout` 字样。

### 11.8.6 并发限制

小程序同时最多 **10 个** `wx.request` 请求并发。超过会自动排队。如果业务上有批量请求需求，建议后端提供批量接口，或在客户端做请求队列管理。

### 11.8.7 完整示例

```javascript
wx.request({
  url: app.globalData.baseUrl + '/v2/movie/in_theaters',
  method: 'GET',
  data: { start: 0, count: 3 },
  header: { 'Content-Type': 'application/json' },
  timeout: 10000,
  enableHttp2: true,        // 开启 HTTP/2
  enableCache: false,       // 是否缓存
  success(res) {
    if (res.statusCode === 200) {
      console.log(res.data)
    } else if (res.statusCode === 401) {
      // token 失效，跳转登录
    } else {
      wx.showToast({ title: '服务异常', icon: 'none' })
    }
  },
  fail(err) {
    if (err.errMsg.indexOf('timeout') !== -1) {
      wx.showToast({ title: '请求超时', icon: 'none' })
    } else {
      wx.showToast({ title: '网络错误', icon: 'none' })
    }
  },
  complete() {
    wx.hideLoading()
  }
})
```

> **坑点提示**：`success` 回调不代表业务成功，只代表"网络请求返回了"。必须根据 `res.statusCode` 和业务数据中的错误码判断是否真正成功。

## 11.9 HTTP、HTTPS与可信域名

### 11.9.1 为什么必须是 HTTPS

出于安全考虑，小程序在正式环境中**只允许 HTTPS 请求**，不支持 HTTP。这意味着你接入的接口必须部署在配置了 SSL 证书的 HTTPS 域名上。HTTP 请求在真机上会被直接拦截。

### 11.9.2 服务器域名配置

在小程序管理后台（mp.weixin.qq.com）→ 开发 → 开发管理 → 开发设置 → 服务器域名中，配置 `request合法域名`：

```
https://api.example.com
```

要求：
- 域名必须已备案（境内服务器）。
- 必须是 HTTPS，且 SSL 证书有效。
- 域名不能使用 IP 地址或 localhost。
- 每月可修改 50 次域名配置。

### 11.9.3 开发环境跳过域名校验

开发阶段，后端可能还没上 HTTPS，可以在微信开发者工具中临时跳过校验：

**方法一**：开发者工具 → 详情 → 本地设置 → 勾选"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"。

**方法二**：在 `project.config.json` 中设置：

```json
{
  "setting": {
    "urlCheck": false
  }
}
```

> **注意**：`urlCheck: false` 只在开发工具中生效，真机预览时仍会校验。真机调试如果要用 HTTP 域名，可以在真机调试模式下勾选"不校验域名"。但**正式发布版必须配置合法 HTTPS 域名**，否则真机无法请求。

### 11.9.4 常见域名问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 开发工具能请求，真机不行 | 域名未配置或不满足 HTTPS | 后台配置合法域名 |
| 请求被拒绝："不在以下 request 合法域名列表中" | 域名未加入后台白名单 | 添加域名到 `request合法域名` |
| "证书错误" | SSL 证书过期/不匹配 | 更新证书 |
| `urlCheck` 已关但仍报错 | 改了 `project.config.json` 未重启工具 | 重启开发者工具 |

## 11.10 本章小结

本章把电影模块从 mock 数据升级为真实服务端数据，涵盖：

1. **全局变量 globalData**：用于存放 baseUrl、token 等全局配置，通过 `getApp()` 访问，非响应式需手动 `setData`。

2. **网络请求封装**：把 `wx.request` 包装成 Promise，集中管理 url、header、错误处理，便于复用。

3. **异步处理模式**：回调 → Promise → async/await，推荐 async/await + `Promise.all` 并行请求提升性能。

4. **wx.request 配置**：url/method/data/header/timeout/并发限制，注意 `Content-Type` 与后端匹配、`success` 不等于业务成功。

5. **HTTPS 与可信域名**：正式环境强制 HTTPS，需在后台配置合法域名；开发阶段可跳过校验但仅限开发工具。

6. **组件化意义**：复用、解耦、协作、可测试——电影首页用 3 个 `movie-list` 组件就完成了三个分类的展示，体现了组件化的核心价值。

下一章将在此基础上实现组件事件通信、电影搜索、下拉刷新、上滑加载更多等功能，进一步完善电影模块。
