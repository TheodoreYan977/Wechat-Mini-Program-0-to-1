# 第17章 小程序云开发实战

## 17.1 初始化云开发连接（wx.cloud.init env）

云开发是微信小程序提供的一套 Serverless 后端服务，包含云数据库、云存储、云函数三大核心能力。使用云开发前，必须先在小程序端初始化连接。

### 基本初始化

在 `app.js` 的 `onLaunch` 生命周期中调用 `wx.cloud.init()`：

```javascript
// app.js
App({
  onLaunch: function () {
    if (!wx.cloud) {
      console.error('请使用 2.2.3 或以上的基础库以使用云开发能力')
    } else {
      wx.cloud.init({
        // env 参数说明：
        // 云环境 ID，可在云开发控制台获取
        // 如果不填写，则使用第一个创建的云环境
        env: 'your-env-id',
        // traceUser: true 表示将用户访问记录到云开发控制台的用户管理中
        traceUser: true,
      })
    }
  }
})
```

### 关键参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| `env` | string | 云环境 ID，在云开发控制台创建环境后获得 |
| `traceUser` | boolean | 是否在用户访问时记录用户信息 |

> **注意事项**：
> 1. `wx.cloud.init()` 在小程序生命周期中只需调用一次，建议放在 `app.js` 的 `onLaunch` 中。
> 2. `env` 是云环境的唯一标识，一个小程序可以创建两个免费环境。开发时建议使用一个开发环境，一个生产环境。
> 3. 基础库版本需在 2.2.3 以上，否则 `wx.cloud` 为 `undefined`。

### 在 app.json 中声明云开发

确保 `project.config.json` 中已配置 `cloudfunctionRoot` 字段：

```json
{
  "cloudfunctionRoot": "cloudfunctions/",
  "miniprogramRoot": "miniprogram/",
  "setting": {
    "es6": true
  }
}
```

## 17.2 云开发——查询数据（db.collection().get()）

云开发数据库是一个 JSON 类型的文档数据库，每条记录就是一个 JSON 对象。

### 获取集合引用并查询

```javascript
// 获取数据库引用
const db = wx.cloud.database()

// 查询 collection 为 "articles" 的所有记录
db.collection('articles').get({
  success: function(res) {
    // res.data 是包含集合中所有记录的数组
    console.log('查询成功:', res.data)
  },
  fail: function(err) {
    console.error('查询失败:', err)
  }
})
```

### 在 Page 中使用

```javascript
Page({
  data: {
    articles: []
  },

  onLoad: function() {
    this.loadArticles()
  },

  loadArticles: function() {
    const db = wx.cloud.database()
    db.collection('articles').get({
      success: res => {
        this.setData({
          articles: res.data
        })
      },
      fail: err => {
        wx.showToast({
          title: '加载失败',
          icon: 'none'
        })
      }
    })
  }
})
```

### 分页查询

默认 `get()` 最多返回 20 条记录（云函数端为 100 条），需要分页时使用 `skip` 和 `limit`：

```javascript
const db = wx.cloud.database()

db.collection('articles')
  .skip(0)    // 跳过前 0 条
  .limit(10)  // 只获取 10 条
  .orderBy('createTime', 'desc')  // 按创建时间降序
  .get({
    success: res => {
      console.log(res.data)
    }
  })
```

> **常见坑**：小程序端 `limit` 最大值为 20，如果需要一次性获取更多数据，请使用云函数（最大 100 条/次）。

## 17.3 在小程序中使用async和await（regenerator-runtime）

微信小程序原生支持 `async/await` 语法，但需要引入 `regenerator-runtime` 来提供运行时支持（较新版本基础库已内置，但旧版本需手动引入）。

### 引入 regenerator-runtime

1. 在小程序的 `npm` 构建中安装：

```bash
npm install --save regenerator-runtime
```

2. 在需要使用 `async/await` 的页面或组件中引入：

```javascript
const regeneratorRuntime = require('regenerator-runtime/runtime')
// 或使用 ES Module 语法
// import regeneratorRuntime from 'regenerator-runtime'
```

### 用 async/await 改写回调

对比回调写法和 async/await 写法：

**回调地狱写法：**

```javascript
loadData: function() {
  const db = wx.cloud.database()
  db.collection('users').get({
    success: res => {
      db.collection('orders').where({
        userId: res.data[0]._id
      }).get({
        success: orderRes => {
          console.log('订单数据:', orderRes.data)
        }
      })
    }
  })
}
```

**async/await 写法（更清晰）：**

```javascript
const db = wx.cloud.database()

async function loadData() {
  try {
    // 获取用户
    const userRes = await db.collection('users').get()
    
    // 根据用户 ID 查询订单
    const orderRes = await db.collection('orders')
      .where({ userId: userRes.data[0]._id })
      .get()
    
    console.log('订单数据:', orderRes.data)
  } catch (err) {
    console.error('查询出错:', err)
  }
}
```

### 封装 Promise 化的数据库操作

```javascript
// utils/dbHelper.js
const db = wx.cloud.database()

/**
 * 查询集合数据
 */
async function queryCollection(collectionName, where = {}, limit = 20) {
  try {
    const res = await db.collection(collectionName)
      .where(where)
      .limit(limit)
      .get()
    return res.data
  } catch (err) {
    console.error(`查询 ${collectionName} 失败:`, err)
    throw err
  }
}

module.exports = {
  queryCollection
}
```

> **注意事项**：从基础库 2.10.2 开始，小程序 `wx.cloud` 相关 API 已原生返回 Promise，可以直接配合 `async/await` 使用，无需额外引入 `regenerator-runtime`。但如果项目中还有其他回调风格的 API（如 `wx.request`），可以用 `async/await` 配合手动 `Promise` 封装。

## 17.4 云开发数据库API入门（where/get/add/update/remove/count）

云开发数据库提供了完整的 CRUD API，核心方法包括 `get`、`add`、`update`、`remove`、`count`。

### 数据库结构概览

```
云开发数据库
├── collection（集合，类似数据库表）
│   └── document（文档/记录，类似一行数据）
│       └── field（字段，类似列）
```

### 五大核心 API 示例

```javascript
const db = wx.cloud.database()
const collection = db.collection('articles')
```

**1. 查询数据 — get()**

```javascript
// 查询全部
collection.get().then(res => {
  console.log('所有文章:', res.data)
})

// 条件查询
collection.where({
  category: 'tech'
}).get().then(res => {
  console.log('技术类文章:', res.data)
})
```

**2. 新增数据 — add()**

```javascript
collection.add({
  data: {
    title: '云开发入门指南',
    content: '这是一篇关于云开发的入门文章...',
    author: '张三',
    createTime: new Date()
  }
}).then(res => {
  console.log('新增成功，记录 ID:', res._id)
})
```

**3. 更新数据 — update()**

```javascript
// 通过文档 ID 更新
collection.doc('article_id_xxx').update({
  data: {
    title: '更新后的标题'
  }
}).then(res => {
  console.log('更新成功，影响记录数:', res.stats.updated)
})
```

**4. 删除数据 — remove()**

```javascript
collection.doc('article_id_xxx').remove().then(res => {
  console.log('删除成功，影响记录数:', res.stats.removed)
})
```

**5. 统计数量 — count()**

```javascript
collection.where({
  category: 'tech'
}).count().then(res => {
  console.log('技术类文章总数:', res.total)
})
```

### 权限说明

云开发数据库有四种权限级别：

| 权限 | 说明 |
|------|------|
| 仅创建者可读写 | 只有记录的创建者可以读写自己的记录 |
| 所有人可读，仅创建者可写 | 所有人可以读，但只有创建者可以修改 |
| 仅管理端可读写 | 只有云函数和管理后台可以操作 |
| 所有人可读写 | 所有人都可以读写（不推荐） |

> **常见坑**：小程序端默认受权限限制，只能操作当前用户创建的数据。如果需要跨用户操作数据，必须通过云函数以管理端权限执行。

## 17.5 云开发数据库的条件查询（eq/lt/gt/in/and/or命令符_.eq()）

简单查询使用对象字面量即可（如 `where({ age: 18 })`），但复杂查询需要使用数据库命令符。

### 引入数据库命令

```javascript
const db = wx.cloud.database()
const _ = db.command  // 获取数据库命令对象
```

### 比较命令

```javascript
// 等于 _.eq()
db.collection('users').where({
  age: _.eq(18)
})

// 不等于 _.neq()
db.collection('users').where({
  status: _.neq('deleted')
})

// 小于 _.lt() / 小于等于 _.lte()
db.collection('products').where({
  price: _.lt(100)  // 价格小于100
})

// 大于 _.gt() / 大于等于 _.gte()
db.collection('products').where({
  price: _.gt(50)  // 价格大于50
})
```

### 范围查询

```javascript
// in 查询 — 字段值在给定数组中
db.collection('articles').where({
  category: _.in(['tech', 'design', 'product'])
})

// nin — 不在数组中
db.collection('articles').where({
  category: _.nin(['spam', 'draft'])
})
```

### 逻辑组合命令（and / or）

```javascript
const _ = db.command

// AND — 多个条件同时满足
db.collection('products').where({
  price: _.gt(50).and(_.lt(200)),  // 50 < price < 200
  stock: _.gt(0)                    // 库存大于0
})

// 等价写法（对象字面量天然就是 AND）
db.collection('products').where({
  price: _.gt(50),
  stock: _.gt(0)
})

// OR — 满足任一条件
db.collection('articles').where(_.or([
  { category: 'tech' },
  { category: 'design' }
]))

// AND + OR 组合
db.collection('products').where({
  status: 'active',
  price: _.lt(100)
}).get()
```

### 实战示例：查询热门且在售的商品

```javascript
const db = wx.cloud.database()
const _ = db.command

async function getHotProducts() {
  const res = await db.collection('products').where(
    _.and([
      { status: 'onSale' },
      { sales: _.gt(500) },
      { rating: _.gte(4.5) }
    ])
  )
  .orderBy('sales', 'desc')
  .limit(10)
  .get()
  
  return res.data
}
```

> **注意事项**：
> 1. `_.eq()` 和直接写值效果相同，如 `where({ age: 18 })` 等价于 `where({ age: _.eq(18) })`。
> 2. 嵌套的 `and`/`or` 可以无限嵌套，但层数过多会影响查询性能。
> 3. `in` 命令的数组最多支持 500 个元素。

## 17.6 那些被微信自动创建的id（_id与_openid的自动生成机制）

在云开发数据库中，每条记录会自动包含两个特殊字段：`_id` 和 `_openid`。

### _id — 记录唯一标识

- **自动生成**：当使用 `add()` 插入数据且不指定 `_id` 时，系统会自动生成一个唯一的 `_id`。
- **格式**：默认生成的是一个递增的字符串 ID（如 `"f5a3a2a5e6b3d4"`）。
- **手动指定**：也可以在 `add()` 时手动指定 `_id`，但不能与已有记录重复。

```javascript
// 自动生成 _id
db.collection('articles').add({
  data: { title: '文章1' }
})
// 返回: { _id: "自动生成的ID", ... }

// 手动指定 _id
db.collection('articles').add({
  data: {
    _id: 'my-custom-id-001',
    title: '文章2'
  }
})
```

### _openid — 用户标识

- **自动填充**：当用户通过小程序调用 `add()` 时，系统会自动在该记录中添加 `_openid` 字段，值为当前用户的微信 `openid`。
- **用途**：用于标识数据的归属者，是实现"仅创建者可读写"权限的基础。
- **云函数中**：在云函数中使用 `add()` 时，`_openid` 不会自动填充（需要手动设置，或通过 `cloud.getWXContext()` 获取后写入）。

```javascript
// 小程序端新增 — _openid 自动填充
db.collection('favorites').add({
  data: {
    articleId: 'xxx',
    createTime: new Date()
  }
}).then(res => {
  // 记录中自动包含: { _id: "...", _openid: "用户openid", articleId: "...", createTime: ... }
  console.log('收藏成功:', res._id)
})
```

### 读取时利用 _openid 过滤

```javascript
// 在"仅创建者可读写"权限下，查询自动只返回当前用户的数据
db.collection('favorites').get().then(res => {
  // res.data 只包含当前 _openid 对应的记录
  console.log('我的收藏:', res.data)
})
```

> **常见坑**：
> 1. `update()` 和 `remove()` 在小程序端默认只能操作当前用户创建的记录（即 `_openid` 匹配的记录）。如果要操作他人记录，必须在云函数中以管理端权限执行。
> 2. `_id` 和 `_openid` 是保留字段，不能在 `data` 中覆盖 `_openid` 的值。
> 3. 使用 `doc(id)` 查询单条记录时，传入的 `id` 就是 `_id` 的值。

## 17.7 云开发版本的读取收藏状态

在文章详情页中，我们需要判断当前用户是否已收藏该文章，以决定收藏按钮的显示状态。

### 实现思路

1. 进入文章详情页时，查询 `favorites` 集合中是否存在 `_openid` 匹配且 `articleId` 匹配的记录。
2. 根据查询结果设置 `isFavorited` 状态。

### 代码实现

```javascript
// pages/article-detail/article-detail.js
const db = wx.cloud.database()

Page({
  data: {
    articleId: '',
    article: {},
    isFavorited: false
  },

  onLoad: function(options) {
    const articleId = options.id
    this.setData({ articleId })
    this.loadArticle(articleId)
    this.checkFavoriteStatus(articleId)
  },

  // 加载文章详情
  async loadArticle(articleId) {
    try {
      const res = await db.collection('articles').doc(articleId).get()
      this.setData({ article: res.data })
    } catch (err) {
      console.error('加载文章失败:', err)
    }
  },

  // 检查收藏状态
  async checkFavoriteStatus(articleId) {
    try {
      const res = await db.collection('favorites').where({
        articleId: articleId
        // _openid 会自动匹配当前用户（在"仅创建者可读写"权限下）
      }).get()
      
      this.setData({
        isFavorited: res.data.length > 0
      })
    } catch (err) {
      console.error('检查收藏状态失败:', err)
    }
  },

  // 切换收藏
  async toggleFavorite() {
    if (this.data.isFavorited) {
      // 取消收藏 — 删除记录
      await this.removeFavorite()
    } else {
      // 添加收藏 — 新增记录
      await this.addFavorite()
    }
  },

  async addFavorite() {
    try {
      await db.collection('favorites').add({
        data: {
          articleId: this.data.articleId,
          createTime: db.serverDate()
        }
      })
      this.setData({ isFavorited: true })
      wx.showToast({ title: '收藏成功' })
    } catch (err) {
      wx.showToast({ title: '收藏失败', icon: 'none' })
    }
  },

  async removeFavorite() {
    try {
      const res = await db.collection('favorites').where({
        articleId: this.data.articleId
      }).get()
      
      if (res.data.length > 0) {
        await db.collection('favorites').doc(res.data[0]._id).remove()
        this.setData({ isFavorited: false })
        wx.showToast({ title: '已取消收藏' })
      }
    } catch (err) {
      wx.showToast({ title: '取消失败', icon: 'none' })
    }
  }
})
```

### WXML 模板

```xml
<!-- pages/article-detail/article-detail.wxml -->
<view class="article-detail">
  <view class="article-content">
    <text>{{article.title}}</text>
    <rich-text nodes="{{article.content}}"></rich-text>
  </view>
  
  <view class="favorite-btn" bindtap="toggleFavorite">
    <text class="icon {{isFavorited ? 'favorited' : ''}}">★</text>
    <text>{{isFavorited ? '已收藏' : '收藏'}}</text>
  </view>
</view>
```

> **注意事项**：`db.serverDate()` 用于在服务端写入当前时间，避免客户端时间不准确的问题。

## 17.8 向云端数据库新增和更新记录（add/doc(id).update/set）

### 新增记录 — add()

```javascript
const db = wx.cloud.database()

// 新增一条文章记录
db.collection('articles').add({
  data: {
    title: '小程序云开发详解',
    content: '本文将详细介绍云开发的...',
    author: '李四',
    category: 'tech',
    tags: ['云开发', '小程序'],
    viewCount: 0,
    likeCount: 0,
    createTime: db.serverDate(),
    updateTime: db.serverDate()
  }
}).then(res => {
  console.log('新增成功，_id:', res._id)
}).catch(err => {
  console.error('新增失败:', err)
})
```

### 更新记录 — doc(id).update()

`update()` 只更新传入的字段，其他字段保持不变：

```javascript
// 更新单个文档
db.collection('articles').doc('article_id_xxx').update({
  data: {
    title: '更新后的标题',
    updateTime: db.serverDate()
  }
}).then(res => {
  console.log('更新成功，影响数量:', res.stats.updated)
})

// where 条件批量更新（仅云函数端支持，小程序端权限受限）
db.collection('articles').where({
  category: 'old_category'
}).update({
  data: {
    category: 'new_category'
  }
})
```

### set() — 替换记录

`set()` 与 `update()` 的区别在于：`set()` 会用传入的 `data` **完全替换**原有记录（`_id` 和 `_openid` 除外），而 `update()` 只做字段级更新。

```javascript
const db = wx.cloud.database()

// 使用 set 替换整条记录
db.collection('articles').doc('article_id_xxx').set({
  data: {
    title: '全新的标题',
    content: '全新的内容'
    // 注意：之前存在的其他字段会被删除！
  }
}).then(res => {
  console.log('设置成功:', res.stats.updated)
})
```

### update vs set 对比

| 特性 | update() | set() |
|------|----------|-------|
| 更新方式 | 字段级更新（合并） | 整条替换 |
| 未传字段 | 保持原值 | 被删除 |
| 适用场景 | 修改部分字段 | 重置整条记录 |

### 实战：浏览量自增

```javascript
// 使用 inc 命令实现原子自增
const _ = db.command

db.collection('articles').doc('article_id_xxx').update({
  data: {
    viewCount: _.inc(1),  // 浏览量 +1
    updateTime: db.serverDate()
  }
})
```

> **常见坑**：
> 1. 小程序端 `update`/`set`/`remove` 默认只能操作当前用户创建的记录。
> 2. `where().update()` 批量更新在小程序端不支持，需要在云函数中执行。
> 3. `set()` 如果文档不存在会报错，不会自动创建。如果需要"不存在则创建"的行为，请使用 `add()`。

## 17.9 插入评论数据

在文章详情页，用户可以对文章发表评论。评论数据需要包含文章 ID、评论内容、用户信息等。

### 评论数据结构设计

```json
{
  "_id": "评论记录ID",
  "_openid": "评论者的openid",
  "articleId": "文章ID",
  "content": "评论正文内容",
  "nickName": "评论者昵称",
  "avatarUrl": "评论者头像URL",
  "createTime": "2025-07-07T10:00:00.000Z"
}
```

### 实现代码

```javascript
// pages/article-detail/article-detail.js
const db = wx.cloud.database()

Page({
  data: {
    articleId: '',
    commentContent: '',
    comments: []
  },

  onLoad: function(options) {
    this.setData({ articleId: options.id })
    this.loadComments(options.id)
  },

  // 输入评论内容
  onCommentInput: function(e) {
    this.setData({ commentContent: e.detail.value })
  },

  // 提交评论
  async submitComment() {
    const content = this.data.commentContent.trim()
    if (!content) {
      wx.showToast({ title: '请输入评论内容', icon: 'none' })
      return
    }

    // 获取用户信息（需先授权）
    const userInfo = await this.getUserInfo()

    try {
      wx.showLoading({ title: '发表中...' })
      
      const res = await db.collection('comments').add({
        data: {
          articleId: this.data.articleId,
          content: content,
          nickName: userInfo.nickName,
          avatarUrl: userInfo.avatarUrl,
          createTime: db.serverDate()
        }
      })

      wx.hideLoading()
      wx.showToast({ title: '评论成功' })
      
      // 清空输入框并刷新评论列表
      this.setData({ commentContent: '' })
      this.loadComments(this.data.articleId)
    } catch (err) {
      wx.hideLoading()
      wx.showToast({ title: '评论失败', icon: 'none' })
      console.error('评论失败:', err)
    }
  },

  // 获取用户信息
  getUserInfo() {
    return new Promise((resolve) => {
      wx.getUserProfile({
        desc: '用于发表评论',
        success: res => resolve(res.userInfo),
        fail: () => resolve({ nickName: '匿名用户', avatarUrl: '' })
      })
    })
  },

  // 加载评论列表
  async loadComments(articleId) {
    try {
      const res = await db.collection('comments')
        .where({ articleId: articleId })
        .orderBy('createTime', 'desc')
        .limit(20)
        .get()
      
      this.setData({ comments: res.data })
    } catch (err) {
      console.error('加载评论失败:', err)
    }
  }
})
```

### WXML 模板

```xml

  <view class="comment-input-area">
    <textarea 
      placeholder="写下你的评论..."

      maxlength="200"

## 17.10 将图片等静态资源上传到小程序云存储（wx.cloud.uploadFile tempFilePath cloudPath）

云存储提供了文件上传、下载和管理能力，非常适合存储用户上传的图片、音频等文件。

### 上传单张图片

```javascript
Page({
  data: {
    imageUrl: ''
  },

  // 选择图片
  chooseImage: function() {
    wx.chooseMedia({
      count: 1,
      mediaType: ['image'],
      sourceType: ['album', 'camera'],
      success: res => {
        const tempFilePath = res.tempFiles[0].tempFilePath
        this.uploadImage(tempFilePath)
      }
    })
  },

  // 上传图片到云存储
  uploadImage: function(tempFilePath) {
    wx.showLoading({ title: '上传中...' })
    
    // 生成云存储路径
    const cloudPath = `images/${Date.now()}-${Math.floor(Math.random(0, 1) * 1000)}.jpg`
    
    wx.cloud.uploadFile({
      cloudPath: cloudPath,       // 云端存储路径
      filePath: tempFilePath,      // 本地临时文件路径
      success: res => {
        console.log('上传成功，fileID:', res.fileID)
        this.setData({ imageUrl: res.fileID })
        wx.hideLoading()
        wx.showToast({ title: '上传成功' })
      },
      fail: err => {
        console.error('上传失败:', err)
        wx.hideLoading()
        wx.showToast({ title: '上传失败', icon: 'none' })
      }
    })
  }
})
```

### 关键参数说明

| 参数 | 说明 |
|------|------|
| `cloudPath` | 文件在云存储中的路径，支持目录结构（如 `images/2025/07/xxx.jpg`） |
| `filePath` | 本地临时文件路径，通常来自 `wx.chooseMedia` 或 `wx.chooseImage` |
| `fileID` | 返回的文件唯一标识，可用于后续下载和展示 |

### 使用 fileID 展示图片

云存储的 `fileID` 可以直接在 `<image>` 组件的 `src` 属性中使用：

```xml
<image src="{{imageUrl}}" mode="aspectFill" />
```

### 下载云存储文件

```javascript
wx.cloud.downloadFile({
  fileID: 'cloud://your-env-id.xxxx/images/xxx.jpg',
  success: res => {
    // res.tempFilePath 是本地临时文件路径
    console.log('下载成功:', res.tempFilePath)
  },
  fail: err => {
    console.error('下载失败:', err)
  }
})
```

### 上传多张图片

```javascript
async uploadMultipleImages(tempFilePaths) {
  const uploadTasks = tempFilePaths.map((filePath, index) => {
    const cloudPath = `images/${Date.now()}-${index}.jpg`
    return wx.cloud.uploadFile({
      cloudPath: cloudPath,
      filePath: filePath
    })
  })

  try {
    const results = await Promise.all(uploadTasks)
    const fileIDs = results.map(r => r.fileID)
    console.log('所有图片上传成功:', fileIDs)
    return fileIDs
  } catch (err) {
    console.error('部分图片上传失败:', err)
  }
}
```

### 删除云存储文件

```javascript
wx.cloud.deleteFile({
  fileList: ['cloud://your-env-id.xxxx/images/xxx.jpg'],
  success: res => {
    console.log('删除成功:', res.fileList)
  }
})
```

> **常见坑**：
> 1. `cloudPath` 不能包含中文和特殊字符，建议只用字母、数字和连字符。
> 2. 云存储路径中相同的 `cloudPath` 会覆盖旧文件，建议用时间戳 + 随机数生成唯一路径。
> 3. `fileID` 的格式为 `cloud://环境ID.xxx/文件路径`，在小程序中可直接作为图片 `src` 使用。
> 4. 免费版云存储空间为 5GB，需注意空间用量。

## 17.11 上传用户音频消息

除了图片，云存储还可以上传音频文件，实现语音消息功能。

### 录音并上传

```javascript
// pages/voice-message/voice-message.js
const recorderManager = wx.getRecorderManager()

Page({
  data: {
    isRecording: false,
    audioFileID: '',
    duration: 0
  },

  onLoad: function() {
    // 监听录音结束事件
    recorderManager.onStop(res => {
      console.log('录音结束:', res)
      this.setData({
        duration: Math.floor(res.duration / 1000)
      })
      this.uploadAudio(res.tempFilePath)
    })

    // 监听录音错误
    recorderManager.onError(err => {
      console.error('录音出错:', err)
      wx.showToast({ title: '录音失败', icon: 'none' })
    })
  },

  // 开始录音
  startRecording: function() {
    recorderManager.start({
      duration: 60000,        // 最长录音时间（毫秒）
      sampleRate: 44100,      // 采样率
      numberOfChannels: 1,    // 录音通道数
      encodeBitRate: 192000,  // 编码码率
      format: 'mp3'           // 音频格式
    })
    this.setData({ isRecording: true })
  },

  // 停止录音
  stopRecording: function() {
    recorderManager.stop()
    this.setData({ isRecording: false })
  },

  // 上传音频到云存储
  uploadAudio: function(tempFilePath) {
    wx.showLoading({ title: '上传中...' })
    
    const cloudPath = `audio/${Date.now()}.mp3`
    
    wx.cloud.uploadFile({
      cloudPath: cloudPath,
      filePath: tempFilePath,
      success: res => {
        this.setData({ audioFileID: res.fileID })
        wx.hideLoading()
        wx.showToast({ title: '上传成功' })
        
        // 将 fileID 保存到数据库
        this.saveAudioRecord(res.fileID)
      },
      fail: err => {
        wx.hideLoading()
        wx.showToast({ title: '上传失败', icon: 'none' })
      }
    })
  },

  // 保存音频记录到数据库
  async saveAudioRecord(fileID) {
    try {
      await wx.cloud.database().collection('voice_messages').add({
        data: {
          fileID: fileID,
          duration: this.data.duration,
          createTime: wx.cloud.database().serverDate()
        }
      })
    } catch (err) {
      console.error('保存记录失败:', err)
    }
  },

  // 播放音频
  playAudio: function() {
    wx.cloud.downloadFile({
      fileID: this.data.audioFileID,
      success: res => {
        const innerAudioContext = wx.createInnerAudioContext()
        innerAudioContext.src = res.tempFilePath
        innerAudioContext.play()
      }
    })
  }
})
```

### WXML 模板

```xml
<view class="voice-container">
  <view class="record-area">
    <button 
      bindtouchstart="startRecording"
      bindtouchend="stopRecording"
      type="{{isRecording ? 'warn' : 'primary'}}"
    >
      {{isRecording ? '松开停止' : '按住录音'}}
    </button>
  </view>

  <view class="audio-player" wx:if="{{audioFileID}}">
    <text>音频时长: {{duration}}秒</text>
    <button size="mini" bindtap="playAudio">播放</button>
  </view>
</view>
```

> **注意事项**：
> 1. 录音功能需要在 `app.json` 中配置 `record` 权限：
>    ```json
>    { "permission": { "scope.record": { "desc": "用于录制语音消息" } } }
>    ```
> 2. 音频文件较大时上传可能较慢，建议添加上传进度监听（`wx.cloud.uploadFile` 返回 `uploadTask` 对象，可监听 `onProgressUpdate`）。
> 3. 录音格式推荐 `mp3`，兼容性最好。

## 17.12 使用云函数（创建云函数、wx.cloud.callFunction、cloud.getWXContext获取openid）

云函数是运行在云端的 Node.js 函数，具有管理端数据库权限，可以执行小程序端无法完成的操作。

### 创建云函数

1. 在项目根目录的 `cloudfunctions` 文件夹下，右键新建云函数，例如 `getOpenid`：

```
cloudfunctions/
  └── getOpenid/
      ├── index.js      # 云函数入口
      ├── package.json  # 依赖配置
      └── node_modules/  # 依赖包
```

2. **index.js** — 云函数代码：

```javascript
// cloudfunctions/getOpenid/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })

exports.main = async (event, context) => {
  // 获取微信上下文信息（包含 openid、appid 等）
  const wxContext = cloud.getWXContext()
  
  return {
    openid: wxContext.OPENID,
    appid: wxContext.APPID,
    unionid: wxContext.UNIONID || null,
    // event 包含从小程序端传入的参数
    eventData: event
  }
}
```

3. **package.json** — 依赖配置：

```json
{
  "name": "getOpenid",
  "version": "1.0.0",
  "description": "获取用户openid的云函数",
  "main": "index.js",
  "dependencies": {
    "wx-server-sdk": "~2.6.3"
  }
}
```

### 在小程序中调用云函数

```javascript
// 小程序端调用云函数
Page({
  data: {
    openid: ''
  },

  onLoad: function() {
    this.getOpenid()
  },

  getOpenid: function() {
    wx.cloud.callFunction({
      name: 'getOpenid',           // 云函数名称
      data: {                       // 传给云函数的参数
        someParam: 'hello'
      },
      success: res => {
        console.log('云函数返回:', res.result)
        this.setData({ openid: res.result.openid })
      },
      fail: err => {
        console.error('调用云函数失败:', err)
      }
    })
  }
})
```

### 使用 async/await 调用云函数

```javascript
async getOpenid() {
  try {
    const res = await wx.cloud.callFunction({
      name: 'getOpenid',
      data: { action: 'getInfo' }
    })
    console.log('openid:', res.result.openid)
    return res.result
  } catch (err) {
    console.error('调用失败:', err)
    throw err
  }
}
```

### cloud.getWXContext() 返回字段

| 字段 | 说明 |
|------|------|
| `OPENID` | 用户唯一标识（同一小程序内唯一） |
| `APPID` | 小程序 AppID |
| `UNIONID` | 用户在开放平台下的唯一标识（需绑定开放平台） |
| `ENV` | 云环境 ID |

### 云函数实战：带参数查询数据

```javascript
// cloudfunctions/searchArticles/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()

exports.main = async (event, context) => {
  const { keyword, page = 0, pageSize = 20 } = event
  
  const _ = db.command
  
  try {
    const result = await db.collection('articles')
      .where({
        title: db.RegExp({ regexp: keyword, options: 'i' })
      })
      .skip(page * pageSize)
      .limit(pageSize)
      .orderBy('createTime', 'desc')
      .get()
    
    return {
      code: 0,
      data: result.data,
      total: result.data.length
    }
  } catch (err) {
    return {
      code: -1,
      message: err.message
    }
  }
}
```

> **注意事项**：
> 1. 云函数部署后才能在小程序端调用。右键云函数文件夹 → "上传并部署：云端安装依赖"。
> 2. 云函数执行有超时限制（默认 3 秒，可在云开发控制台修改，最长 60 秒）。
> 3. 云函数中的数据库操作具有管理端权限，不受数据库安全规则限制。
> 4. 每次修改云函数代码后需要重新部署才能生效。

## 17.13 本章小结

本章全面介绍了小程序云开发的核心实战内容，涵盖以下知识点：

| 知识点 | 核心内容 |

**核心要点回顾**：

1. **云开发三大能力**：云数据库（JSON 文档数据库）、云存储（文件存储）、云函数（Node.js 运行环境），构成完整的 Serverless 后端。

2. **权限模型**：小程序端操作受安全规则限制（默认只能操作自己的数据），云函数端拥有管理端权限，可以跨用户操作。

3. **异步编程**：推荐使用 `async/await` 替代回调嵌套，使代码更清晰可维护。

4. **数据设计**：`_id` 是记录唯一标识，`_openid` 是用户标识，利用它们可以构建数据归属和权限体系。

5. **文件上传**：云存储支持图片、音频等多种文件类型，`fileID` 可直接在小程序中展示使用。
