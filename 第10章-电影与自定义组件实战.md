# 第10章 电影与自定义组件实战

本章通过一个电影模块的完整实现，把上一章学到的 Component 知识落地到真实场景中。我们将依次实现三个组件：星级评分组件 `stars`、单个电影卡片组件 `movie`、电影列表组件 `movie-list`，并在组件嵌套中体会组件化的威力。

## 10.1 电影模块结构分析

### 10.1.1 需求分析

电影模块首页要展示三个分类的电影列表："正在热映"、"即将上映"、"Top250"。每个分类显示前 3 部电影，点击"更多"跳转到对应分类的完整列表页。每个电影项要展示海报、标题、星级评分、评分分数。

### 10.1.2 组件拆分

根据"单一职责"原则，把 UI 拆成三个粒度的组件：

```
movie-list（电影列表组件，包含分类标题 + 列表容器）
└── movie（单个电影卡片组件，展示海报/标题/评分）
    └── stars（星级评分组件，把分数转换为星星）
```

目录结构：

```
components/
├── stars/
│   ├── stars.json
│   ├── stars.wxml
│   ├── stars.wxss
│   └── stars.js
├── movie/
│   ├── movie.json
│   ├── movie.wxml
│   ├── movie.wxss
│   └── movie.js
└── movie-list/
    ├── movie-list.json
    ├── movie-list.wxml
    ├── movie-list.wxss
    └── movie-list.js
```

> **设计要点**：组件粒度不是越细越好。`stars` 之所以独立出来，是因为它会被 `movie` 复用，未来也可能被搜索结果页复用；`movie` 独立出来是因为它会在首页列表、更多页、搜索结果页中反复使用。如果一个组件只在一处使用，过早抽象反而增加维护成本。

### 10.1.3 数据结构约定

后端返回的单个电影数据结构如下，后续组件开发都基于这个结构：

```javascript
{
  id: 1292052,
  title: '肖申克的救赎',
  images: { small: 'http://...small.jpg', large: 'http://...large.jpg' },
  rating: { average: 9.7, max: 10 },
  year: '1994',
  genres: ['犯罪', '剧情']
}
```

注意评分字段是嵌套对象 `rating.average`，范围 0~10。

## 10.2 编写stars组件（星级评分组件的实现）

星级评分组件的职责：接收一个 0~10 的分数，渲染成 5 颗星（每颗星代表 2 分），并显示具体分数。

### 10.2.1 stars.json

```json
{
  "component": true,
  "usingComponents": {}
}
```

### 10.2.2 stars.wxml

```html
<view class="stars">
  <block wx:for="{{starsArray}}" wx:key="index">
    <text class="star {{item}}">{{iconMap[item]}}</text>
  </block>
  <text class="score" wx:if="{{score > 0}}">{{score}}</text>
</view>
```

这里用 `iconMap` 把状态映射成具体的星号字符，便于后续切换成图标字体或图片。

### 10.2.3 stars.wxss

```css
.stars {
  display: flex;
  align-items: center;
}
.star {
  font-size: 28rpx;
  margin-right: 4rpx;
}
.star.full {
  color: #f5a623;
}
.star.half {
  color: #f5a623;
}
.star.empty {
  color: #cccccc;
}
.score {
  margin-left: 10rpx;
  font-size: 24rpx;
  color: #888;
}
```

### 10.2.4 stars.js

```javascript
Component({
  properties: {
    score: {
      type: Number,
      value: 0
    }
  },
  data: {
    starsArray: [],   // 例如 ['full','full','half','empty','empty']
    iconMap: {
      full: '★',
      half: '★',
      empty: '☆'
    }
  },
  observers: {
    'score': function (newVal) {
      this.setData({ starsArray: this.computeStars(newVal) })
    }
  },
  lifetimes: {
    attached() {
      this.setData({ starsArray: this.computeStars(this.data.score) })
    }
  },
  methods: {
    /**
     * 把 0~10 的分数转换为 5 个状态值
     * 每颗星代表 2 分
     */
    computeStars(score) {
      const result = []
      // 处理异常值
      if (!score || score <= 0) {
        return ['empty', 'empty', 'empty', 'empty', 'empty']
      }
      if (score > 10) score = 10

      // 转换为 5 分制
      const starScore = score / 2
      const fullCount = Math.floor(starScore)
      const hasHalf = starScore - fullCount >= 0.5
      const emptyCount = 5 - fullCount - (hasHalf ? 1 : 0)

      for (let i = 0; i < fullCount; i++) result.push('full')
      if (hasHalf) result.push('half')
      for (let i = 0; i < emptyCount; i++) result.push('empty')

      return result
    }
  }
})
```

> **设计要点**：
> - `computeStars` 放在 `methods` 中而不是 `data` 中，因为它是方法不是数据。
> - 同时在 `observers` 和 `attached` 中处理：`attached` 处理首次渲染，`observers` 处理后续属性变化。这样无论父组件何时传入分数都能正确显示。
> - 对异常值（`null`、负数、超过 10）做了防御，避免渲染出 6 颗星这种异常情况。

### 10.2.5 使用 stars 组件

```json
{
  "usingComponents": {
    "stars": "/components/stars/stars"
  }
}
```

```html
<stars score="{{9.7}}"></stars>
<stars score="{{item.rating.average}}"></stars>
```

> **坑点提示**：如果父组件传入的 `score` 是字符串（如从 `dataset` 取值），`type: Number` 会自动转换；但如果传入的是 `undefined`，转换后会变成 `NaN`，`computeStars` 中需要额外判断。上面代码中 `if (!score)` 已覆盖 `NaN` 的情况（`!NaN === true`）。

## 10.3 编写movie组件（单个电影卡片组件）

`movie` 组件负责展示单个电影的卡片：海报、标题、年份、星级评分。它内部会使用 `stars` 组件。

### 10.3.1 movie.json

```json
{
  "component": true,
  "usingComponents": {
    "stars": "/components/stars/stars"
  }
}
```

> **注意**：组件内部引用其他组件，也是在自身 json 的 `usingComponents` 中声明，与页面写法一致。

### 10.3.2 movie.wxml

```html
<view class="movie" bindtap="onMovieTap">
  <image class="poster" src="{{movie.images.small}}" mode="aspectFill" lazy-load></image>
  <view class="info">
    <view class="title">{{movie.title}}</view>
    <view class="meta">
      <stars score="{{movie.rating.average}}"></stars>
    </view>
    <view class="year">{{movie.year}}</view>
  </view>
</view>
```

### 10.3.3 movie.wxss

```css

}

  margin-top: 4rpx;
  font-size: 22rpx;
  color: #999;

```

### 10.3.4 movie.js

```javascript
Component({
  properties: {
    movie: {
      type: Object,
      value: {}
    }
  },
  methods: {
    onMovieTap() {
      // 点击卡片，向父组件抛出事件，由父组件决定跳转逻辑
      this.triggerEvent('movietap', {
        movieId: this.data.movie.id,
        title: this.data.movie.title
      })
    }
  }
})
```

> **设计要点**：组件本身不直接做页面跳转（`wx.navigateTo`），而是通过 `triggerEvent` 把点击事件抛给父级。这样组件保持"纯展示 + 事件通知"的职责，跳转逻辑由父级控制，组件复用性更强。比如同一组件在首页点击跳详情页，在搜索页点击可能只是收藏——逻辑由父级决定。

## 10.4 编写movie-list组件（电影列表组件，组件嵌套组件）

`movie-list` 组件负责展示一个分类的电影列表，包含分类标题（带"更多"按钮）和横向滚动的电影卡片列表。它内部循环使用 `movie` 组件，形成"组件嵌套组件"的结构。

### 10.4.1 movie-list.json

```json
{
  "component": true,
  "usingComponents": {
    "movie": "/components/movie/movie"
  }
}
```

### 10.4.2 movie-list.wxml

```html
<view class="movie-list">
  <!-- 分类标题栏 -->
  <view class="header">
    <text class="title">{{title}}</text>
    <view class="more" bindtap="onMoreTap" wx:if="{{movies.length > 0}}">
      <text>更多</text>
      <text class="arrow">›</text>
    </view>
  </view>

  <!-- 电影列表（横向滚动） -->
  <scroll-view scroll-x class="list" wx:if="{{movies.length > 0}}">
    <view class="list-inner">
      <movie
        wx:for="{{movies}}"
        wx:key="id"
        movie="{{item}}"
        bind:movietap="onMovieTap"
      ></movie>
    </view>
  </scroll-view>

  <!-- 加载中占位 -->
  <view class="loading" wx:if="{{movies.length === 0}}">
    <text>加载中...</text>
  </view>
</view>
```

> **注意**：横向滚动使用 `scroll-view scroll-x`，其子元素必须设置为 `inline-block` 或 `inline-flex`，并且不能换行（`white-space: nowrap`），否则无法横向滚动。这是常见坑点。

### 10.4.4 movie-list.js

```javascript
Component({
  properties: {
    title: {
      type: String,
      value: ''
    },
    movies: {
      type: Array,
      value: []
    },
    // 分类标识，用于跳转更多页时传递参数
    category: {
      type: String,
      value: ''
    }
  },
  methods: {
    onMoreTap() {
      // 抛出"更多"点击事件，由页面处理跳转
      this.triggerEvent('moretap', {
        category: this.data.category,
        title: this.data.title
      })
    },
    onMovieTap(e) {
      // 子组件 movie 抛上来的事件，继续向上抛
      this.triggerEvent('movietap', e.detail)
    }
  }
})
```

> **设计要点**：`movie-list` 把子组件 `movie` 的事件"转发"给自己的父级，这是组件嵌套时的常见模式——事件层层冒泡，数据层层下传。这种单向数据流让组件关系清晰可维护。

### 10.4.5 在页面中使用 movie-list

```json
{

    "movie-list": "/components/movie-list/movie-list"
```

```html
  movies="{{top250}}"
  bind:moretap="onMoreTap"
```

```javascript
// pages/movie/movie.js
Page({
  data: {
    inTheaters: [],
    comingSoon: [],
    top250: []
  },
  onLoad() {
    // 暂时用 mock 数据，下一章替换为真实接口
    this.setData({
      inTheaters: [
        { id: 1, title: '电影A', images: { small: '' }, rating: { average: 8.5 }, year: '2025' }
      ]
    })
  },
  onMoreTap(e) {
    const { category, title } = e.detail
    wx.navigateTo({
      url: `/pages/more-movie/more-movie?category=${category}&title=${title}`
    })
  },
  onMovieTap(e) {
    const { movieId, title } = e.detail
    wx.navigateTo({
      url: `/pages/movie-detail/movie-detail?id=${movieId}&title=${title}`
    })
  }
})
```

> **坑点提示**：`bind:movietap` 中的事件名要与子组件 `triggerEvent` 的第一个参数完全一致（小写）。事件名不支持驼峰，推荐全小写加连字符。

## 10.5 本章小结

本章通过电影模块实战，完成了三个组件的封装：

1. **stars 组件**：把 0~10 分转换为 5 颗星展示，是"纯展示型"组件的典型。学习了 `observers` 监听属性变化、`methods` 封装计算逻辑、异常值防御。

2. **movie 组件**：单个电影卡片，内部嵌套 `stars` 组件，并对外抛出 `movietap` 事件。学习了组件嵌套、`triggerEvent` 事件传递、职责分离（组件只通知不跳转）。

3. **movie-list 组件**：电影列表，内部循环嵌套 `movie` 组件，并向上转发子组件事件。学习了横向滚动布局、事件冒泡转发、组件间单向数据流。

核心收获：
- **组件设计的关键是职责单一**：每个组件只做一件事，通过属性接收数据、通过事件向上通信。
- **组件嵌套时事件层层转发**：子组件事件 → 中间组件转发 → 页面处理。保持数据向下、事件向上的单向流动。
- **横向滚动要用 `scroll-view scroll-x`**，子元素必须 `inline` 且不换行。
- **样式隔离**：组件 wxss 默认只作用于组件内部，不会污染页面，这是 Component 相比 template 的重要优势。
