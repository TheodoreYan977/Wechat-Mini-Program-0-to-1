# 第05章 文章详情页面

## 5.1 跳转到文章详情页面

在小程序中，页面之间的跳转通过路由API完成。最常用的跳转方式是`wx.navigateTo`，它会保留当前页面并跳转到目标页面，形成页面栈。

### 基本跳转

```javascript
// pages/posts/posts.js
Page({
  onPostTap(event) {
    wx.navigateTo({
      url: '/pages/post-detail/post-detail'
    });
  }
});
```

### 常用路由API对比

| API | 行为 | 是否可返回 | 页面栈变化 |
|-----|------|-----------|-----------|
| `wx.navigateTo` | 保留当前页，跳转到新页 | 是（wx.navigateBack） | push（栈深度+1） |
| `wx.redirectTo` | 关闭当前页，跳转到新页 | 否 | replace（替换栈顶） |
| `wx.switchTab` | 跳转到 tabBar 页面 | Tab栏切换 | 清空非Tab页 |
| `wx.reLaunch` | 关闭所有页面，打开新页 | 否 | 重置栈 |
| `wx.navigateBack` | 返回上一页 | — | pop（栈深度-1） |

### 路由跳转的URL写法

```javascript
// 绝对路径（推荐，从根目录开始）
wx.navigateTo({
  url: '/pages/post-detail/post-detail'
});

// 相对路径（容易出错，不推荐）
wx.navigateTo({
  url: '../post-detail/post-detail'
});
```

> **⚠️ 注意**：
> - `wx.navigateTo`的页面栈最多10层，超过会报错。深层级跳转应考虑使用`wx.redirectTo`或自定义组件切换。
> - 跳转到 tabBar 页面必须使用`wx.switchTab`，不能用`navigateTo`。
> - URL中不能包含特殊字符，参数需要用`encodeURIComponent`编码。

## 5.2 不要在template上注册事件

这是一个非常常见的初学者错误。在文章列表中，我们用`template`复用了文章卡片结构，但事件不应该绑定在`<template>`标签上。

### ❌ 错误写法

```xml
<!-- 错误：在template标签上绑定事件，事件不会触发 -->
<template is="postItem" data="{{...item}}" bindtap="onPostTap">
</template>
```

`<template>`是一个纯结构标签，渲染后不会生成真实节点，所以在它上面绑定的事件**不会被触发**。

### ✅ 正确写法：在外层view上绑定事件

```xml
<!-- 正确：用view包裹template，在view上绑定事件 -->
<view class="post-list">
  <block wx:for="{{postList}}" wx:key="id">
    <view class="post-item" bindtap="onPostTap" data-post-id="{{item.id}}">
      <template is="postItem" data="{{...item}}"></template>
    </view>
  </block>
</view>
```

### 为什么template上不能绑定事件

`<template>`标签在编译后会被替换为它内部的内容，本身不会渲染为任何DOM节点。事件的绑定依赖于真实节点，因此：

- `bindtap`、`catchtap`等事件绑定在`<template>`上无效
- `class`、`style`等属性写在`<template>`上也不生效

> **💡 实践建议**：如果模板内部需要交互，将事件绑定在包裹模板的外层`view`上，通过`data-*`属性传递数据。

## 5.3 页面间传递参数的3种方式

跳转到文章详情页时，需要告诉详情页"用户点击了哪篇文章"，即传递文章id。小程序页面间传参主要有以下方式。

### 思路：文章id的确定与传递

整体流程：

```
文章列表页                     文章详情页
  │                              │
  │  1. 用户点击文章卡片           │
  │  2. 获取被点击文章的id         │
  │  3. 通过URL参数传递id ───────→ 4. onLoad中接收id参数
  │                              │ 5. 根据id查找文章详情数据
  │                              │ 6. 渲染详情页面
```

### 方式一：URL查询参数（最常用）

通过跳转URL的查询字符串传递参数：

```javascript
// 列表页：传递参数
Page({
  onPostTap(event) {
    const postId = event.currentTarget.dataset.postId;
    wx.navigateTo({
      url: '/pages/post-detail/post-detail?id=' + postId
    });
  }
});
```

```javascript
// 详情页：接收参数
Page({
  onLoad(options) {
    // options.id 就是URL中?后面的id参数
    console.log('接收到的文章id：', options.id);
    this.loadPostDetail(options.id);
  },

  loadPostDetail(postId) {
    // 根据id查找文章数据...
  }
});
```

传多个参数：

```javascript
wx.navigateTo({
  url: '/pages/post-detail/post-detail?id=' + postId + '&title=' + encodeURIComponent(postTitle)
});
```

### 方式二：组件的自定义属性 data-*

要从被点击的元素上获取数据，使用`data-*`自定义属性：

```xml
<view class="post-list">
  <block wx:for="{{postList}}" wx:key="id">
    <!-- data-post-id 自定义属性，存储文章id -->
    <view 
      class="post-item" 
      bindtap="onPostTap" 
      data-post-id="{{item.id}}"
      data-index="{{index}}">
      <template is="postItem" data="{{...item}}"></template>
    </view>
  </block>
</view>
```

### data-* 的命名规则

`data-`后面的属性名使用**小写字母和连字符**（kebab-case），在JS中会自动转为**驼峰命名**：

```xml
<!-- WXML中写法（kebab-case） -->
<view data-post-id="1" data-user-name="zhangsan" bindtap="onTap">
```

```javascript
// JS中读取（自动转为驼峰）
onTap(event) {
  console.log(event.currentTarget.dataset.postId);   // 1
  console.log(event.currentTarget.dataset.userName); // zhangsan
}
```

| WXML写法 | JS中读取 | 说明 |
|----------|---------|------|
| `data-id` | `dataset.id` | 单词 |
| `data-post-id` | `dataset.postId` | 连字符转驼峰 |
| `data-postId` | `dataset.postid` | ⚠️大写会被转为小写 |
| `data-post-ID` | `dataset.postId` | ⚠️不推荐 |

> **⚠️ 坑点**：`data-*`属性名一律用小写+连字符（`data-post-id`），不要用驼峰（`data-postId`），否则读取时会因大小写转换产生困惑。

### 方式三：event事件对象

事件处理函数会收到一个`event`对象，其中包含触发事件的节点信息：

```javascript
Page({
  onPostTap(event) {
    // event 对象核心属性
    console.log(event.type);              // 事件类型：tap
    console.log(event.currentTarget);     // 当前组件（绑定事件的组件）
    console.log(event.target);            // 触发事件的组件（可能是子元素）
    console.log(event.detail);            // 额外信息（如x/y坐标）
    console.log(event.currentTarget.dataset);  // data-* 自定义属性集合

    // 获取文章id
    const postId = event.currentTarget.dataset.postId;
    console.log('文章id：', postId);
  }
});
```

### event 对象结构

```javascript
{
  type: 'tap',
  timeStamp: 1234567890,
  target: {
    id: '',
    dataset: { postId: 1 },
    offsetLeft: 0,
    offsetTop: 0
  },
  currentTarget: {
    id: '',
    dataset: { postId: 1 },
    offsetLeft: 0,
    offsetTop: 0
  },
  detail: { x: 120, y: 240 },
  touches: [...],
  changedTouches: [...]
}
```

### 获取页面参数值：onLoad的options参数

```javascript
// pages/post-detail/post-detail.js
Page({
  onLoad(options) {
    // options 包含URL中所有查询参数
    console.log(options);
    // { id: '1', title: '...' }

    const postId = options.id;
    this.setData({
      postId: postId
    });
    this.loadPostDetail(postId);
  }
});
```

> **💡 小结**：页面间传参的完整链路是——列表页用`data-*`存储id → 事件对象`currentTarget.dataset`读取id → 拼接到`navigateTo`的URL → 详情页`onLoad(options)`接收id。

## 5.4 编译模式与场景值

### 编译模式

在微信开发者工具中，"编译模式"决定小程序以什么入口启动。默认编译从首页启动，但调试详情页时每次都要先进入列表页再点击跳转，效率很低。

**自定义编译模式**可以指定启动页面和参数：

1. 点击开发者工具顶部"编译"旁的下拉箭头
2. 选择"添加编译模式"
3. 填写：
   - 模式名称：如"文章详情"
   - 启动页面：`pages/post-detail/post-detail`
   - 启动参数：`id=1`
4. 保存后选择该模式编译，直接进入详情页

```
启动参数填写格式：
id=1

多个参数：
id=1&title=测试
```

> **💡 实用技巧**：为每个需要调试的页面创建独立的编译模式，能大幅提升开发效率，特别是在开发深层级页面时。

### 场景值

场景值描述用户进入小程序的途径，比如扫码、搜索、分享等。在`App.onLaunch`和`App.onShow`中可以获取：

```javascript
// app.js
App({
  onLaunch(options) {
    console.log('场景值：', options.scene);
    // 1001: 发现栏小程序主入口
    // 1011: 扫描二维码
    // 1023: 来自好友分享
    // 1058: 公众号菜单
  },

  onShow(options) {
    console.log('进入场景：', options.scene);
  }
});
```

常见场景值：

| 场景值 | 含义 |
|-------|------|
| 1001 | 发现栏小程序主入口 |
| 1011 | 扫描二维码 |
| 1023 | 来自好友分享 |
| 1035 | 公众号自定义菜单 |
| 1058 | 公众号文章 |
| 1089 | 微信聊天主界面下拉 |

> **⚠️ 注意**：场景值仅在`App`的`onLaunch`和`onShow`中获取，页面中可通过`wx.getEnterOptionsSync()`获取。

## 5.5 读取文章详情数据

详情页拿到文章id后，需要从数据源中查找对应的文章详情。

### 从本地数据模块读取

```javascript
// data/data.js
const postList = [
  {
    id: 1,
    title: '深入理解小程序双线程模型',
    content: '小程序采用视图层和逻辑层分离...',
    detail: '这是文章的完整内容...',
    img: '/images/post/s-1.png',
    author: '张三',
    date: '2025-01-15',
    views: 1024,
    collection: 56
  },
  // ...更多文章
];

module.exports = { postList };
```

```javascript
// pages/post-detail/post-detail.js
const postData = require('../../data/data.js');

Page({
  data: {
    postId: '',
    postDetail: null
  },

  onLoad(options) {
    const postId = options.id;
    this.loadPostDetail(postId);
  },

  loadPostDetail(postId) {
    // 从数据源中查找id匹配的文章
    const post = postData.postList.find(item => {
      return item.id == postId;  // 注意：URL参数是字符串，用==或转Number
    });

    if (post) {
      this.setData({
        postId: postId,
        postDetail: post
      });
    } else {
      wx.showToast({
        title: '文章不存在',
        icon: 'none'
      });
    }
  }
});
```

> **⚠️ 注意**：`options.id`是字符串类型（来自URL），而数据中的`id`可能是数字类型。用`==`（宽松相等）或`Number(postId)`进行类型转换后再比较。

### 从服务器接口获取（进阶）

实际项目中通常从后端API获取：

```javascript
loadPostDetail(postId) {
  wx.request({
    url: 'https://api.example.com/posts/' + postId,
    method: 'GET',
    success: (res) => {
      if (res.statusCode === 200) {
        this.setData({
          postDetail: res.data
        });
      }
    },
    fail: (err) => {
      wx.showToast({
        title: '加载失败',
        icon: 'none'
      });
    }
  });
}
```

## 5.6 一张图理解小程序的事件与页面参数传递

```
┌──────────────────────────────────────────────────────┐
│                   列表页 (posts)                      │
│                                                      │
│  <view bindtap="onPostTap" data-post-id="{{id}}">    │
│                  │               │                   │
│           ① 绑定事件        ② 存储数据                 │
│                  │               │                   │
│                  ▼               ▼                   │
│         onPostTap(event) {                            │
│           const id = event.currentTarget.dataset.postId│
│                          │                           │
│                    ③ 读取数据                         │
│                          │                           │
│           wx.navigateTo({                            │
│             url: '/pages/detail/detail?id=' + id     │
│           })                  │                      │
│                        ④ 传递参数                     │
│         }                  │                         │
└────────────────────────────┼──────────────────────────┘
                             │
                             ▼
┌──────────────────────────────────────────────────────┐
│                  详情页 (detail)                      │
│                                                      │
│  onLoad(options) {                                   │
│         ⑤ 接收参数                                    │
│    const id = options.id                             │
│         ⑥ 查找数据                                    │
│    const post = postData.find(id)                    │
│         ⑦ 渲染页面                                    │
│    this.setData({ postDetail: post })                │
│  }                                                   │
│                                                      │
│  <view>{{postDetail.title}}</view>                   │
│  <view>{{postDetail.content}}</view>                 │
└──────────────────────────────────────────────────────┘
```

**七步链路总结**：

| 步骤 | 环节 | 技术 |
|------|------|------|
| ① | 绑定事件 | `bindtap` |
| ② | 存储数据 | `data-post-id` |
| ③ | 读取数据 | `event.currentTarget.dataset` |
| ④ | 传递参数 | `wx.navigateTo` URL查询参数 |
| ⑤ | 接收参数 | `onLoad(options)` |
| ⑥ | 查找数据 | `Array.find()` |
| ⑦ | 渲染页面 | `this.setData()` |

### JS逻辑

```javascript
// pages/post-detail/post-detail.js
const postData = require('../../data/data.js');

Page({
  data: {
    postId: '',
    postDetail: null,
    isCollected: false
  },

  onLoad(options) {
    const postId = options.id;
    this.loadPostDetail(postId);
    this.checkCollection(postId);
  },

  loadPostDetail(postId) {
    const post = postData.postList.find(item => item.id == postId);
    if (post) {
      this.setData({ postId, postDetail: post });
      // 动态设置导航栏标题
      wx.setNavigationBarTitle({ title: post.title });
    }
  },

  checkCollection(postId) {
    const collections = wx.getStorageSync('collections') || {};
    this.setData({
      isCollected: !!collections[postId]
    });
  },

  onCollectionTap() {
    const collections = wx.getStorageSync('collections') || {};
    const postId = this.data.postId;

    if (this.data.isCollected) {
      // 取消收藏
      collections[postId] = false;
      wx.showToast({ title: '已取消收藏', icon: 'none' });
    } else {
      // 添加收藏
      collections[postId] = true;
      wx.showToast({ title: '收藏成功', icon: 'success' });
    }

    wx.setStorageSync('collections', collections);
    this.setData({ isCollected: collections[postId] });
  },

  onShareTap() {
    // 转发提示
    wx.showToast({ title: '点击右上角转发', icon: 'none' });
  },

  onCommentTap() {
    wx.navigateTo({
      url: '/pages/comment/comment?id=' + this.data.postId
    });
  }
});
```

## 5.8 垂直居中问题的经典解决方法

在小程序开发中，垂直居中是最高频的布局需求之一。以下是几种经典解决方案。

### 方法一：Flex布局（推荐）

```css
/* 父容器使用flex */
.container {
  display: flex;
  flex-direction: column;    /* 或 row */
  justify-content: center;   /* 主轴居中 */
  align-items: center;       /* 交叉轴居中 */
  height: 200rpx;
}
```

```xml
<view class="container">
  <view class="item">居中内容</view>
</view>
```

这是最通用的方案，兼容性最好，代码最简洁。

### 方法二：Flex + margin:auto

```css
.parent {
  display: flex;
  height: 200rpx;
}

.child {
  margin: auto;  /* 在flex容器中自动居中 */
}
```

### 方法三：绝对定位 + transform

```css
.parent {
  position: relative;
  height: 200rpx;
}

.child {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
```

> **⚠️ 注意**：`transform: translate(-50%, -50%)`的百分比是相对于元素自身尺寸的，而`top: 50%`是相对于父容器的，两者配合实现完美居中。

### 方法四：绝对定位 + margin:auto（已知宽高）

```css
.parent {
  position: relative;
  height: 200rpx;
}

.child {
  position: absolute;
  width: 100rpx;
  height: 60rpx;
  top: 0;
  bottom: 0;
  left: 0;
  right: 0;
  margin: auto;
}
```

### 方法五：line-height（单行文本）

```css
/* 单行文本垂直居中，line-height等于容器高度 */
.single-line {
  height: 80rpx;
  line-height: 80rpx;
  text-align: center;
}
```

> **⚠️ 注意**：`line-height`方案仅适用于**单行文本**，多行文本会失效。

### 方法六：table-cell

```css
.parent {
  display: table-cell;
  vertical-align: middle;
  height: 200rpx;
  text-align: center;
}
```

### 实战：底部操作栏图标+文字垂直居中

```css
.footer-item {
  display: flex;
  flex-direction: column;
  align-items: center;       /* 水平居中 */
  justify-content: center;   /* 垂直居中 */
  height: 100rpx;
}

.footer-icon {
  width: 40rpx;
  height: 40rpx;
  margin-bottom: 6rpx;
}
```

### 各方案对比

| 方案 | 适用场景 | 是否需知宽高 | 兼容性 |
|------|---------|-------------|--------|
| Flex布局 | 通用 | 否 | 优秀 |
| Flex + margin:auto | 通用 | 否 | 优秀 |
| 绝对定位 + transform | 弹窗/浮层 | 否 | 优秀 |
| 绝对定位 + margin:auto | 固定尺寸元素 | 是 | 优秀 |
| line-height | 单行文本 | — | 优秀 |
| table-cell | 老式兼容 | 否 | 优秀 |

## 5.9 动态设置导航栏标题（wx.setNavigationBarTitle）

默认情况下，导航栏标题由`app.json`中页面的`navigationBarTitleText`配置决定。但在文章详情页，标题应该是当前文章的标题，这就需要动态设置。

### 基本用法

```javascript
Page({
  onLoad(options) {
    const post = this.findPost(options.id);
    if (post) {
      wx.setNavigationBarTitle({
        title: post.title
      });
    }
  }
});
```

### 完整示例

```javascript
Page({
  data: {
    postDetail: null
  },

  onLoad(options) {
    this.loadPostDetail(options.id);
  },

  loadPostDetail(postId) {
    const post = postData.postList.find(item => item.id == postId);
    if (post) {
      this.setData({ postDetail: post });

      // 动态设置导航栏标题
      wx.setNavigationBarTitle({
        title: post.title,
        success() {
          console.log('标题设置成功');
        },
        fail(err) {
          console.error('标题设置失败：', err);
        }
      });
    }
  }
});
```

### 配置页面导航栏（app.json或页面json）

```json
// pages/post-detail/post-detail.json
{
  "navigationBarTitleText": "文章详情",
  "navigationBarBackgroundColor": "#ffffff",
  "navigationBarTextStyle": "black"
}
```

> **💡 建议**：在页面json中设置一个默认标题（如"文章详情"），在`onLoad`中动态替换为真实文章标题。如果数据加载失败，至少有一个合理的默认标题。

### 动态设置导航栏颜色

```javascript
// 动态修改导航栏颜色
wx.setNavigationBarColor({
  frontColor: '#ffffff',     // 前景色（文字颜色），仅#ffffff或#000000
  backgroundColor: '#ff6700', // 背景色
  animation: {
    duration: 300,
    timingFunc: 'easeIn'
  }
});
```

> **⚠️ 注意**：`frontColor`只支持`#ffffff`（白色）和`#000000`（黑色）两个值，不能使用其他颜色。

## 5.10 本章小结

本章围绕"文章详情页面"的实现，串联了小程序开发中几个核心知识点：

### 知识点回顾

| 知识点 | 核心内容 |
|-------|---------|
| 页面跳转 | `wx.navigateTo`保留当前页跳转，页面栈最多10层 |
| template事件 | template是结构标签，不能绑定事件，需在外层view绑定 |
| 参数传递 | `data-*`存储数据 → `event.currentTarget.dataset`读取 → URL查询参数传递 → `onLoad(options)`接收 |
| 编译模式 | 自定义编译模式可指定启动页面和参数，提升调试效率 |
| 场景值 | `App.onLaunch`的`options.scene`标识用户进入途径 |
| 数据加载 | `Array.find()`根据id查找数据，注意字符串/数字类型转换 |
| 垂直居中 | Flex布局是最通用的垂直居中方案 |
| 动态标题 | `wx.setNavigationBarTitle`在`onLoad`中设置页面标题 |

### 事件与参数传递完整链路

```
data-* 属性 → event 对象 → navigateTo URL → onLoad options → 数据查找 → setData 渲染
```

### 常见坑点清单

1. **template上绑定事件无效**：事件必须绑定在真实渲染的节点上
2. **data-*命名混乱**：WXML中用`data-post-id`，JS中用`dataset.postId`（驼峰）
3. **URL参数类型问题**：`options.id`是字符串，比较时注意类型转换
4. **页面栈溢出**：`navigateTo`最多10层，深层跳转用`redirectTo`
5. **导航栏颜色限制**：`frontColor`仅支持黑白两色
