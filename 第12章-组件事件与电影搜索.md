# 第12章 组件事件与电影搜索

本章在电影模块的基础上，深入讲解组件的事件机制（内部事件与自定义事件），并完成"更多电影"页面、电影搜索功能、下拉刷新、上滑加载更多等实战特性，最后处理 `wx:key` 警告等细节问题。

## 12.1 组件的事件（组件内部事件）

组件内部的事件指的是组件 wxml 中绑定的事件，例如点击、输入、滑动等。这些事件的处理函数定义在组件 js 的 `methods` 中，与页面事件写法一致，但作用域是组件自身。

### 12.1.1 基本用法

```html
<!-- components/movie/movie.wxml -->
<view class="movie" bindtap="onMovieTap">
  <image src="{{movie.images.small}}" mode="aspectFill"></image>
  <view class="title">{{movie.title}}</view>
</view>
```

```javascript
// components/movie/movie.js
Component({
  properties: {
    movie: Object
  },
  methods: {
    onMovieTap() {
      console.log('卡片被点击了', this.data.movie.title)
      // 这里可以做一些组件内部的逻辑，比如高亮、动画
    }
  }
})
```

### 12.1.2 事件修饰与传参

与页面一样，组件内部事件可用 `data-*` 传参，用 `catch` 阻止冒泡：

```html
<view class="movie" bindtap="onMovieTap">
  <image src="{{movie.images.small}}" catchtap="onPosterTap" data-id="{{movie.id}}"></image>
  <view class="title">{{movie.title}}</view>
</view>
```

```javascript
methods: {
  onMovieTap() {
    console.log('整个卡片点击')
  },
  onPosterTap(e) {
    // catchtap 阻止了冒泡，onMovieTap 不会触发
    console.log('海报点击', e.currentTarget.dataset.id)
  }
}
```

> **注意**：`bindtap` 会冒泡，`catchtap` 不会冒泡。在组件内部有嵌套点击区域时，用 `catch` 防止内外事件同时触发。

### 12.1.3 组件事件的捕获阶段

小程序事件分捕获阶段和冒泡阶段，`capture-bind` / `capture-catch` 用于捕获阶段：

```html
<view capture-bind:tap="onCaptureTap" bindtap="onBubbleTap">
  <!-- 事件触发顺序：onCaptureTap → 子元素事件 → onBubbleTap -->
</view>
```

实际项目中捕获阶段用得较少，了解即可。

## 12.2 组件的自定义事件（triggerEvent）

组件内部事件只能被组件自己感知。当组件需要把信息通知父页面/父组件时，使用 `triggerEvent` 抛出自定义事件，这是组件向父级通信的核心机制。

### 12.2.1 triggerEvent 语法

```javascript
this.triggerEvent('事件名', 事件数据对象, 事件选项)
```

- **事件名**：全小写，推荐用连字符分隔，如 `movietap`、`more-tap`。不支持驼峰。
- **事件数据**：放在 `e.detail` 中传递给父级。
- **事件选项**：`{ bubbles: false, composed: false, capturePhase: false }`，默认全 false，一般不用。

### 12.2.2 在 movie 组件中抛出事件

```javascript
// components/movie/movie.js
Component({
  properties: {
    movie: Object
  },
  methods: {
    onMovieTap() {
      // 把电影 id 和 title 抛给父级
      this.triggerEvent('movietap', {
        movieId: this.data.movie.id,
        title: this.data.movie.title
      })
    }
  }
})
```

### 12.2.3 父组件监听自定义事件

在父级 wxml 中用 `bind:事件名` 或 `bind事件名` 监听：

```html
<!-- components/movie-list/movie-list.wxml -->
<movie
  wx:for="{{movies}}"
  wx:key="id"
  movie="{{item}}"
  bind:movietap="onMovieTap"
></movie>
```

```javascript
// components/movie-list/movie-list.js
Component({
  methods: {
    onMovieTap(e) {
      // e.detail 就是子组件 triggerEvent 传入的对象
      console.log('收到 movie 组件的事件', e.detail)
      // { movieId: 1292052, title: '肖申克的救赎' }

      // 继续向上抛给页面
      this.triggerEvent('movietap', e.detail)
    }
  }
})
```

### 12.2.4 页面监听事件

```html
<!-- pages/movie/movie.wxml -->
<movie-list
  movies="{{inTheaters}}"
  bind:movietap="onMovieTap"
  bind:moretap="onMoreTap"
></movie-list>
```

```javascript
// pages/movie/movie.js
Page({
  onMovieTap(e) {
    const { movieId, title } = e.detail
    wx.navigateTo({
      url: `/pages/movie-detail/movie-detail?id=${movieId}&title=${title}`
    })
  },
  onMoreTap(e) {
    const { category, title } = e.detail
    wx.navigateTo({
      url: `/pages/more-movie/more-movie?category=${category}&title=${title}`
    })
  }
})
```

### 12.2.5 事件通信全景图

```
[页面 movie] ← bind:movietap ← [movie-list] ← bind:movietap ← [movie]
     ↑                              ↑                              |
     |   setData 向下传数据          |  properties 向下传数据        | triggerEvent 向上抛事件
     |                              |                              v
     +------------------------------+------------------------------+
```

数据向下（properties），事件向上（triggerEvent），这是小程序组件通信的标准模式。

> **坑点提示**：
> - 事件名必须全小写，`movieTap` 不会被识别，应写成 `movietap` 或 `movie-tap`。
> - `bind:movietap` 和 `bindmovietap` 等价，前者更清晰推荐使用。
> - `e.detail` 才是子组件传的数据，`e.currentTarget.dataset` 是 wxml 上 `data-*` 的数据，两者不要混淆。

## 12.3 跳转到more-movie页面

电影首页点击"更多"按钮时，需要跳转到 `more-movie` 页面并携带分类参数。

### 12.3.1 movie-list 抛出 moretap 事件

```html
<!-- components/movie-list/movie-list.wxml -->
<view class="more" bindtap="onMoreTap">
  <text>更多</text>
  <text class="arrow">›</text>
</view>
```

```javascript
// components/movie-list/movie-list.js
Component({
  properties: {
    title: String,
    movies: Array,
    category: String
  },
  methods: {
    onMoreTap() {
      this.triggerEvent('moretap', {
        category: this.data.category,
        title: this.data.title
      })
    }
  }
})
```

### 12.3.2 页面处理跳转

```javascript
// pages/movie/movie.js
Page({
  onMoreTap(e) {
    const { category, title } = e.detail
    wx.navigateTo({
      url: `/pages/more-movie/more-movie?category=${category}&title=${encodeURIComponent(title)}`
    })
  }
})
```

> **注意**：`title` 可能包含中文，作为 URL 参数时建议用 `encodeURIComponent` 编码，在目标页用 `decodeURIComponent` 解码，避免特殊字符导致的参数解析错误。`wx.navigateTo` 只能跳转非 tabBar 页面；跳转 tabBar 页面要用 `wx.switchTab`。

### 12.3.3 注册页面

在 `app.json` 中注册页面路径：

```json
{
  "pages": [
    "pages/movie/movie",
    "pages/more-movie/more-movie"
  ]
}
```

`more-movie` 页面展示某个分类的完整电影列表，支持分页加载。

<view class="more-movie">

### 12.4.4 页面 js

```javascript
// pages/more-movie/more-movie.js
const http = require('../../utils/http.js')

Page({
  data: {
    category: '',
    title: '',
    movies: [],
    start: 0,
    count: 18,
    isLoading: false,
    noMore: false
  },
  onLoad(options) {
    const category = options.category
    const title = decodeURIComponent(options.title || '更多电影')
    this.setData({ category, title })
    wx.setNavigationBarTitle({ title })
    this.loadMovies(true)
  },
  async loadMovies(isRefresh) {
    if (this.data.isLoading) return
    if (!isRefresh && this.data.noMore) return

    this.setData({ isLoading: true })

    const start = isRefresh ? 0 : this.data.start
    try {
      const data = await http.getMovieList(this.data.category, start, this.data.count)
      const newMovies = data.subjects || []
      const total = data.total || 0

      this.setData({
        movies: isRefresh ? newMovies : this.data.movies.concat(newMovies),
        start: start + newMovies.length,
        noMore: this.data.movies.length + newMovies.length >= total
      })
    } catch (err) {
      wx.showToast({ title: '加载失败', icon: 'none' })
    } finally {
      this.setData({ isLoading: false })
      wx.stopPullDownRefresh()
    }
  },
  onMovieTap(e) {
    const { movieId, title } = e.detail
    wx.navigateTo({
      url: `/pages/movie-detail/movie-detail?id=${movieId}&title=${encodeURIComponent(title)}`
    })
  }
})
```

> **设计要点**：
> - `loadMovies(isRefresh)` 参数区分"下拉刷新"（从头加载）和"上滑加载"（追加加载）。
> - `isLoading` 防止重复请求，`noMore` 防止到底后无效请求。
> - `start` 记录已加载数量，作为下次请求的起点。

## 12.5 电影搜索功能

搜索功能是电影模块的重要特性。实现要点：搜索框、防抖优化、搜索结果展示。

### 12.5.1 搜索框实现

在 `more-movie` 页面顶部添加搜索框（也可以独立成搜索页）：

```html
<!-- pages/more-movie/more-movie.wxml -->
<view class="search-bar">
  <icon type="search" size="14" color="#999"></icon>
  <input
    class="search-input"
    type="text"
    placeholder="搜索电影"
    value="{{keyword}}"
    bindinput="onSearchInput"
    bindconfirm="onSearchConfirm"
    confirm-type="search"
  />
  <icon wx:if="{{keyword}}" type="clear" size="14" color="#999" bindtap="onClearTap"></icon>
</view>
```

```css
.search-bar {
  display: flex;
  align-items: center;
  background: #fff;
  border-radius: 8rpx;
  padding: 10rpx 20rpx;
  margin-bottom: 30rpx;
}
.search-input {
  flex: 1;
  margin: 0 16rpx;
  font-size: 28rpx;
  height: 60rpx;
}
```

### 12.5.2 防抖处理

用户输入时每次按键都会触发 `bindinput`，如果每次都发请求会造成大量无效请求和界面闪烁。**防抖（debounce）** 的作用是：只在用户停止输入一段时间后才发起请求。

```javascript
Page({
  data: {
    keyword: '',
    movies: [],
    isSearching: false
  },

  // 防抖定时器
  debounceTimer: null,

  onSearchInput(e) {
    const keyword = e.detail.value
    this.setData({ keyword })

    // 清除上次的定时器
    if (this.debounceTimer) {
      clearTimeout(this.debounceTimer)
    }

    // 500ms 后执行搜索
    this.debounceTimer = setTimeout(() => {
      if (keyword.trim()) {
        this.searchMovies(keyword)
      } else {
        this.setData({ movies: [], isSearching: false })
      }
    }, 500)
  },

  async searchMovies(keyword) {
    this.setData({ isSearching: true })
    try {
      const data = await http.searchMovies(keyword, 0, 18)
      this.setData({
        movies: data.subjects || [],
        start: (data.subjects || []).length,
        noMore: (data.subjects || []).length >= (data.total || 0)
      })
    } catch (err) {
      wx.showToast({ title: '搜索失败', icon: 'none' })
    } finally {
      this.setData({ isSearching: false })
    }
  },

  onSearchConfirm(e) {
    // 点击键盘搜索按钮，立即搜索（跳过防抖）
    const keyword = e.detail.value
    if (this.debounceTimer) clearTimeout(this.debounceTimer)
    if (keyword.trim()) {
      this.searchMovies(keyword)
    }
  },

  onClearTap() {
    this.setData({ keyword: '', movies: [], isSearching: false })
  }
})
```

在 `http.js` 中添加搜索接口：

```javascript
function searchMovies(keyword, start = 0, count = 18) {
  return new Promise((resolve, reject) => {
    wx.request({
      url: app.globalData.baseUrl + '/v2/movie/search',
      method: 'GET',
      data: { q: keyword, start, count },
      success(res) {
        res.statusCode === 200 ? resolve(res.data) : reject(new Error('status:' + res.statusCode))
      },
      fail(err) { reject(err) }
    })
  })
}

module.exports = { getMovieList, searchMovies }
```

> **防抖要点**：
> - 定时器 ID 存在 `this` 上（非 `data`），因为它是内部状态，不需要触发视图更新。
> - 每次输入先 `clearTimeout` 清除上次定时器，重新计时。
> - 防抖延迟一般 300~800ms，本例用 500ms。
> - `bindconfirm` 是用户点击键盘"搜索"键时触发，应跳过防抖立即搜索。
> - 组件中做防抖时，定时器变量不能写在 `data` 里，应直接作为组件实例属性（在 `created` 中初始化或直接 `this.timer`）。

## 12.6 实现页面的下拉刷新操作

下拉刷新是移动端的常见交互：用户在页面顶部下拉，触发数据重新加载。

### 12.6.1 开启下拉刷新

方式一：在页面 json 中配置（推荐）：

```json
{
  "enablePullDownRefresh": true,
  "backgroundColor": "#f5f5f5",
  "backgroundTextStyle": "dark"
}
```

方式二：在 `app.json` 的 `window` 中全局开启：

```json
{
  "window": {
    "enablePullDownRefresh": true
  }
}
```

> **注意**：`app.json` 全局开启会让所有页面都有下拉刷新，一般不推荐。建议在需要的页面单独开启。

### 12.6.2 处理下拉刷新事件

```javascript
Page({
  onPullDownRefresh() {
    // 重置分页状态
    this.setData({ start: 0, noMore: false })
    // 重新加载数据
    this.loadMovies(true)
    // loadMovies 内部 finally 中已调用 wx.stopPullDownRefresh()
  }
})
```

### 12.6.3 停止下拉刷新动画

数据加载完成后**必须**调用 `wx.stopPullDownRefresh()` 停止刷新动画，否则下拉动画会一直挂着：

```javascript
async loadMovies(isRefresh) {
  try {
    // ...
  } finally {
    wx.stopPullDownRefresh()   // 关键！
  }
}
```

> **坑点提示**：忘记调用 `wx.stopPullDownRefresh()` 是最常见的下拉刷新 bug，表现为下拉后一直转圈不复位。建议放在 `finally` 中确保一定执行。

## 12.7 JSON配置中的backgroundColor属性

下拉刷新时，页面顶部会露出 `backgroundColor` 配置的背景色。这个属性在页面 json 和 app.json 的 window 中都可以配置。

### 12.7.1 作用

`backgroundColor` 影响的是：
- 下拉刷新时露出的背景区域颜色
- 窗口的背景色（页面内容未铺满时的空白区域）

它**不影响**页面 wxml 根元素的实际背景色。也就是说，如果你的 `wxss` 给 `page` 设了背景色，用户看到的是 wxss 的颜色；但下拉时露出的还是 json 中的 `backgroundColor`。

### 12.7.2 配置

```json
{
  "backgroundColor": "#f5f5f5",
  "backgroundTextStyle": "dark",
  "enablePullDownRefresh": true
}
```

- `backgroundColor`：背景色，十六进制颜色值。
- `backgroundTextStyle`：下拉刷新 loading 文字样式，`dark`（深色）或 `light`（浅色）。根据背景色选择，保证可见性。

### 12.7.3 常见问题

如果你的页面是深色背景，下拉时露出白色会很突兀，需要把 `backgroundColor` 也设为深色，`backgroundTextStyle` 设为 `light`：

```json
{
  "backgroundColor": "#1a1a1a",
  "backgroundTextStyle": "light"
}
```

> **注意**：`backgroundColor` 只支持十六进制颜色，不支持 `rgb()`、`rgba()` 或颜色关键字（如 `red`）。

## 12.8 实现上滑加载更多数据

上滑加载更多是列表页的标配交互：用户滑到底部时自动加载下一页数据。

### 12.8.1 onReachBottom 事件

小程序提供了 `onReachBottom` 页面生命周期函数，当页面滚动到底部时自动触发：

```javascript
Page({
  data: {
    movies: [],
    start: 0,
    count: 18,
    isLoading: false,
    noMore: false
  },

  onReachBottom() {
    // 到底时加载更多
    this.loadMovies(false)
  },

  async loadMovies(isRefresh) {
    // 防止重复加载
    if (this.data.isLoading) return
    // 已加载全部则不再请求
    if (!isRefresh && this.data.noMore) return

    this.setData({ isLoading: true })

    const start = isRefresh ? 0 : this.data.start
    try {
      const data = await http.getMovieList(this.data.category, start, this.data.count)
      const newMovies = data.subjects || []
      const total = data.total || 0

      this.setData({
        movies: isRefresh ? newMovies : this.data.movies.concat(newMovies),
        start: start + newMovies.length,
        noMore: this.data.movies.length + newMovies.length >= total
      })
    } catch (err) {
      wx.showToast({ title: '加载失败', icon: 'none' })
    } finally {
      this.setData({ isLoading: false })
    }
  }
})
```

### 12.8.2 触底距离配置

`onReachBottom` 的触发距离可通过 `onReachBottomDistance` 配置（单位 px），表示距底部多少像素时触发：

```json
{
  "onReachBottom": true,
  "onReachBottomDistance": 50
}
```

默认 50px。值越小越晚触发（更接近底部），值越大越早触发。

> **注意**：`onReachBottom` 是页面级事件，只能在 `Page` 中使用，组件内无法直接监听。如果列表在组件内部（如 `scroll-view`），需要用 `scroll-view` 的 `bindscrolltolower` 事件代替。

### 12.8.3 scroll-view 的触底加载

如果列表区域是局部滚动（用 `scroll-view`），则用 `bindscrolltolower`：

```html
<scroll-view
  scroll-y
  style="height: 100vh"
  bindscrolltolower="onScrollToLower"
  lower-threshold="50"
>
  <view wx:for="{{movies}}" wx:key="id">{{item.title}}</view>
</scroll-view>
```

```javascript
Page({
  onScrollToLower() {
    this.loadMovies(false)
  }
})
```

### 12.8.4 加载状态提示

良好的体验是在底部显示加载状态：

```html
<view class="loading-more" wx:if="{{isLoading}}">
  <text>加载中...</text>
</view>
<view class="no-more" wx:if="{{noMore && movies.length > 0}}">
  <text>没有更多了</text>
</view>
```

> **设计要点**：
> - `isLoading` 防止 `onReachBottom` 重复触发导致并发请求。
> - `noMore` 在已加载条数 ≥ 总数时置 true，停止后续请求。
> - 追加数据用 `concat` 而非覆盖：`this.data.movies.concat(newMovies)`。

## 12.9 消除wx:key的警告提示

### 12.9.1 警告从何而来

在 `wx:for` 循环中如果不写 `wx:key`，开发者工具会报警告：

```
Now you can provide attr "wx:key" for a "wx:for" to improve performance.
```

这是因为没有 `wx:key` 时，当列表数据变化（增删改），小程序会重新渲染整个列表，无法复用已有节点，性能差且可能导致输入框失焦等问题。

### 12.9.2 正确写法

`wx:key` 有两种写法：

**写法一：使用 item 的某个唯一属性名（字符串）**

```html
<view wx:for="{{movies}}" wx:key="id">
  {{item.title}}
</view>
```

这里 `id` 是每个 movie 对象的唯一标识字段。注意 `wx:key="id"` 中的 `id` 是属性名，不要加 `{{}}`。

**写法二：使用 `*this`（当 item 本身就是唯一基本类型）**

```html
<view wx:for="{{['a','b','c']}}" wx:key="*this">
  {{item}}
</view>
```

### 12.9.3 常见错误写法

```html
<!-- 错误 1：用 index 作为 key -->
<view wx:for="{{movies}}" wx:key="index">...</view>
```

用 `index` 作 key 虽能消除警告，但当列表顺序变化（插入、删除、排序）时，`index` 不能稳定标识同一条数据，会导致渲染错乱。**只有在列表永不变化时才能用 index**。

```html
<!-- 错误 2：加了大括号 -->
<view wx:for="{{movies}}" wx:key="{{item.id}}">...</view>
```

`wx:key` 的值是属性名或 `*this`，**不能**用 `{{}}`，加了反而会失效。

```html
<!-- 错误 3：写了不存在的属性 -->
<view wx:for="{{movies}}" wx:key="key">...</view>
```

如果 item 对象没有 `key` 字段，警告虽消失但 key 机制无效。

### 12.9.4 组件循环中的 wx:key

```html
<movie wx:for="{{movies}}" wx:key="id" movie="{{item}}"></movie>
```

组件和普通元素一样，`wx:key` 写在组件标签上即可。

> **总结**：`wx:key` 不是可选项，是必须项。始终为 `wx:for` 提供稳定的唯一 key，用数据项的 `id` 字段，避免用 `index`。

## 12.10 本章小结

本章完善了电影模块的事件通信与交互功能：

1. **组件事件**：组件内部事件用 `bindtap` 等绑定，处理函数定义在 `methods` 中；`catch` 阻止冒泡。

2. **自定义事件 triggerEvent**：组件通过 `this.triggerEvent('事件名', 数据)` 向父级通信，父级用 `bind:事件名` 监听，数据在 `e.detail` 中。数据向下（properties），事件向上（triggerEvent）是标准通信模式。

3. **页面跳转**：`wx.navigateTo` 携带参数跳转，中文参数用 `encodeURIComponent` 编码；目标页 `onLoad(options)` 接收参数。

4. **more-movie 页面**：展示分类完整列表，支持分页加载，用 `isLoading` / `noMore` 控制加载状态。

5. **搜索功能**：搜索框 + `bindinput` + **防抖**（500ms 延时，`clearTimeout` + `setTimeout`），`bindconfirm` 立即搜索。

6. **下拉刷新**：页面 json `enablePullDownRefresh: true`，`onPullDownRefresh` 处理，**必须** `wx.stopPullDownRefresh()` 停止动画。

7. **backgroundColor**：影响下拉刷新露出的背景区域，支持十六进制色值，配合 `backgroundTextStyle` 调整 loading 文字可见性。

8. **上滑加载更多**：`onReachBottom` 页面事件（或 `scroll-view` 的 `bindscrolltolower`），用 `concat` 追加数据，`isLoading` 防重复，`noMore` 防越界。

9. **wx:key**：`wx:for` 必须配 `wx:key`，用数据项的唯一 `id` 字段，不加 `{{}}`，不用 `index`（除非列表永不变）。

至此，电影模块已具备完整的列表展示、详情跳转、搜索、分页、下拉刷新、上滑加载等功能。这套组件化 + 网络请求 + 事件通信的模式，可以推广到小程序中绝大多数列表类业务场景。
