# 第08章 完善文章页面

## 8.1 分享功能

### 为什么需要分享

分享是小程序获取新用户的核心渠道。微信提供了多种分享能力：

| 分享方式 | 触发方式 | API/配置 |
|---------|---------|---------|
| 转发给好友/群聊 | 右上角菜单 → 转发 / 按钮触发 | `onShareAppMessage` |
| 分享到朋友圈 | 右上角菜单 → 分享到朋友圈 | `onShareTimeline` |
| 分享图片到聊天 | 生成分享海报 | `wx.shareImageMessage` |
| 分享视频到聊天 | 分享视频内容 | `wx.shareVideoMessage` |

### 开启分享

默认情况下，小程序页面右上角菜单的"转发"选项可用，但"分享到朋友圈"需要在页面中定义`onShareTimeline`才会显示。

```javascript
// pages/post-detail/post-detail.js
Page({
  // 转发给好友
  onShareAppMessage() {
    return {
      title: this.data.postDetail.title,
      path: '/pages/post-detail/post-detail?id=' + this.data.postId,
      imageUrl: this.data.postDetail.img
    };
  },

  // 分享到朋友圈
  onShareTimeline() {
    return {
      title: this.data.postDetail.title,
      query: 'id=' + this.data.postId,
      imageUrl: this.data.postDetail.img
    };
  }
});
```

### 分享卡片预览

用户点击转发后，会弹出一个分享卡片，卡片包含：

```
┌─────────────────────────┐
│  ┌─────────┐            │
│  │  封面图  │  文章标题    │  ← title
│  │ imageUrl │  小程序名称   │
│  └─────────┘            │
└─────────────────────────┘
```

> **💡 提示**：如果不配置`imageUrl`，小程序会自动截取当前页面作为分享卡片图片。配置自定义图片可以更好地控制分享效果。

## 8.2 onShareAppMessage()详解

### 完整参数

```javascript
Page({
  onShareAppMessage(res) {
    // res.from: 触发来源
    //   'button' — 页面内按钮触发
    //   'menu'   — 右上角菜单触发
    // res.target: 如果是button触发，target是触发分享的button对象
    // res.webviewUrl: 页面中包含web-view组件时，返回web-view的URL

    return {
      title: '分享标题',           // 分享标题
      path: '/pages/xxx/xxx?id=1', // 分享路径（打开后进入的页面）
      imageUrl: '/images/share.png', // 分享图片（5:4比例）
      promise: undefined           // 异步分享（见下方说明）
    };
  }
});
```

### 参数详解

#### title - 分享标题

```javascript
// 简单文字
title: '这是一篇好文章'

// 动态拼接
title: this.data.postDetail.title + ' - ' + this.data.postDetail.author

// 带引导语
title: '🔥 ' + this.data.postDetail.title
```

> **⚠️ 注意**：标题不能超过35个字符，超出会被截断。避免使用特殊符号和表情过多。

#### path - 分享路径

```javascript
// ✅ 正确：绝对路径 + 查询参数
path: '/pages/post-detail/post-detail?id=' + this.data.postId

// ✅ 正确：带多个参数
path: '/pages/post-detail/post-detail?id=' + id + '&from=share'

// ❌ 错误：相对路径
path: '../post-detail/post-detail?id=1'

// ❌ 错误：缺少开头的 /
path: 'pages/post-detail/post-detail?id=1'
```

> **⚠️ 关键**：`path`必须是绝对路径（以`/`开头）。接收方通过`onLoad(options)`获取参数，与普通页面跳转一致。

#### image - 分享图片

```javascript
// 本地图片
imageUrl: '/images/share-default.png'

// 网络图片
imageUrl: 'https://example.com/images/share.png'

// 文章封面图
imageUrl: this.data.postDetail.img

// 动态生成的分享海报
imageUrl: this.data.shareImageUrl
```

**图片规格**：

| 属性 | 要求 |
|------|------|
| 比例 | 5:4（推荐） |
| 最小尺寸 | 500×400 px |
| 格式 | PNG / JPG |
| 文件大小 | 网络图片不超过1MB |

> **💡 提示**：如果不设置`imageUrl`，微信会自动截取当前页面生成分享图。但自动截图可能不够美观，建议配置专用分享图。

### 区分分享来源

```javascript
onShareAppMessage(res) {
  if (res.from === 'button') {
    // 页面内按钮触发
    console.log('来自按钮分享', res.target);
    const postId = res.target.dataset.postId;
    return {
      title: '快来看看这篇文章',
      path: '/pages/post-detail/post-detail?id=' + postId,
      imageUrl: '/images/share-btn.png'
    };
  } else {
    // 右上角菜单触发
    return {
      title: this.data.postDetail.title,
      path: '/pages/post-detail/post-detail?id=' + this.data.postId,
      imageUrl: this.data.postDetail.img
    };
  }
}
```

### 异步分享（promise）

当分享内容需要从服务器获取时，使用`promise`：

```javascript
onShareAppMessage() {
  const promise = new Promise(resolve => {
    // 模拟从服务器获取分享信息
    wx.request({
      url: 'https://api.example.com/share-info',
      data: { postId: this.data.postId },
      success(res) {
        resolve({
          title: res.data.title,
          path: res.data.path,
          imageUrl: res.data.imageUrl
        });
      }
    });
  });

  return {
    title: '正在加载分享内容...',
    path: '/pages/post-detail/post-detail?id=' + this.data.postId,
    promise: promise  // promise resolve后会更新分享内容
  };
}
```

> **⚠️ 注意**：`promise`方式适用于分享内容需要异步获取的场景。设置`promise`后，用户点击转发时，会等待`promise` resolve后再弹出分享面板。

## 8.3 分享到朋友圈（onShareTimeline）

### 基本用法

```javascript
Page({
  onShareTimeline() {
    return {
      title: this.data.postDetail.title,     // 分享标题
      query: 'id=' + this.data.postId,       // 查询参数（注意：不是path）
      imageUrl: this.data.postDetail.img      // 分享图片
    };
  }
});
```

### 与onShareAppMessage的区别

| 特性 | onShareAppMessage（转发好友） | onShareTimeline（朋友圈） |
|------|---------------------------|------------------------|
| 触发方式 | 菜单/按钮 | 仅菜单 |
| 参数传递 | `path`（完整路径+参数） | `query`（仅查询参数） |
| 打开方式 | 直接进入小程序页面 | 单页模式打开 |
| 图片比例 | 5:4 | 1:1 |
| 是否可带按钮触发 | 是（open-type="share"） | 否 |

### query参数

朋友圈分享不使用`path`，而是使用`query`传递参数：

```javascript
// ❌ 错误：朋友圈分享不能使用path
onShareTimeline() {
  return {
    title: '文章标题',
    path: '/pages/post-detail/post-detail?id=1'  // 不支持
  };
}

// ✅ 正确：使用query传递参数
onShareTimeline() {
  return {
    title: '文章标题',
    query: 'id=1&from=timeline'  // 仅传查询参数
  };
}
```

打开分享到朋友圈的小程序时，页面通过`onLoad(options)`接收参数：

```javascript
Page({
  onLoad(options) {
    console.log(options.id);       // '1'
    console.log(options.from);     // 'timeline'
  }
});
```

### 朋友圈单页模式

分享到朋友圈的小程序，打开后是**单页模式**（单页应用），特点：

- 只展示当前页面，无法跳转到其他页面
- 页面顶部显示"来自：小程序名称"
- 不显示底部tabBar
- 导航栏有"前往小程序"入口

```javascript
// 检测是否在单页模式下打开
Page({
  onLoad(options) {
    if (options.from === 'timeline') {
      // 朋友圈单页模式
      this.setData({ isSinglePage: true });
      wx.setNavigationBarTitle({
        title: '文章详情（完整内容请前往小程序）'
      });
    }
  }
});
```

```xml
<!-- 单页模式下显示引导 -->
<view wx:if="{{isSinglePage}}" class="single-page-tip">
  <text>查看完整内容，请点击右上角"前往小程序"</text>
</view>
```

> **⚠️ 注意**：分享到朋友圈的功能需要在页面json中开启，或确保页面中定义了`onShareTimeline`函数。

### 朋友圈分享图片

```javascript
onShareTimeline() {
  return {
    title: this.data.postDetail.title,
    query: 'id=' + this.data.postId,
    imageUrl: this.data.postDetail.img  // 1:1比例的图片
  };
}
```

朋友圈分享图片推荐**1:1**正方形比例，最小尺寸300×300px。

## 8.4 两种分享模式（主动分享与被动分享）

### 主动分享

用户主动点击"转发"或"分享到朋友圈"：

```javascript
// 主动分享不需要特殊处理，定义好onShareAppMessage即可
Page({
  onShareAppMessage() {
    return {
      title: this.data.postDetail.title,
      path: '/pages/post-detail/post-detail?id=' + this.data.postId
    };
  }
});
```

### 被动分享（按钮触发）

通过`<button open-type="share">`触发分享，适合做自定义分享入口：

```xml
<!-- 页面内分享按钮 -->
<button 
  open-type="share"
  class="share-btn"
  data-post-id="{{postId}}">
  <image src="/images/icon/share.png" class="share-icon"></image>
  <text>分享给好友</text>
</button>
```

```javascript
Page({
  onShareAppMessage(res) {
    if (res.from === 'button') {
      // 按钮触发
      const postId = res.target.dataset.postId;
      return {
        title: '推荐一篇好文章给你',
        path: '/pages/post-detail/post-detail?id=' + postId,
        imageUrl: '/images/share-invite.png'
      };
    } else {
      // 菜单触发
      return {
        title: this.data.postDetail.title,
        path: '/pages/post-detail/post-detail?id=' + this.data.postId
      };
    }
  }
});
```

### 两种模式对比

| 特性 | 主动分享（菜单） | 被动分享（按钮） |
|------|----------------|----------------|
| 触发方式 | 右上角菜单 | 页面内按钮 |
| 触发入口 | 通用 | 自定义UI引导 |
| res.from | `'menu'` | `'button'` |
| 分享内容 | 可统一配置 | 可按按钮区分 |
| 用户体验 | 隐蔽，需找到菜单 | 直观，引导性强 |
| 适用场景 | 通用 | 营销活动、邀请奖励 |

## 8.5 微信开放能力解析（open-type按钮能力）

### 什么是open-type

`<button>`组件的`open-type`属性可以触发微信内置的开放能力，无需自己实现复杂逻辑。

### 常用open-type值

| open-type值 | 功能 | 说明 |
|-------------|------|------|
| `share` | 触发转发 | 等同于点击右上角转发 |
| `getPhoneNumber` | 获取手机号 | 需企业资质，返回加密手机号 |
| `getUserInfo` | 获取用户信息 | 头像、昵称等（已废弃，改用头像昵称填写） |
| `chooseAvatar` | 选择头像 | 用户选择微信头像 |
| `contact` | 客服会话 | 打开客服聊天 |
| `openSetting` | 打开设置页 | 引导用户开启权限 |
| `feedback` | 意见反馈 | 打开反馈页面 |
| `launchApp` | 打开APP | 从小程序打开关联的APP |
| `chooseAddress` | 获取地址 | 获取用户收货地址 |

### share - 转发分享

```xml
<button open-type="share" class="custom-share-btn">
  分享给好友
</button>
```

### getPhoneNumber - 获取手机号

```xml
<button 
  open-type="getPhoneNumber" 
  bindgetphonenumber="onGetPhoneNumber"
  class="phone-btn">
  一键登录
</button>
```

```javascript
Page({
  onGetPhoneNumber(event) {
    // event.detail.errMsg: 是否成功
    // event.detail.encryptedData: 加密数据
    // event.detail.code: 动态令牌（推荐方式）

    if (event.detail.errMsg === 'getPhoneNumber:ok') {
      // 用户同意授权
      const code = event.detail.code;

      // 将code发送到服务器，服务器调用微信接口解密手机号
      wx.request({
        url: 'https://api.example.com/phone',
        method: 'POST',
        data: { code: code },
        success(res) {
          console.log('手机号：', res.data.phoneNumber);
        }
      });
    } else {
      // 用户拒绝
      console.log('用户拒绝授权手机号');
    }
  }
});
```

> **⚠️ 注意**：
> - `getPhoneNumber`需要小程序已完成微信认证（企业主体）。
> - 2023年10月起，推荐使用`code`方式获取手机号，不再直接返回`encryptedData`。
> - 获取手机号是收费接口，每次调用收取费用。

## 8.6 事件对象中target和currentTarget的区别

### 概念定义

- **target**：触发事件的**原始**组件（事件冒泡的源头）
- **currentTarget**：事件绑定的**当前**组件（绑定事件的元素本身）

### 经典场景

```xml
<view 
  class="outer" 
  bindtap="onOuterTap"
  data-name="outer">
  
  <view 
    class="inner" 
    bindtap="onInnerTap"
    data-name="inner">
    
    <text class="text">点击我</text>
    
  </view>
</view>
```

当用户点击`<text>`元素时：

| 事件处理函数 | event.target | event.currentTarget |
|-------------|-------------|---------------------|
| onInnerTap | `<text>`（触发源） | `<view class="inner">`（绑定者） |
| onOuterTap | `<text>`（触发源，冒泡上来） | `<view class="outer">`（绑定者） |

### 代码验证

```javascript
Page({
  onInnerTap(event) {
    console.log('--- inner tap ---');
    console.log('target:', event.target.dataset.name);        // undefined（text没有data-name）
    console.log('currentTarget:', event.currentTarget.dataset.name);  // 'inner'
    console.log('target id:', event.target.id);                // text的id
    console.log('currentTarget id:', event.currentTarget.id);  // inner的id
  },

  onOuterTap(event) {
    console.log('--- outer tap ---');
    console.log('target:', event.target.dataset.name);         // undefined（来自text）
    console.log('currentTarget:', event.currentTarget.dataset.name);   // 'outer'
  }
});
```

### 用图理解

```
点击 <text>
    │
    │  target = <text>  ←──── 事件源头
    │
    ▼
<view class="inner" bindtap="onInnerTap">
    │  currentTarget = <view class="inner">  ←──── 绑定事件的元素
    │
    │  (事件冒泡)
    ▼
<view class="outer" bindtap="onOuterTap">
    │  currentTarget = <view class="outer">  ←──── 绑定事件的元素
    │  target 仍然是 <text>  ←──── 源头不变
    ▼
```

### 实际应用场景

#### 场景1：列表项点击

```xml
<view class="list-item" bindtap="onItemTap" data-id="{{item.id}}">
  <image src="{{item.img}}"></image>
  <text>{{item.title}}</text>
  <view class="delete-btn" catchtap="onDelete" data-id="{{item.id}}">×</view>
</view>
```

```javascript
Page({
  onItemTap(event) {
    // ✅ 用 currentTarget 获取列表项id
    // 因为用户可能点击了image、text或空白区域
    // target可能是image或text，currentTarget始终是list-item
    const id = event.currentTarget.dataset.id;
    console.log('点击了列表项：', id);
  },

  onDelete(event) {
    // ✅ 用 catchtap 阻止冒泡，删除按钮不会触发onItemTap
    const id = event.currentTarget.dataset.id;
    console.log('删除列表项：', id);
  }
});
```

> **💡 关键结论**：当需要获取绑定事件的元素上的`data-*`数据时，**始终用`event.currentTarget.dataset`**，不要用`event.target.dataset`。因为`target`可能是子元素，没有对应的`data-*`属性。

#### 场景2：阻止冒泡

```xml
<!-- bindtap：会冒泡 -->
<view class="mask" bindtap="onCloseModal">
  <view class="modal" bindtap="onStopPropagation">
    <!-- 点击modal内部不会关闭弹窗 -->
    <text>弹窗内容</text>
  </view>
</view>

<!-- catchtap：阻止冒泡（更简洁） -->
<view class="mask" bindtap="onCloseModal">
  <view class="modal" catchtap="onStopPropagation">
    <text>弹窗内容</text>
  </view>
</view>
```

```javascript
Page({
  onCloseModal() {
    this.setData({ showModal: false });
  },

  onStopPropagation() {
    // 空函数，仅阻止事件冒泡
    // 或者处理modal内部逻辑
  }
});
```

### bind vs catch

| 事件绑定方式 | 行为 |
|-------------|------|
| `bindtap` | 触发事件，**继续冒泡** |
| `catchtap` | 触发事件，**阻止冒泡** |
| `mut-bind` | 互斥绑定（少见） |
| `capture-bind:tap` | 捕获阶段触发 |
| `capture-catch:tap` | 捕获阶段触发并阻止 |

```xml
<!-- catchtap 阻止冒泡 -->
<view bindtap="onOuter">
  <view catchtap="onInner">
    点击这里只触发onInner，不会触发onOuter
  </view>
</view>
```

### 完整的事件对象

```javascript
{
  type: 'tap',                    // 事件类型
  timeStamp: 1234567890,          // 时间戳
  target: {                       // 事件源
    id: 'text-element',
    dataset: {},                  // target上的data-*
    offsetLeft: 10,
    offsetTop: 20,
    offsetWidth: 100,
    offsetHeight: 30
  },
  currentTarget: {                // 当前绑定元素
    id: 'list-item',
    dataset: { id: '1', name: '文章' },
    offsetLeft: 0,
    offsetTop: 50,
    offsetWidth: 350,
    offsetHeight: 100
  },
  detail: { x: 120, y: 240 },    // 事件细节（tap事件为坐标）
  touches: [{                     // 触摸点信息
    clientX: 120,
    clientY: 240,
    pageX: 120,
    pageY: 240,
    identifier: 0
  }],
  changedTouches: [...]           // 变化的触摸点
}
```

### target与currentTarget对比总结

| 特性 | target | currentTarget |
|------|--------|---------------|
| 定义 | 事件触发的源头 | 事件绑定的元素 |
| 冒泡中是否变化 | 不变（始终是源头） | 变化（随冒泡层级变化） |
| 获取data-*的可靠性 | 低（可能是子元素） | 高（始终是绑定元素） |
| 使用场景 | 判断点击了哪个子元素 | 获取绑定元素的数据 |

> **⚠️ 核心记忆**：
> - 需要`data-*`数据 → 用`event.currentTarget.dataset`
> - 需要判断具体点了哪个子元素 → 用`event.target`
> - 需要阻止冒泡 → 用`catchtap`替代`bindtap`

### 事件对象使用原则

```

```

### 常见坑点

1. **朋友圈分享用path**：朋友圈分享不支持`path`，必须用`query`
2. **target取不到data**：`event.target`可能是子元素，获取`data-*`要用`event.currentTarget`
3. **getPhoneNumber需认证**：个人小程序无法使用，需企业认证
4. **分享路径写错**：`path`必须以`/`开头的绝对路径
5. **单页模式限制**：朋友圈分享打开是单页模式，不能跳转其他页面
6. **catchtap误用**：在需要冒泡的场景误用`catchtap`会导致父级事件不触发
