```javascript
// pages/post-detail/post-detail.js
const postData = require('../../data/data.js');

Page({
  data: {
    postId: '',
    postDetail: null,

    this.setData({
      isCollected: !!collections[postId]
    });
  }
});
```

## 6.2 文章收藏功能

### 条件渲染 wx:if 与 wx:else

小程序提供`wx:if`、`wx:elif`、`wx:else`进行条件渲染：

```xml
<view class="footer-item" bindtap="onCollectionTap">
  <!-- 根据收藏状态显示不同图标 -->
  <block wx:if="{{isCollected}}">
    <image src="/images/icon/star-active.png" class="footer-icon"></image>
    <text class="active-text">已收藏</text>
  </block>
  <block wx:else>
    <image src="/images/icon/star.png" class="footer-icon"></image>
    <text>收藏</text>
  </block>
</view>
```

也可以用三元表达式简化：

```xml
<view class="footer-item {{isCollected ? 'active' : ''}}" bindtap="onCollectionTap">
  <image 
    src="{{isCollected ? '/images/icon/star-active.png' : '/images/icon/star.png'}}" 
    class="footer-icon">
  </image>
  <text>{{isCollected ? '已收藏' : '收藏'}}</text>
</view>
```

### 实现收藏单击功能

```javascript
// pages/post-detail/post-detail.js
Page({
  data: {
    postId: '',
    postDetail: null,
    isCollected: false
  },

  onCollectionTap() {
    // 1. 获取当前收藏状态
    const postsCollected = wx.getStorageSync('collections') || {};

    // 2. 切换收藏状态
    const postId = this.data.postId;
    postsCollected[postId] = !postsCollected[postId];

    // 3. 更新缓存
    wx.setStorageSync('collections', postsCollected);

    // 4. 更新页面数据
    this.setData({
      isCollected: postsCollected[postId]
    });

    // 5. 提示用户
    wx.showToast({
      title: postsCollected[postId] ? '收藏成功' : '已取消收藏',
      icon: postsCollected[postId] ? 'success' : 'none',
      duration: 1500
    });
  }
});
```

### 交互反馈 wx.showToast

`wx.showToast`是轻量级的提示方式，适合操作反馈：

```javascript
wx.showToast({
  title: '收藏成功',        // 提示内容
  icon: 'success',         // 图标：success/error/loading/none
  duration: 2000,          // 显示时长（毫秒）
  mask: false,             // 是否显示透明蒙层，防止触摸穿透
  image: '/images/custom.png'  // 自定义图片（覆盖icon）
});
```

| icon值 | 效果 | 适用场景 |
|--------|------|---------|
| `success` | 显示成功图标 | 操作成功 |
| `error` | 显示失败图标 | 操作失败 |
| `loading` | 显示加载图标 | 异步加载中 |
| `none` | 无图标，纯文字 | 普通提示 |

> **⚠️ 注意**：
> - `wx.showToast`与`wx.hideToast`配合使用，`loading`图标需要手动调用`hideToast`关闭。
> - `title`文字最多7个汉字，超出会被截断。
> - `icon`为`none`时，文字可显示两行。

## 6.3 本地缓存的重要性及应用举例

### 为什么需要本地缓存

小程序的本地缓存（Storage）是持久化存储在用户设备上的数据，即使小程序被关闭，数据依然保留。它在实际开发中有广泛应用：

| 应用场景 | 说明 |
|---------|------|
| 用户偏好设置 | 记住夜间模式、字体大小等偏好 |
| 收藏/点赞状态 | 离线记录用户收藏的文章 |
| 登录状态 | 缓存token，避免重复登录 |
| 草稿保存 | 评论草稿、表单临时数据 |
| 数据预加载 | 首页数据缓存，提升打开速度 |
| 离线数据 | 无网络时展示缓存数据 |

### 缓存API

```javascript
// 同步API（推荐，代码更直观）
wx.setStorageSync('key', value);      // 写入
const data = wx.getStorageSync('key'); // 读取
wx.removeStorageSync('key');           // 删除指定key
wx.clearStorageSync();                 // 清空所有缓存

// 异步API（数据量大时使用，避免阻塞）
wx.setStorage({
  key: 'key',
  data: value,
  success() {
    console.log('写入成功');
  }
});

wx.getStorage({
  key: 'key',
  success(res) {
    console.log('读取到：', res.data);
  }
});
```

### 应用举例：收藏状态管理

```javascript
// 收藏数据结构设计
// collections: { "1": true, "2": false, "3": true }
// key是文章id，value是是否收藏

// 收藏管理工具类
const collectionUtil = {
  // 获取所有收藏
  getAll() {
    return wx.getStorageSync('collections') || {};
  },

  // 判断是否已收藏
  isCollected(postId) {
    const collections = this.getAll();
    return !!collections[postId];
  },

  // 切换收藏状态
  toggle(postId) {
    const collections = this.getAll();
    collections[postId] = !collections[postId];
    wx.setStorageSync('collections', collections);
    return collections[postId];
  },

  // 获取收藏列表（所有已收藏的文章id）
  getCollectedIds() {
    const collections = this.getAll();
    return Object.keys(collections).filter(id => collections[id]);
  }
};

module.exports = collectionUtil;
```

```javascript
// 在页面中使用
const collectionUtil = require('../../utils/collection.js');

Page({
  onCollectionTap() {
    const isCollected = collectionUtil.toggle(this.data.postId);
    this.setData({ isCollected });
    wx.showToast({
      title: isCollected ? '收藏成功' : '已取消收藏',
      icon: isCollected ? 'success' : 'none'
    });
  }
});
```

### 应用举例：评论草稿保存

```javascript
Page({
  data: {
    draftContent: ''
  },

  onInput(event) {
    this.data.draftContent = event.detail.value;
  },

  // 页面卸载时保存草稿
  onUnload() {
    if (this.data.draftContent) {
      wx.setStorageSync('comment_draft_' + this.data.postId, this.data.draftContent);
    }
  },

  onLoad(options) {
    // 恢复草稿
    const draft = wx.getStorageSync('comment_draft_' + options.id);
    if (draft) {
      this.setData({ draftContent: draft });
    }
  },

  // 提交后清除草稿
  onSubmit() {
    // ...提交逻辑
    wx.removeStorageSync('comment_draft_' + this.data.postId);
  }
});
```

> **⚠️ 缓存限制**：
> - 单个key上限为1MB，所有缓存上限为10MB。
> - `wx.setStorageSync`的`data`可以是任意可序列化的类型（对象、数组、字符串、数字、布尔值）。
> - 不要在缓存中存储敏感信息（密码、token等可加密后存储）。

```javascript
Page({
  data: {
    images: []
  },

  onChooseImage() {
    wx.chooseMedia({
      count: 9 - this.data.images.length,  // 最多选9张
      mediaType: ['image'],                 // 媒体类型
      sourceType: ['album', 'camera'],      // 相册或拍照
      sizeType: ['compressed'],             // 压缩图
      success: (res) => {

        const images = this.data.images.concat(
          tempFiles.map(file => file.tempFilePath)
        );
        this.setData({ images });
      },
      fail(err) {
        console.log('取消选择：', err);
      }
    });
  }
});
```

### 在评论页获取数据

```javascript
// pages/comment/comment.js
const { commentList } = require('../../data/commentData.js');

Page({
  data: {
    postId: '',
    commentList: []
  },

  onLoad(options) {
    const postId = options.id;
    this.loadComments(postId);
  },

  loadComments(postId) {
    const comments = commentList[postId] || [];

    // 按时间倒序排列（最新在前）
    comments.sort((a, b) => {
      return new Date(b.createTime) - new Date(a.createTime);
    });

    this.setData({
      postId: postId,
      commentList: comments
    });
  }
});
```

### 从服务器接口获取（进阶）

```javascript
loadComments(postId) {

    success: (res) => {
      if (res.statusCode === 200) {
        this.setData({ commentList: res.data.list });
      }
    },
    complete() {
      wx.hideLoading();
    }
  });
}
```

## 6.8 previewImage实现图片预览

`wx.previewImage`是小程序内置的图片预览功能，支持双指缩放、左右滑动浏览。

### 基本用法

```javascript
Page({
  onPreviewImage(event) {
    // 从data-*获取当前点击的图片URL和图片列表
    const current = event.currentTarget.dataset.current;
    const urls = event.currentTarget.dataset.urls;

    wx.previewImage({
      current: current,   // 当前显示图片的URL
      urls: urls,          // 需要预览的图片URL列表
      showmenu: true       // 长按弹出菜单（保存等）
    });
  }
});
```

### WXML绑定

```xml
<image 
  src="{{img}}" 
  class="comment-image"
  data-current="{{img}}"
  data-urls="{{item.images}}"
  bindtap="onPreviewImage">
</image>
```

### 关键参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| `urls` | Array | 必填，需要预览的图片链接列表 |
| `current` | String | 选填，当前显示的图片URL（必须是urls中的某一项） |

## 6.10 wx:if与hidden控制元素的显示和隐藏

### 两种方式的区别

```xml
<!-- wx:if：通过创建/销毁节点控制 -->
<view wx:if="{{isVisible}}">wx:if控制的元素</view>

<!-- hidden：通过CSS display控制 -->
<view hidden="{{!isVisible}}">hidden控制的元素</view>
```

### 核心区别

| 特性 | wx:if | hidden |
|------|-------|--------|
| 实现方式 | 创建/移除DOM节点 | CSS `display:none` |
| 初始渲染 | `false`时不渲染 | 始终渲染，只是隐藏 |
| 切换开销 | 高（需创建/销毁节点） | 低（仅切换CSS） |
| 首次开销 | `false`时为零 | 始终渲染 |
| 数据绑定 | 切换时重新执行 | 保持不变 |
| 适用场景 | 不频繁切换 | 频繁切换 |

### 使用场景选择

```xml
<!-- 场景1：条件很少改变 → 用 wx:if -->
<view wx:if="{{userRole === 'admin'}}">
  管理员面板
</view>

<!-- 场景2：频繁切换 → 用 hidden -->
<view class="tab-content" hidden="{{currentTab !== 0}}">
  Tab1内容
</view>
<view class="tab-content" hidden="{{currentTab !== 1}}">
  Tab2内容
</view>

<!-- 场景3：初始可能不需要渲染 → 用 wx:if -->
<view wx:if="{{hasError}}">
  错误提示
</view>
```

### wx:if 的惰性渲染

```xml
<!-- wx:if为false时，内部的内容完全不渲染 -->
<view wx:if="{{false}}">
  <!-- 这里的数据绑定 {{heavyData}} 不会执行 -->
  <text>{{heavyData}}</text>
</view>
```

### block wx:if 控制多个元素

```xml
<!-- 用block包裹多个元素一起控制 -->
<block wx:if="{{isLoggedIn}}">
  <view>用户名：{{user.name}}</view>
  <view>头像：<image src="{{user.avatar}}"></image></view>
  <button bindtap="onLogout">退出登录</button>
</block>

<block wx:else>
  <button bindtap="onLogin">请登录</button>
</block>
```

> **⚠️ 注意**：
> - 不要同时用`wx:if`和`hidden`控制同一个元素，会引起混乱。
> - `hidden`的值为`true`时隐藏，`false`时显示，与`wx:if`逻辑相反，容易混淆。
> - 频繁切换的Tab面板务必用`hidden`，避免重复创建/销毁组件。

## 6.12 input组件详解

### input组件核心属性

```xml
<input
  value="{{inputValue}}"           <!-- 初始内容 -->
  type="text"                       <!-- 键盘类型 -->
  password="{{false}}"             <!-- 是否密码框 -->

  placeholder-class="placeholder"  <!-- 占位符样式类 -->
```

### type键盘类型

| type值 | 键盘类型 | 适用场景 |
|--------|---------|---------|
| `text` | 文本键盘 | 普通文字输入 |

| `idcard` | 身份证键盘 | 身份证号输入 |

| `go` | 前往 |

### bindinput事件

`bindinput`是input最核心的事件，每次输入内容变化时触发：

```javascript
Page({
  data: {
    inputValue: ''
  },

  onInput(event) {
    // event.detail.value 是当前输入框的值
    const value = event.detail.value;

    // 实时更新数据
    this.setData({
      inputValue: value
    });

    console.log('当前输入：', value);
    console.log('输入长度：', value.length);
  }
});
```

> **⚠️ 注意**：`bindinput`的`event.detail.value`返回输入框**完整内容**，不是增量字符。每次输入都会触发，注意避免在处理函数中做耗时操作。

### 屏蔽评论关键字

```javascript
Page({
  data: {
    inputValue: ''
  },

  // 敏感词列表
  blockedWords: ['广告', '微信号', '加我', '垃圾', '色情'],

  onInput(event) {
    let value = event.detail.value;

    // 屏蔽敏感词
    for (const word of this.blockedWords) {
      if (value.includes(word)) {
        // 替换为星号
        value = value.replace(new RegExp(word, 'g'), '*'.repeat(word.length));
        wx.showToast({
          title: '包含敏感词',
          icon: 'none',
          duration: 1000
        });
      }
    }

    this.setData({ inputValue: value });

    // 返回处理后的值，覆盖输入框内容
    return value;
  }
});
```

> **💡 原理**：`bindinput`事件处理函数如果`return`一个字符串，该字符串会覆盖输入框的当前内容。利用这个特性可以实现输入过滤、自动格式化等功能。

## 6.13 本章小结

### 知识点回顾

| 知识点 | 核心内容 |
|-------|---------|
| 收藏功能 | 本地缓存存储收藏状态，`wx:if`/`wx:else`切换UI |
| 交互反馈 | `wx.showToast`轻提示，`wx.showModal`模态框，`wx.showActionSheet`底部菜单 |
| 本地缓存 | `wx.setStorageSync`/`wx.getStorageSync`持久化存储，上限10MB |
| 图片选择 | `wx.chooseMedia`支持相册和拍照 |
| 图片预览 | `wx.previewImage`传入`current`和`urls` |
| 条件渲染 | `wx:if`创建/销毁节点，`hidden`仅切换display |
| input组件 | `bindinput`实时获取输入，`return`值可覆盖输入内容 |
| 评论提交 | 内容校验→敏感词过滤→防重复提交→网络请求 |

### wx:if vs hidden 选择原则

```
频繁切换 → hidden（性能好）
不常切换 → wx:if（节省初始渲染）
```

### 常见坑点

1. **hidden的逻辑反转**：`hidden="{{true}}"`是隐藏，`hidden="{{false}}"`是显示，与`wx:if`相反
2. **input的return值**：`bindinput`中`return`的字符串会覆盖输入框内容
3. **previewImage的current**：必须是`urls`数组中已存在的URL
4. **缓存类型转换**：`getStorageSync`返回的数据类型可能与存入时不一致（如数字变字符串）
5. **防重复提交**：网络请求期间用`isSubmitting`标志锁定，避免用户多次点击
