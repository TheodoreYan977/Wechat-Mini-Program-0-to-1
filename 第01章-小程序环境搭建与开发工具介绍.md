# 第01章 小程序环境搭建与开发工具介绍

## 1.1 认识微信小程序

### 1.1.1 什么是微信小程序

微信小程序（Mini Program）是一种不需要下载安装即可使用的应用，它实现了"触手可及"的梦想——用户扫一扫或搜一下即可打开应用。小程序运行在微信客户端内部，基于微信提供的运行环境执行，使用类HTML/CSS/JavaScript的技术栈进行开发。

从技术架构上看，小程序并非运行在浏览器中，而是运行在微信自研的双线程运行环境中：

- **视图层（View Layer）**：由WebView渲染（多个页面对应多个WebView实例）
- **逻辑层（Logic Layer）：** 由JSCore/V8引擎运行JavaScript代码

两层通过微信客户端的Native桥接层进行通信，这种设计既保证了安全沙箱隔离，又提供了接近原生的性能体验。

```javascript
// 小程序的双线程通信示意（逻辑层发送数据到视图层）
this.setData({
  message: 'Hello Mini Program'
});
// setData调用后，数据会经过Native桥接层序列化传递到WebView渲染层
```

### 1.1.2 与原生App优劣对比

| 对比维度 | 微信小程序 | 原生App |
|---------|-----------|---------|
| 开发成本 | 低，一套代码跨平台 | 高，iOS/Android分别开发 |
| 用户体验 | 接近原生，但仍有差距 | 最佳性能体验 |
| 安装方式 | 无需安装，扫码即用 | 需下载安装 |
| 分发渠道 | 微信生态内分享传播 | 应用商店审核上架 |
| 推广成本 | 低，社交裂变容易 | 高，获客成本高 |
| 功能权限 | 受微信限制，能力有限 | 可调用全部系统能力 |
| 包体限制 | 主包2MB，总包20MB | 无硬性限制 |
| 版本更新 | 即时生效，无需用户手动升级 | 需用户手动更新 |

**小程序的优势**在于：开发快、获客成本低、社交传播能力强、免安装。适合轻量级、高频次、低复杂度的应用场景，如电商、工具、资讯、服务预约等。

**小程序的劣势**在于：性能不及原生、能力受微信限制（如后台运行受限）、无法脱离微信独立存在。

### 1.1.3 Web前端开发者与小程序

对于有Web前端经验的开发者来说，学习小程序的门槛非常低：

- **WXML** ≈ HTML（语法略有不同，使用标签不同）
- **WXSS** ≈ CSS（新增了rpx响应式单位）
- **JavaScript** 语法基本一致（支持ES6+，但不支持DOM/BOM操作）
- **JSON** 用于配置

核心区别在于小程序**没有DOM和BOM**，不能使用`document`、`window`等对象，所有UI更新必须通过`setData`机制完成。

```javascript
// ❌ 浏览器中的写法，小程序中会报错
document.getElementById('title').innerText = 'Hello';

// ✅ 小程序中的正确写法
this.setData({
  title: 'Hello'
});
```

### 1.1.4 小程序是一个生态

微信小程序不仅仅是一项技术，更是一个完整的生态体系：

- **开发生态**：官方开发者工具、云开发、第三方框架（uni-app、Taro、mpvue等）
- **能力生态**：支付、登录、分享、订阅消息、直播、客服等开放能力
- **流量生态**：微信搜索、附近的小程序、社交分享、公众号关联
- **商业生态**：小程序广告、电商交易、服务变现

理解小程序是"生态"这一点很重要，它意味着你的应用天然拥有10亿+微信用户的潜在触达能力，以及成熟的商业化基础设施。

## 1.2 注册小程序账号

开发小程序的第一步是注册一个微信小程序账号。

### 注册步骤

1. **访问注册页面**：打开浏览器，访问 `https://mp.weixin.qq.com/`，点击右上角"立即注册"

2. **选择账号类型**：在注册页面选择"小程序"

3. **填写邮箱信息**：使用一个未注册过微信公众平台、未绑定过个人微信的邮箱作为注册邮箱

4. **邮箱激活**：登录邮箱查收激活邮件，点击激活链接

5. **信息登记**：
   - **个人主体**：需填写身份证信息并扫码验证
   - **企业主体**：需填写营业执照信息和对公账户（可使用微信认证）

6. **获取AppID**：注册成功后，在 `开发管理 → 开发设置` 中可以看到你的**AppID（小程序ID）**

> **⚠️ 注意**：AppID是小程序的唯一标识，在创建项目时必须填写。个人开发者无法使用微信支付等部分高级能力，如需支付功能需注册企业主体。

### AppID与AppSecret

```
AppID(小程序ID)：wx1234567890abcdef
AppSecret(小程序密钥)：xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

- **AppID**：公开标识，用于项目配置
- **AppSecret**：私密密钥，用于服务端调用微信API时签名验证，**切勿泄露或前端硬编码**

## 1.3 微信开发者工具的下载及安装

### 下载

1. 访问官方下载页面：`https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html`
2. 根据操作系统选择对应版本：
   - Windows 64位（稳定版 Stable Build）
   - macOS（Intel芯片 / Apple Silicon）
   - Linux

> **建议**：生产开发推荐使用**稳定版（Stable Build）**，如果想体验新特性可使用**预发布版（RC版）**或**开发版（Nightly）**。

### 安装

- **Windows**：双击下载的`.exe`安装包，按向导完成安装，默认安装路径为 `C:\Program Files (x86)\Tencent\微信web开发者工具`
- **macOS**：将下载的`.dmg`文件中的应用拖入"应用程序"文件夹

安装完成后，使用微信扫码登录开发者工具。

## 1.4 新建第一个项目

### 创建项目步骤

1. 打开微信开发者工具，使用微信扫码登录
2. 点击左侧"+"号或"小程序"标签页的"+"按钮
3. 在新建项目页面填写以下信息：

| 配置项 | 说明 |
|-------|------|
| 项目名称 | 自定义，如 `my-first-miniapp` |
| 目录 | 选择项目存放的空文件夹 |
| AppID | 填入注册的小程序AppID，或选择"测试号" |
| 开发模式 | 选择"小程序" |
| 后端服务 | 可选择"不使用云服务"或"微信云开发" |
| 模板 | 选择"JavaScript-基础模板" |

4. 点击"确定"创建项目

### 项目目录结构

创建完成后，项目目录结构如下：

```
my-first-miniapp/
├── pages/
│   ├── index/          # 首页
│   │   ├── index.js    # 页面逻辑
│   │   ├── index.json  # 页面配置
│   │   ├── index.wxml  # 页面结构
│   │   └── index.wxss  # 页面样式
│   └── logs/           # 日志页
│       ├── logs.js
│       ├── logs.json
│       ├── logs.wxml
│       └── logs.wxss
├── utils/              # 工具函数
│   └── util.js
├── app.js              # 应用逻辑
├── app.json            # 全局配置
├── app.wxss            # 全局样式
├── project.config.json # 项目配置
└── sitemap.json        # 搜索索引配置
```

点击工具栏的"编译"按钮，模拟器中即可看到小程序运行效果。

## 1.5 微信开发者工具界面功能介绍

微信开发者工具是开发小程序的核心IDE，集编辑、调试、预览、发布于一体。

### 1.5.1 模拟器

模拟器位于界面左侧，用于在PC上模拟小程序运行效果。

- **机型切换**：顶部下拉可选iPhone、Android等不同机型
- **缩放**：支持50%/75%/100%/125%等缩放比例
- **网络模拟**：可模拟WiFi/4G/3G/2G网络环境
- **实时预览**：代码修改后自动编译刷新

> **⚠️ 注意**：模拟器的渲染结果与真机可能存在细微差异，最终效果务必以真机为准。

### 1.5.2 资源管理器

界面左上方的"资源管理器"面板展示项目的文件树结构，支持：

- 新建/删除/重命名文件和文件夹
- 文件搜索（快捷键 `Ctrl+P`）
- 在文件树中直接编辑文件名

### 1.5.3 编辑器与调试面板

**编辑器**是代码编辑区域，支持：

- 语法高亮（WXML/WXSS/JS/JSON）
- 代码补全（小程序API智能提示）
- 格式化代码（快捷键 `Shift+Alt+F`）
- 多标签页编辑

**调试面板**位于下方，类似Chrome DevTools，包含多个面板：

| 面板 | 功能 |
|------|------|
| Console | JavaScript控制台日志输出 |
| Sources | 源码查看与断点调试 |
| Network | 网络请求监控 |
| Storage | 本地缓存数据查看 |
| AppData | 页面数据实时查看与编辑 |
| Wxml | WXML结构与样式检查（类似Elements） |
| Sensor | 模拟地理位置、陀螺仪等传感器 |
| Audit | 代码质量与体验评分 |

> **💡 实战技巧**：调试数据绑定时，打开`AppData`面板可以实时查看和修改页面data，非常方便定位数据问题。

### 1.5.4 工具栏

工具栏位于顶部，常用功能包括：

| 按钮 | 功能说明 |
|------|---------|
| 编译 | 编译代码并在模拟器刷新（快捷键 `Ctrl+B`） |
| 预览 | 生成二维码，用手机微信扫码真机预览 |
| 真机调试 | 扫码后在真机上开启远程调试 |
| 清缓存 | 清除各类缓存（编译/数据/文件/网络等） |
| 上传 | 上传代码到微信服务器，用于提交审核发布 |
| 版本管理 | Git版本控制集成 |

### 1.5.5 菜单栏

菜单栏提供更多功能入口：

- **设置 → 项目设置**：可配置ES6转ES5、上传代码时自动压缩等选项
- **设置 → 编辑设置**：配置编辑器字体大小、自动换行等
- **工具 → 构建npm**：构建npm依赖（使用第三方npm包时必须执行）
- **工具 → 云开发**：进入云开发控制台（需开通）
- **工具 → 自动化测试**：小程序自动化测试脚手架

### 1.5.6 调试小程序

调试小程序的常用方法：

```javascript
// 1. console打印日志
console.log('调试信息', this.data);
console.error('错误信息');

// 2. 使用AppData面板查看数据
// 打开调试面板 → AppData，可实时查看各页面的data对象

// 3. 断点调试
// 在Sources面板中打开JS文件，点击行号设置断点，触发操作即可暂停

// 4. 使用wx.getLogManager记录日志（会被收集到线上日志）
const logger = wx.getLogManager();
logger.info('用户点击了按钮', { userId: 123 });
```

> **⚠️ 注意**：`console.log`在生产环境中不会展示给用户，但大量日志会影响性能，建议发布前清理。

### 1.5.7 快速打开官方开发文档

- 在代码中选中API名称，按`F1`可快速跳转到官方文档
- 菜单栏 `帮助 → 开发者文档` 直接打开官方文档网站
- 在编辑器中输入API前缀如`wx.`会弹出智能提示，点击提示旁的文档图标可跳转

### 1.5.8 常见操作问题

**问题1：编译后模拟器白屏**

可能原因：JS代码语法错误。检查Console面板是否有红色错误信息，修复语法错误后重新编译。

**问题2：真机预览与模拟器不一致**

真机环境与模拟器存在差异，尤其是CSS渲染和API兼容性。建议开发过程中多用真机预览验证。

**问题3：出现"不在以下request合法域名列表中"错误**

开发阶段可在 `设置 → 项目设置` 中勾选"不校验合法域名"来临时解决。上线前需在管理后台配置合法域名。

**问题4：代码上传后版本号不更新**

每次上传代码需手动填写版本号，确保版本号递增（如1.0.0 → 1.0.1）。

**问题5：开发者工具频繁卡顿或崩溃**

- 清除缓存：`工具 → 清缓存 → 全部清除`
- 降低模拟器缩放比例
- 关闭不必要的断点
- 升级到最新版开发者工具

本章从零开始介绍了微信小程序的完整开发环境搭建流程：

- 开发者工具集成了编辑、编译、调试、预览、发布全流程
