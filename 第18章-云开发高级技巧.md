## 18.1 显示评论数据（联表查询lookup、数据组装）

在文章详情页中，评论数据通常分散在多个集合中。例如，评论集合存储了评论内容和用户 `openid`，但用户昵称和头像需要从用户集合中获取。这就需要使用联表查询（`lookup`）来合并数据。

### 数据结构示例

```
comments 集合:
{
  "_id": "comment_001",
  "_openid": "user_openid_a",
  "articleId": "article_001",
  "content": "这篇文章写得很好！",
  "createTime": "2025-07-07T10:00:00.000Z"
}

users 集合:
{
  "_id": "user_001",
  "_openid": "user_openid_a",
  "nickName": "张三",
  "avatarUrl": "cloud://xxx/avatars/zhangsan.jpg"
}
```

### 使用 aggregate.lookup 联表查询

联表查询需要使用聚合操作中的 `lookup` 阶段，只能在**云函数端**使用：

```javascript
// cloudfunctions/getComments/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()

exports.main = async (event, context) => {
  const { articleId } = event
  
  try {
    const result = await db.collection('comments')
      .aggregate()
      .match({
        articleId: articleId
      })
      .lookup({
        from: 'users',              // 关联的集合
        localField: '_openid',       // 本地字段（comments 集合）
        foreignField: '_openid',     // 外部字段（users 集合）
        as: 'userInfo'               // 输出字段名
      })
      .sort({
        createTime: -1
      })
      .limit(20)
      .end()
    
    // 处理联表结果 — userInfo 是一个数组
    const comments = result.list.map(item => {
      const user = item.userInfo[0] || {}
      return {
        _id: item._id,
        content: item.content,
        createTime: item.createTime,
        nickName: user.nickName || '匿名用户',
        avatarUrl: user.avatarUrl || ''
      }
    })
    
    return {
      code: 0,
      data: comments
    }
  } catch (err) {
    console.error('联表查询失败:', err)
    return { code: -1, message: err.message }
  }
}
```

### 替代方案：小程序端手动组装数据

如果不想用云函数，也可以在小程序端分两次查询后手动组装：

```javascript
async loadCommentsWithUserInfo(articleId) {
  const db = wx.cloud.database()
  
  // 1. 查询评论列表
  const commentRes = await db.collection('comments')
    .where({ articleId })
    .orderBy('createTime', 'desc')
    .get()
  
  // 2. 收集所有不重复的 openid
  const openids = [...new Set(commentRes.data.map(c => c._openid))]
  
  // 3. 查询对应的用户信息（需要权限允许）
  const userRes = await db.collection('users')
    .where({ _openid: db.command.in(openids) })
    .get()
  
  // 4. 构建 openid -> userInfo 的映射
  const userMap = {}
  userRes.data.forEach(u => {
    userMap[u._openid] = u
  })
  
  // 5. 组装最终数据
  const comments = commentRes.data.map(comment => ({
    ...comment,
    nickName: userMap[comment._openid]?.nickName || '匿名用户',
    avatarUrl: userMap[comment._openid]?.avatarUrl || ''
  }))
  
  this.setData({ comments })
}
```

> **注意事项**：
> 1. `lookup` 只能在云函数端的聚合操作中使用，小程序端不支持。
> 2. `lookup` 结果中关联的字段是一个数组（`as` 指定的字段），即使只匹配到一条也是数组。
> 3. 手动组装方案中，`users` 集合的权限需要设置为"所有人可读"，否则无法查询其他用户的信息。

## 18.2 文章收藏计数功能

在文章列表或详情页中，显示收藏数（有多少人收藏了这篇文章）是常见需求。

### 基础实现：使用 count()

最简单的收藏计数方式是查询 `favorites` 集合中某篇文章的收藏记录数量：

```javascript
// 小程序端 — 统计单篇文章的收藏数
async getFavoriteCount(articleId) {
  const db = wx.cloud.database()
  const res = await db.collection('favorites')
    .where({ articleId })
    .count()
  
  return res.total
}
```

### 批量获取多篇文章的收藏数

在文章列表页，需要同时获取每篇文章的收藏数：

```javascript
// cloudfunctions/getArticleList/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()
const _ = db.command
const $ = db.command.aggregate

exports.main = async (event, context) => {
  const { page = 0, pageSize = 10 } = event
  
  try {
    // 使用聚合查询，联表统计收藏数
    const result = await db.collection('articles')
      .aggregate()
      .sort({ createTime: -1 })
      .skip(page * pageSize)
      .limit(pageSize)
      .lookup({
        from: 'favorites',
        localField: '_id',
        foreignField: 'articleId',
        as: 'favoritesList'
      })
      .addFields({
        favoriteCount: $.size('$favoritesList')
      })
      .project({
        favoritesList: 0  // 不返回完整的收藏列表，只保留计数
      })
      .end()
    
    return {
      code: 0,
      data: result.list
    }
  } catch (err) {
    return { code: -1, message: err.message }
  }
}
```

### 在文章模型中维护收藏数字段

更高效的方式是在 `articles` 集合中维护一个 `favoriteCount` 字段，收藏/取消收藏时同步更新：

```javascript
// 收藏文章时 — 计数 +1
const _ = db.command

async function addFavorite(articleId) {
  const db = wx.cloud.database()
  
  // 1. 新增收藏记录
  await db.collection('favorites').add({
    data: {
      articleId,
      createTime: db.serverDate()
    }
  })
  
  // 2. 文章收藏数 +1
  await db.collection('articles').doc(articleId).update({
    data: {
      favoriteCount: _.inc(1)
    }
  })
}

// 取消收藏时 — 计数 -1
async function removeFavorite(articleId, favoriteId) {
  const db = wx.cloud.database()
  
  // 1. 删除收藏记录
  await db.collection('favorites').doc(favoriteId).remove()
  
  // 2. 文章收藏数 -1
  await db.collection('articles').doc(articleId).update({
    data: {
      favoriteCount: _.inc(-1)
    }
  })
}
```

> **常见坑**：
> 1. 小程序端的 `update` 操作受权限限制，只能更新自己创建的文章。如果要更新他人的文章收藏数，必须在云函数中执行。
> 2. 上述"两步操作"（先增删收藏记录，再更新计数）不是原子操作，可能出现数据不一致。解决方案见 18.9 节使用云函数实现原子计数。
> 3. `count()` 在数据量大时较慢，建议缓存计数结果。

## 18.3 编写第一个云函数（云函数的结构、node_modules依赖、部署）

### 云函数目录结构

```
cloudfunctions/
  └── myFirstFunction/
      ├── index.js          # 入口文件（必须）
      ├── package.json      # 依赖配置（必须）
      ├── config.json       # 可选配置
      └── node_modules/     # 依赖包（部署时自动安装）
```

### index.js — 云函数入口

```javascript
// cloudfunctions/myFirstFunction/index.js
const cloud = require('wx-server-sdk')

// 初始化云开发
cloud.init({
  env: cloud.DYNAMIC_CURRENT_ENV  // 自动使用当前云环境
})

// 云函数入口
exports.main = async (event, context) => {
  // event: 小程序端调用时传入的参数对象
  // context: 包含调用上下文信息
  
  const { action, data } = event
  
  console.log('收到参数:', event)
  
  switch (action) {
    case 'create':
      return await createRecord(data)
    case 'query':
      return await queryRecords(data)
    case 'update':
      return await updateRecord(data)
    default:
      return { code: -1, message: '未知操作类型' }
  }
}

// 新增记录
async function createRecord(data) {
  const db = cloud.database()
  try {
    const res = await db.collection('todos').add({
      data: {
        title: data.title,
        completed: false,
        createTime: db.serverDate()
      }
    })
    return { code: 0, data: { _id: res._id } }
  } catch (err) {
    return { code: -1, message: err.message }
  }
}

// 查询记录
async function queryRecords(data) {
  const db = cloud.database()
  try {
    const res = await db.collection('todos')
      .where({ completed: false })
      .limit(20)
      .get()
    return { code: 0, data: res.data }
  } catch (err) {
    return { code: -1, message: err.message }
  }
}

// 更新记录
async function updateRecord(data) {
  const db = cloud.database()
  try {
    const res = await db.collection('todos').doc(data.id).update({
      data: { completed: data.completed }
    })
    return { code: 0, data: { updated: res.stats.updated } }
  } catch (err) {
    return { code: -1, message: err.message }
  }
}
```

### package.json — 依赖配置

```json
{
  "name": "myFirstFunction",
  "version": "1.0.0",
  "description": "我的第一个云函数",
  "main": "index.js",
  "dependencies": {
    "wx-server-sdk": "~2.6.3"
  },
  "author": "",
  "license": "ISC"
}
```

### 安装依赖

在云函数目录下执行：

```bash
cd cloudfunctions/myFirstFunction
npm install
```

### 部署云函数

**方式一：通过开发者工具部署**

1. 右键点击云函数文件夹 `myFirstFunction`
2. 选择"上传并部署：云端安装依赖"（推荐，不需要本地安装 `node_modules`）
3. 或选择"上传并部署：所有文件"（需先本地 `npm install`）

**方式二：命令行部署**

```bash
# 安装 cloudbase-cli
npm install -g @cloudbase/cli

# 登录
tcb login

# 部署云函数
tcb fn deploy myFirstFunction
```

### 云函数执行流程

```
小程序端 wx.cloud.callFunction
    ↓
云端接收请求，创建云函数实例
    ↓
执行 exports.main(event, context)
    ↓
返回结果给小程序端
```

> **注意事项**：
> 1. 每个云函数都有独立的 `node_modules`，需要在每个云函数目录下单独安装依赖。
> 2. 云函数是无状态的，每次调用可能是不同的实例，不要在函数外部维护全局状态。
> 3. 云函数默认超时时间为 3 秒，可在云开发控制台或 `config.json` 中修改（最长 60 秒）：
>    ```json
>    {
>      "timeout": 10,
>      "memorySize": 256
>    }
>    ```

## 18.4 为什么需要在本地调试云函数

云函数部署到云端后，调试面临以下问题：

### 云端调试的痛点

1. **无法断点调试**：云端运行无法设置断点，只能通过 `console.log` 打印日志，效率低下。
2. **日志查看不便**：需要打开云开发控制台 → 云函数 → 日志，操作繁琐且有延迟。
3. **修改后需重新部署**：每次修改代码都要重新上传部署，耗时 10-30 秒。
4. **无法调试 Node.js 原生模块**：部分报错信息在云端日志中不够详细。
5. **难以模拟数据**：在云端难以模拟各种边界条件的 `event` 参数。

### 本地调试的优势

| 对比项 | 云端运行 | 本地调试 |
|--------|----------|----------|
| 断点调试 | 不支持 | 支持 |
| 代码修改生效 | 需重新部署 | 即时生效 |
| 日志查看 | 控制台查看 | 控制台实时输出 |
| 模拟参数 | 需从小程序传 | 可直接在调试界面配置 |
| 调试速度 | 慢（部署+网络） | 快（本地执行） |

> **总结**：本地调试能大幅提升云函数开发效率，特别是在处理复杂逻辑和定位 Bug 时，断点调试能力不可或缺。

## 18.5 如何本地调试云函数（开发者工具本地调试功能、断点调试）

### 开启本地调试

1. **安装本地依赖**：在云函数目录下执行 `npm install`
2. **配置本地调试**：在开发者工具中，右键云函数文件夹 → "本地调试"
3. 或在云开发控制台 → 云函数 → 选择函数 → "本地调试"标签页

### 使用开发者工具本地调试

**步骤一：开启本地调试模式**

```
右键云函数 → 本地调试
```

开发者工具会打开本地调试面板。

**步骤二：配置测试参数**

在调试面板的"测试参数"区域，输入模拟的 `event` 参数：

```json
{
  "action": "create",
  "data": {
    "title": "测试待办事项"
  }
}
```

**步骤三：运行测试**

点击"运行测试"按钮，云函数将在本地执行，输出结果会显示在面板下方。

### 断点调试

1. 在云函数 `index.js` 中设置断点（点击行号左侧）
2. 在本地调试面板中勾选"启用断点调试"
3. 点击"运行测试"
4. 代码执行到断点处会暂停，可以查看变量、调用栈等

```javascript
exports.main = async (event, context) => {
  const { action, data } = event
  
  // ← 在这行设置断点
  console.log('收到参数:', event)
  
  if (action === 'create') {
    const result = await createRecord(data)
    // ← 在这里设置断点，检查 result
    return result
  }
}
```

### 模拟微信上下文

本地调试时，`cloud.getWXContext()` 会返回模拟数据。可以在调试面板中配置模拟的 `openid`：

```json
{
  "OPENID": "test_openid_001",
  "APPID": "wxXXXXXXXXXXXX",
  "UNIONID": "test_unionid_001"
}
```

### 本地调试限制

> **注意事项**：
> 1. 本地调试时，云函数操作数据库使用的是**本地环境**的云开发连接，确保 `cloud.init` 中的 `env` 与目标环境一致。
> 2. 某些微信特有的 API（如 `cloud.getOpenData`）在本地调试时可能无法正常工作。
> 3. 本地调试的文件上传/下载操作会实际操作云存储，注意不要误传生产环境的文件。
> 4. 本地调试完成后，记得重新部署到云端才能在小程序中生效。

## 18.6 上传云函数到云端（上传并部署、云端安装依赖）

### 部署方式对比

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| 上传并部署：云端安装依赖 | 只上传代码，依赖在云端安装 | 推荐，部署快，不依赖本地环境 |
| 上传并部署：所有文件 | 上传代码和 node_modules | 网络不好或依赖安装有问题时 |
| 上传：云端安装依赖（不上传 node_modules） | 只上传代码 | 同第一种 |

### 通过开发者工具部署

1. 右键点击云函数文件夹
2. 选择 **"上传并部署：云端安装依赖"**
3. 等待部署完成（通常 10-30 秒）
4. 部署成功后会弹出提示

### 批量部署

如果修改了多个云函数，可以右键 `cloudfunctions` 目录 → "上传并部署所有云函数"。

### 使用 CLI 工具部署

```javascript
wx.cloud.callFunction({

> **常见坑**：
> 1. 部署时如果 `package.json` 中依赖版本不兼容，云端安装可能失败。建议锁定版本号（如 `"wx-server-sdk": "2.6.3"` 而非 `"^2.6.3"`）。
> 2. 云函数名称即为文件夹名称，修改函数名需要同时修改文件夹名并重新部署。
> 3. 部署后云函数可能有冷启动延迟（首次调用较慢），后续调用会快很多。

## 18.7 云函数查询特性（云函数端使用admin权限查询、批量操作）

云函数端操作数据库拥有**管理端权限**，这是云函数最大的优势之一。

### 权限对比

| 操作 | 小程序端 | 云函数端 |
|------|----------|----------|
| 读取自己的数据 | 支持 | 支持 |
| 读取他人的数据 | 受权限规则限制 | 支持管理端权限 |
| 修改他人的数据 | 不支持 | 支持 |
| 删除他人的数据 | 不支持 | 支持 |
| 批量更新 | 不支持 | 支持 |
| 批量删除 | 不支持 | 支持 |
| 单次查询上限 | 20 条 | 100 条 |

### 云函数端批量查询

```javascript
// cloudfunctions/batchQuery/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()

exports.main = async (event, context) => {
  // 云函数端单次可获取 100 条
  const MAX_LIMIT = 100
  
  // 先获取总数
  const countResult = await db.collection('articles').count()
  const total = countResult.total
  
  // 计算需要分几次查询
  const batchTimes = Math.ceil(total / MAX_LIMIT)
  
  // 并发查询所有数据
  const tasks = []
  for (let i = 0; i < batchTimes; i++) {
    const promise = db.collection('articles')
      .skip(i * MAX_LIMIT)
      .limit(MAX_LIMIT)
      .get()
    tasks.push(promise)
  }
  
  // 等待所有查询完成
  const results = await Promise.all(tasks)
  
  // 合并结果
  const allData = results.reduce((acc, cur) => acc.concat(cur.data), [])
  
  return {
    code: 0,
    total: total,
    data: allData
  }
}
```

### 云函数端批量更新

```javascript
// cloudfunctions/batchUpdate/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()

exports.main = async (event, context) => {
  const _ = db.command
  
  try {
    // 批量更新：将所有 status 为 'draft' 的文章改为 'published'
    const result = await db.collection('articles')
      .where({
        status: 'draft'
      })
      .update({
        data: {
          status: 'published',
          updateTime: db.serverDate()
        }
      })
    
    return {
      code: 0,
      message: `成功更新 ${result.stats.updated} 条记录`
    }
  } catch (err) {
    return { code: -1, message: err.message }
  }
}
```

### 云函数端批量删除

// cloudfunctions/batchDelete/index.js
exports.main = async (event, context) => {

        expireTime: _.lt(db.serverDate())

```javascript
// utils/cloudHelper.js

/**
 * 统一封装云函数调用
 * @param {string} name - 云函数名称
 * @param {object} data - 传入参数
 * @returns {Promise<object>} - 返回 result
 */
async function callFunction(name, data = {}) {
  try {
    const res = await wx.cloud.callFunction({
      name,
      data
    })
    
    // 统一处理返回格式
    if (res.result && res.result.code === 0) {
      return res.result.data
    } else {
      throw new Error(res.result?.message || '云函数返回异常')
    }
  } catch (err) {
    console.error(`调用云函数 ${name} 失败:`, err)
    wx.showToast({
      title: err.message || '操作失败',
      icon: 'none'
    })
    throw err
  }
}

module.exports = { callFunction }
```

## 18.9 改写文章收藏数量（用云函数实现原子计数）

在 18.2 节中，我们在小程序端分两步操作（新增收藏记录 + 更新计数），这种方式不是原子操作，可能因网络问题导致数据不一致。使用云函数可以将两步操作合并为一个事务。

### 问题分析

```
非原子操作的风险：
步骤1: add favorites 记录 → 成功
步骤2: update favoriteCount → 失败（网络断开）
结果: 有收藏记录，但计数没增加 → 数据不一致
```

### 云函数实现原子计数

```javascript
// cloudfunctions/toggleFavorite/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()
const _ = db.command

exports.main = async (event, context) => {
  const { articleId, action } = event  // action: 'add' 或 'remove'
  const wxContext = cloud.getWXContext()
  const openid = wxContext.OPENID
  
  try {
    if (action === 'add') {
      // === 添加收藏 ===
      
      // 1. 检查是否已收藏（防止重复）
      const existing = await db.collection('favorites').where({
        articleId: articleId,
        _openid: openid
      }).get()
      
      if (existing.data.length > 0) {
        return { code: -1, message: '已收藏过该文章' }
      }
      
      // 2. 新增收藏记录
      await db.collection('favorites').add({
        data: {
          articleId: articleId,
          createTime: db.serverDate()
        }
      })
      
      // 3. 文章收藏数 +1（管理端权限，可更新他人文章）
      await db.collection('articles').doc(articleId).update({
        data: {
          favoriteCount: _.inc(1)
        }
      })
      
      return { code: 0, message: '收藏成功' }
      
    } else if (action === 'remove') {
      // === 取消收藏 ===
      
      // 1. 查找并删除收藏记录
      const existing = await db.collection('favorites').where({
        articleId: articleId,
        _openid: openid
      }).get()
      
      if (existing.data.length === 0) {
        return { code: -1, message: '未收藏该文章' }
      }
      
      await db.collection('favorites').doc(existing.data[0]._id).remove()
      
      // 2. 文章收藏数 -1
      await db.collection('articles').doc(articleId).update({
        data: {
          favoriteCount: _.inc(-1)
        }
      })
      
      return { code: 0, message: '取消收藏成功' }
    }
    
    return { code: -1, message: '未知操作' }
    
  } catch (err) {
    console.error('收藏操作失败:', err)
    return { code: -1, message: err.message }
  }
}
```

### 使用数据库事务（更强一致性）

对于要求严格一致性的场景，可以使用数据库事务：

```javascript
// cloudfunctions/toggleFavoriteTransaction/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()
const _ = db.command

exports.main = async (event, context) => {
  const { articleId, action } = event
  const openid = cloud.getWXContext().OPENID
  
  try {
    // 开启事务
    const transaction = await db.runTransaction(async transaction => {
      // 在事务中查询收藏记录
      const favoriteRes = await transaction.collection('favorites')
        .where({ articleId, _openid: openid })
        .get()
      
      if (action === 'add') {
        if (favoriteRes.data.length > 0) {
          throw new Error('已收藏过')
        }
        
        // 事务中新增
        await transaction.collection('favorites').add({
          data: { articleId, createTime: db.serverDate() }
        })
        
        // 事务中更新计数
        await transaction.collection('articles').doc(articleId).update({
          data: { favoriteCount: _.inc(1) }
        })
        
      } else {
        if (favoriteRes.data.length === 0) {
          throw new Error('未收藏')
        }
        
        await transaction.collection('favorites')
          .doc(favoriteRes.data[0]._id)
          .remove()
        
        await transaction.collection('articles').doc(articleId).update({
          data: { favoriteCount: _.inc(-1) }
        })
      }
      
      return { success: true }
    })
    
    return { code: 0, message: '操作成功', data: transaction }
    
  } catch (err) {
    // 事务中任一步骤失败，所有操作自动回滚
    return { code: -1, message: err.message }
  }
}
```

### 小程序端调用

```javascript
// pages/article-detail/article-detail.js
Page({
  data: {
    articleId: '',
    isFavorited: false,
    favoriteCount: 0
  },

  async toggleFavorite() {
    const action = this.data.isFavorited ? 'remove' : 'add'
    
    try {
      wx.showLoading({ title: '处理中...' })
      
      const res = await wx.cloud.callFunction({
        name: 'toggleFavorite',
        data: {
          articleId: this.data.articleId,
          action: action
        }
      })
      
      wx.hideLoading()
      
      if (res.result.code === 0) {
        // 更新本地状态
        this.setData({
          isFavorited: action === 'add',
          favoriteCount: this.data.favoriteCount + (action === 'add' ? 1 : -1)
        })
        wx.showToast({
          title: action === 'add' ? '收藏成功' : '已取消收藏'
        })
      } else {
        wx.showToast({ title: res.result.message, icon: 'none' })
      }
    } catch (err) {
      wx.hideLoading()
      wx.showToast({ title: '操作失败', icon: 'none' })
    }
  }
})
```

> **注意事项**：
> 1. 事务操作要求所有集合在同一个云环境中。
> 2. 事务中不支持聚合操作。
> 3. 事务有超时限制（默认 5 秒），复杂操作可能超时。
> 4. `_.inc()` 是原子自增命令，即使不用事务，自增操作本身也是原子的。

## 18.10 修正文章评论数量（云函数聚合统计）

评论数量可能因为各种原因（如删除评论时计数未同步）出现偏差，需要定期修正。

### 问题场景

```
articles 集合中的 commentCount: 15
但 comments 集合中该文章的实际评论数: 12
→ 计数不准确，需要修正
```

### 云函数实现聚合统计

```javascript
// cloudfunctions/fixCommentCount/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()
const _ = db.command
const $ = db.command.aggregate

exports.main = async (event, context) => {
  const { articleId, fixAll = false } = event
  
  try {
    if (articleId) {
      // 修正单篇文章的评论数
      await fixSingleArticle(articleId)
      return { code: 0, message: '修正完成' }
    } else if (fixAll) {
      // 修正所有文章的评论数
      const result = await fixAllArticles()
      return { code: 0, message: `共修正 ${result.fixed} 篇文章`, data: result }
    }
    
    return { code: -1, message: '请指定 articleId 或 fixAll' }
  } catch (err) {
    return { code: -1, message: err.message }
  }
}

// 修正单篇文章评论数
async function fixSingleArticle(articleId) {
  // 使用聚合统计评论数
  const countResult = await db.collection('comments')
    .where({ articleId: articleId })
    .count()
  
  const realCount = countResult.total
  
  // 更新文章的评论数字段
  await db.collection('articles').doc(articleId).update({
    data: { commentCount: realCount }
  })
  
  console.log(`文章 ${articleId} 评论数修正为 ${realCount}`)
  return realCount
}

// 修正所有文章评论数（分批处理）
async function fixAllArticles() {
  const MAX_LIMIT = 100
  let fixed = 0
  let offset = 0
  
  while (true) {
    // 分批获取文章
    const articlesRes = await db.collection('articles')
      .skip(offset)
      .limit(MAX_LIMIT)
      .field({ _id: true })  // 只获取 _id 字段，减少数据传输
      .get()
    
    if (articlesRes.data.length === 0) break
    
    // 并发修正每篇文章
    const fixTasks = articlesRes.data.map(article => 
      fixSingleArticle(article._id)
    )
    
    await Promise.all(fixTasks)
    fixed += articlesRes.data.length
    offset += MAX_LIMIT
    
    console.log(`已处理 ${fixed} 篇文章`)
  }
  
  return { fixed }
}
```

### 使用聚合统计带条件的评论数

如果评论有状态（如"已删除"、"待审核"），需要更精确的统计：

```javascript
async function getValidCommentCount(articleId) {
  const result = await db.collection('comments')
    .aggregate()
    .match({
      articleId: articleId,
      status: 'published'  // 只统计已发布的评论
    })
    .group({
      _id: null,
      count: $.sum(1)
    })
    .end()
  
  return result.list.length > 0 ? result.list[0].count : 0
}
```

### 定时触发修正（定时云函数）

可以配置定时触发器，每天凌晨自动修正评论数：

```json

    {

      "type": "timer",

  ]

### 小程序端手动触发修正

```javascript

> 2. 定时云函数的触发有最小间隔限制（免费版最小 1 分钟触发一次）。
> 3. 聚合操作 `aggregate` 在数据量大时比 `count()` 更灵活，但性能略低。

## 18.11 本章小结

本章深入探讨了云开发的高级技巧，核心知识点如下：

| 知识点 | 核心内容 |
|--------|----------|
| 联表查询 | `aggregate.lookup` 实现跨集合数据合并 |
| 收藏计数 | `count()` 统计与 `favoriteCount` 字段维护 |
| 云函数编写 | 结构、依赖管理、部署流程 |
| 本地调试 | 断点调试、参数模拟、微信上下文模拟 |
| 云函数部署 | 云端安装依赖、CLI 部署、config.json 配置 |
| 管理端权限 | 批量查询/更新/删除、突破小程序端权限限制 |
| 云函数调用 | `callFunction` 三种调用风格与统一封装 |
| 原子计数 | 云函数事务保证数据一致性 |
| 聚合统计 | `aggregate.group` 统计与定时修正 |

**核心要点回顾**：

1. **联表查询是核心能力**：`lookup` 可以在云函数端实现类似 SQL JOIN 的功能，解决跨集合数据组装问题。

2. **云函数是权限突破的关键**：小程序端受安全规则限制，云函数拥有管理端权限，能执行跨用户操作和批量操作。

3. **本地调试大幅提升效率**：善用断点调试能力，避免频繁部署和查看云端日志。

4. **事务保证数据一致性**：对于多步操作（如收藏+计数），使用 `runTransaction` 确保原子性。

5. **聚合操作解决复杂统计**：`aggregate` 提供了 `match`、`group`、`lookup` 等强大的数据处理能力。

6. **定时触发实现自动化**：通过 `config.json` 配置定时触发器，可以实现数据修正、报表生成等自动化任务。
