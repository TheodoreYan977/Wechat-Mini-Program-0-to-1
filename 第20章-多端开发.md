# 第20章 多端开发

## 20.1 什么是多端框架（将小程序代码编译为 iOS/Android 原生应用）

### 多端开发的概念

多端开发是指使用一套代码库，通过编译或转译技术，生成可以在多个平台（微信小程序、iOS App、Android App、H5、甚至其他小程序平台）上运行的应用。

### 小程序多端框架的工作原理

```
┌─────────────────────────────────────────────┐
│              统一代码库                       │
│  (WXML + WXSS + JS + 组件/API 适配层)        │
└──────────────────┬──────────────────────────┘
                   │
         ┌─────────┼─────────┐
         │         │         │
         ▼         ▼         ▼
   ┌──────────┐ ┌──────────┐ ┌──────────┐
   │ 微信小程序 │ │ iOS App  │ │Android App│
   │ (原生运行) │ │(原生运行) │ │(原生运行) │
   └──────────┘ └──────────┘ └──────────┘
```

### 微信多端框架

微信官方提供了多端开发能力，可以将小程序代码编译为 iOS 和 Android 原生应用：

- **编译型**：将小程序的 WXML/WXSS/JS 编译为对应平台的原生代码
- **运行时兼容**：提供运行时层，模拟小程序的组件和 API
- **原生渲染**：编译后的 App 使用原生 UI 组件渲染，性能接近原生

### 与传统跨端方案的区别

| 特性 | 微信多端 | React Native | Flutter | Hybrid |
|------|----------|-------------|---------|--------|
| 开发语言 | WXML/WXSS/JS | JS/JSX | Dart | HTML/CSS/JS |
| 渲染方式 | 原生渲染 | 原生渲染 | 自绘引擎 | WebView |
| 小程序兼容 | 原生兼容 | 不兼容 | 不兼容 | 不兼容 |
| 学习成本 | 低（会小程序即可） | 中 | 高 | 低 |
| 性能 | 接近原生 | 接近原生 | 接近原生 | 一般 |

> **注意事项**：微信多端框架的核心优势在于**代码复用**——已有的小程序代码可以低成本地编译为原生 App，而不需要重新用 Swift/Kotlin 开发。

## 20.2 多端开发的高效率与低成本

### 效率对比

| 开发方式 | 人力需求 | 开发周期 | 维护成本 |
|----------|----------|----------|----------|
| 原生开发（iOS+Android+小程序） | 3+ 人 | 3-6 个月 | 高（3 套代码） |
| 多端开发 | 1-2 人 | 1-2 个月 | 低（1 套代码） |

### 多端开发的核心优势

**1. 代码复用率高**

```
小程序代码 → 多端编译 → iOS App + Android App
代码复用率: 80%-95%（仅少量平台特定代码需单独编写）
```

**2. 统一技术栈**

- 只需掌握小程序开发技术
- 不需要分别学习 Swift（iOS）和 Kotlin（Android）
- 团队技术栈统一，沟通成本低

**3. 一致的用户体验**

- 多端 UI 和交互逻辑一致
- 品牌形象统一
- 功能同步发布

**4. 降低维护成本**

- Bug 修复只需改一处
- 新功能同时上线多端
- 不需要维护多套代码库

### 适用场景分析

**适合多端开发的场景：**
- 已有成熟小程序，希望快速扩展到 App
- 创业项目，资源有限但需要覆盖多端
- 内容型/电商型应用，交互不极端复杂
- 企业内部应用，对原生体验要求不极致

**不太适合的场景：**
- 对原生性能有极致要求的应用（如大型游戏）
- 需要深度使用平台特有功能的应用
- 对 App 包体大小有严格限制的场景

> **成本估算**：一个功能完整的小程序转化为多端 App，迁移成本约为原生开发的 20%-30%，但能覆盖 90%+ 的功能需求。

## 20.3 多端应用账号体系指南

### 20.3.1 不需要微信特有功能的多端应用

如果多端 App **不需要使用微信支付、微信登录、微信分享等微信特有功能**，账号体系相对简单：

**自有账号体系：**

```javascript
// 使用通用的用户名/密码登录
Page({
  data: {
    username: '',
    password: ''
  },

  async login() {
    try {
      const res = await wx.request({
        url: 'https://your-api.com/login',
        method: 'POST',
        data: {
          username: this.data.username,
          password: this.data.password
        }
      })
      
      if (res.data.success) {
        // 保存 token
        wx.setStorageSync('token', res.data.token)
        wx.setStorageSync('userInfo', res.data.userInfo)
      }
    } catch (err) {
      console.error('登录失败:', err)
    }
  }
})
```

**第三方登录（非微信）：**

```javascript
// 手机号验证码登录
async smsLogin(phone, code) {
  const res = await wx.request({
    url: 'https://your-api.com/sms-login',
    method: 'POST',
    data: { phone, code }
  })
  return res.data
}
```

### 20.3.2 需要使用微信特有功能的多端应用

如果多端 App 需要使用微信登录、微信支付等功能，需要处理小程序和 App 两个平台的微信账号打通：

**账号体系架构：**

```
┌─────────────────────────────────────┐
│           自有后端服务器              │
│                                     │
│  ┌─────────────────────────────┐    │
│  │      用户表 (users)          │    │
│  │  • id (主键)                 │    │
│  │  • wx_openid_mp (小程序openid)│   │
│  │  • wx_openid_app (App openid) │   │
│  │  • wx_unionid (unionid)      │    │
│  │  • phone                     │    │
│  │  • nickname                  │    │
│  └─────────────────────────────┘    │
│                                     │
│  ┌─────────────────────────────┐    │
│  │    统一登录接口               │    │
│  │  • 小程序: openid + unionid   │   │
│  │  • App: openid + unionid      │   │
│  │  • 通过 unionid 打通账号      │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```

**关键点：使用 UnionID 打通账号**

```
微信开放平台 (open.weixin.qq.com)
    ↓ 绑定小程序和移动应用
同一个微信用户
    ↓
小程序 → openid_mp + unionid
App    → openid_app + unionid
         ↑
    unionid 相同 → 同一个用户
```

> **注意事项**：UnionID 是打通小程序和 App 微信账号的关键。同一个微信用户在不同应用中 openid 不同，但 UnionID 相同（前提是应用绑定在同一个微信开放平台账号下）。

### 20.3.3 微信开发者平台（注册多端应用、获取 AppID）

**注册步骤：**

1. 访问 [微信开放平台](https://open.weixin.qq.com)
2. 注册开发者账号（需企业资质，需付费 300 元/年）
3. 完成开发者认证

**创建移动应用：**

1. 登录微信开放平台 → "管理中心" → "移动应用"
2. 点击"创建移动应用"
3. 填写应用信息：
   - 应用名称
   - 应用简介
   - 应用官网
   - 应用图标
   - Android 应用签名
   - iOS Bundle ID
4. 提交审核（通常 1-7 个工作日）
5. 审核通过后获得 **AppID** 和 **AppSecret**

**绑定小程序：**

```
微信开放平台 → 管理中心 → 移动应用 → 关联小程序
                                        ↓
                          绑定同一主体下的小程序
                                        ↓
                              共享 UnionID 体系
```

**所需资质清单：**

| 项目 | 要求 |
|------|------|
| 开放平台账号 | 企业/个体工商户 |
| 认证费用 | 300 元/年 |
| 移动应用 | 需提供应用信息 + 包名/签名 |
| iOS 应用 | 需提供 Bundle ID |
| Android 应用 | 需提供包名 + 应用签名 |
| 小程序关联 | 需同一主体 |

## 20.4 多端应用新手指南（创建多端项目、配置多端编译）

### 创建多端项目

**方式一：使用微信开发者工具创建**

1. 打开微信开发者工具
2. 点击"新建项目"
3. 选择"小程序多端项目"模板
4. 填写项目信息：
   - 项目名称
   - 目录
   - AppID（小程序的 AppID）
   - 开发模式选择"多端开发"

**方式二：在现有小程序基础上添加多端配置**

```json
// app.json 添加多端配置
{
  "pages": [
    "pages/index/index",
    "pages/login/login"
  ],
  "window": {
    "navigationBarTitleText": "多端应用"
  },
  "mp-multiplatform": {
    "enable": true,
    "targets": ["ios", "android"]
  }
}
```

### 多端编译配置

在项目根目录创建或修改 `project.config.json`：

```json
{
  "miniprogramRoot": "miniprogram/",
  "cloudfunctionRoot": "cloudfunctions/",
  "setting": {
    "es6": true,
    "enhance": true,
    "postcss": true,
    "minified": true
  },
  "compileType": "miniprogram",
  "multiPlatform": {
    "enable": true,
    "ios": {
      "appid": "wx_ios_appid",
      "bundleId": "com.yourcompany.app",
      "version": "1.0.0"
    },
    "android": {
      "appid": "wx_android_appid",
      "packageName": "com.yourcompany.app",
      "version": "1.0.0"
    }
  }
}
```

### 多端条件编译

使用条件编译处理平台差异：

```javascript
// JS 条件编译
// #ifdef MP-WEIXIN
console.log('运行在微信小程序')
// #endif

// #ifdef APP-IOS
console.log('运行在 iOS App')
// #endif

// #ifdef APP-ANDROID
console.log('运行在 Android App')
// #endif

// #ifndef MP-WEIXIN
console.log('运行在非微信小程序平台')
// #endif
```

```xml
<!-- WXML 条件编译 -->
<!-- #ifdef MP-WEIXIN -->
<view>仅在小程序中显示</view>
<!-- #endif -->

<!-- #ifdef APP-IOS -->
<view>仅在 iOS App 中显示</view>
<!-- #endif -->
```

```css
/* WXSS 条件编译 */
/* #ifdef APP-IOS */
.ios-only {
  padding-top: env(safe-area-inset-top);
}
/* #endif */

/* #ifdef APP-ANDROID */
.android-only {
  padding-top: 24px;
}
/* #endif */
```

### API 适配层

编写统一的 API 适配层处理平台差异：

```javascript
// utils/platform.js
function getPlatform() {
  // #ifdef MP-WEIXIN
  return 'miniprogram'
  // #endif
  
  // #ifdef APP-IOS
  return 'ios'
  // #endif
  
  // #ifdef APP-ANDROID
  return 'android'
  // #endif
}

// 获取系统信息
function getSystemInfo() {
  return new Promise((resolve) => {
    // 所有平台都支持 wx.getSystemInfo
    wx.getSystemInfo({
      success: resolve
    })
  })
}

// 网络请求适配
function request(options) {
  // 小程序使用 wx.request
  // App 端也兼容 wx.request（多端框架已适配）
  return new Promise((resolve, reject) => {
    wx.request({
      ...options,
      success: resolve,
      fail: reject
    })
  })
}

// 本地存储适配
const storage = {
  set(key, value) {
    wx.setStorageSync(key, value)
  },
  get(key) {
    return wx.getStorageSync(key)
  },
  remove(key) {
    wx.removeStorageSync(key)
  }
}

module.exports = {
  getPlatform,
  getSystemInfo,
  request,
  storage
}
```

> **注意事项**：
> 1. 多端编译后，`wx.*` API 在 App 端由多端框架的运行时层提供兼容。
> 2. 部分小程序专有 API（如 `wx.openSetting`、`wx.getUserProfile`）在 App 端不可用，需要做适配。
> 3. 首次创建多端项目需要下载多端编译工具链，可能需要几分钟。

## 20.6 多端应用的注意事项（API 兼容性、UI 适配差异、平台限制）

### API 兼容性

部分微信小程序 API 在多端 App 中不可用或行为不同：

```javascript
// utils/api-compatibility.js

// API 兼容性检测表
const API_COMPATIBILITY = {
  'wx.cloud': { miniprogram: true, ios: false, android: false },
  'wx.requestPayment': { miniprogram: true, ios: true, android: true },
  'wx.login': { miniprogram: true, ios: true, android: true },
  'wx.getUserProfile': { miniprogram: true, ios: false, android: false },
  'wx.openSetting': { miniprogram: true, ios: false, android: false },
  'wx.getShareInfo': { miniprogram: true, ios: false, android: false },
}

// 安全调用 API
function safeCallAPI(apiName, params = {}) {
  const platform = getPlatform()
  const compatible = API_COMPATIBILITY[apiName]
  
  if (!compatible || !compatible[platform]) {
    console.warn(`${apiName} 在 ${platform} 平台不可用`)
    return Promise.reject(new Error(`API 不可用: ${apiName}`))
  }
  
  return new Promise((resolve, reject) => {
    const api = apiName.split('.').reduce((obj, key) => obj[key], wx)
    api({
      ...params,
      success: resolve,
      fail: reject
    })
  })
}
```

### UI 适配差异

**1. 状态栏与安全区域：**

```css
/* 适配 iOS 刘海屏和 Android 状态栏 */
.page-container {
  /* 使用 safe-area 适配 */
  padding-top: env(safe-area-inset-top);
  padding-bottom: env(safe-area-inset-bottom);
  padding-left: env(safe-area-inset-left);
  padding-right: env(safe-area-inset-right);
}

/* 自定义导航栏高度适配 */
.custom-nav {
  height: calc(44px + env(safe-area-inset-top));
  padding-top: env(safe-area-inset-top);
}
```

```javascript
// 获取状态栏和导航栏高度
function getNavigationBarInfo() {
  const systemInfo = wx.getSystemInfoSync()
  
  // #ifdef APP-IOS
  const statusBarHeight = systemInfo.statusBarHeight
  const navBarHeight = 44  // iOS 导航栏标准高度
  // #endif
  
  // #ifdef APP-ANDROID
  const statusBarHeight = systemInfo.statusBarHeight
  const navBarHeight = 48  // Android 导航栏标准高度
  // #endif
  
  // #ifdef MP-WEIXIN
  const menuButton = wx.getMenuButtonBoundingClientRect()
  const statusBarHeight = systemInfo.statusBarHeight
  const navBarHeight = (menuButton.top - statusBarHeight) * 2 + menuButton.height
  // #endif
  
  return {
    statusBarHeight,
    navBarHeight,
    totalHeight: statusBarHeight + navBarHeight
  }
}
```

**2. 字体大小适配：**

```css
/* 使用 rpx 实现自适应（小程序和小程序多端 App 通用） */
.title {
  font-size: 32rpx;  /* 自动根据屏幕宽度缩放 */
}

/* 但在 App 端，rpx 的基准可能与小程序不同，需要测试 */
```

**3. 滚动行为差异：**

```xml
<!-- iOS 滚动有弹性效果，Android 没有 -->
<!-- 需要统一处理 -->
<scroll-view 
  scroll-y 
  enhanced
  bounces="{{platform === 'ios'}}"
  show-scrollbar="{{false}}"
>
  <!-- 内容 -->
</scroll-view>
```

### 平台限制

| 限制项 | 小程序 | iOS App | Android App |
|--------|--------|---------|-------------|
| 包体大小 | 2MB（主包） | 无限制（建议<100MB） | 无限制（建议<100MB） |
| 云开发 | 支持 | 不支持（需自建后端） | 不支持（需自建后端） |
| 微信支付 | 原生支持 | 需集成 SDK | 需集成 SDK |
| 分享 | 微信内分享 | 系统分享/微信 SDK | 系统分享/微信 SDK |
| 推送通知 | 订阅消息 | APNs 推送 | FCM/厂商推送 |
| 后台运行 | 受限 | 支持 | 支持 |
| 文件系统 | 受限 | 完整访问 | 完整访问 |

> **常见坑**：
> 1. 云开发（`wx.cloud`）在多端 App 中不可用，App 端需要自建后端服务。
> 2. `wx.getUserProfile` 在 App 端不可用，需要使用各平台原生的用户授权方式。
> 3. iOS 和 Android 的返回键行为不同：Android 有物理/手势返回键，iOS 需要在 UI 中提供返回按钮。
> 4. App 端的网络请求可能遇到 HTTPS 证书问题和 CORS 限制不同的情况。

## 20.7 选择多端开发 SDK（Taro/uni-app/Remax 对比）

除了微信官方的多端能力，社区也有优秀的第三方多端框架：

### 主流多端框架对比

| 特性 | Taro | uni-app | Remax | 微信多端 |
|------|------|---------|-------|----------|
| 开发方 | 京东（凹凸实验室） | DCloud | 蚂蚁集团 | 微信官方 |
| 语法 | React/Vue | Vue | React | 小程序原生 |
| 支持平台 | 全平台 | 全平台 | 支付宝系 | iOS/Android |
| 原生渲染 | 编译型 | 编译型+运行时 | 运行时 | 编译型 |
| 小程序兼容 | 需适配 | 需适配 | 需适配 | 原生兼容 |
| 性能 | 接近原生 | 接近原生 | 接近原生 | 接近原生 |

## 20.8 微信身份登录（多端 App 中的微信登录流程）

### 登录流程对比

**小程序中的微信登录（简单）：**

```
小程序前端 → wx.login() → code
    ↓
发送 code 到后端
    ↓
后端用 code + appid + secret → 换取 openid + session_key
    ↓
后端返回自定义 token
```

**多端 App 中的微信登录（复杂）：**

```
App 前端 → 微信 SDK 授权 → 获取 code
    ↓
发送 code 到后端
    ↓
后端用 code + 移动应用 appid + secret → 换取 access_token + openid
    ↓
后端获取 unionid（如需要）
    ↓
后端返回自定义 token
```

### 统一登录架构设计

```javascript
// 后端 — 统一登录接口设计
// POST /api/auth/wx-login
// 参数: { platform: 'miniprogram' | 'app', code: string }

async function wxLogin(platform, code) {
  let openid, unionid, userInfo
  
  if (platform === 'miniprogram') {
    // 小程序登录 — 使用小程序的 appid + secret
    const result = await requestWxApi(
      'https://api.weixin.qq.com/sns/jscode2session',
      {
        appid: MINIPROGRAM_APPID,
        secret: MINIPROGRAM_SECRET,
        js_code: code,
        grant_type: 'authorization_code'
      }
    )
    openid = result.openid
    unionid = result.unionid  // 可能不存在（未绑定开放平台）
    
  } else if (platform === 'app') {
    // App 登录 — 使用移动应用的 appid + secret
    const result = await requestWxApi(
      'https://api.weixin.qq.com/sns/oauth2/access_token',
      {
        appid: APP_APPID,        // 移动应用的 AppID
        secret: APP_SECRET,       // 移动应用的 AppSecret
        code: code,
        grant_type: 'authorization_code'
      }
    )
    openid = result.openid
    unionid = result.unionid
    const accessToken = result.access_token
    
    // 获取用户信息
    const userInfoResult = await requestWxApi(
      'https://api.weixin.qq.com/sns/userinfo',
      {
        access_token: accessToken,
        openid: openid
      }
    )
    userInfo = {
      nickName: userInfoResult.nickname,
      avatarUrl: userInfoResult.headimgurl
    }
  }
  
  // 通过 unionid 查找或创建用户
  let user = await User.findOne({ where: { wx_unionid: unionid } })
  
  if (!user) {
    // 新用户注册
    user = await User.create({
      wx_openid_mp: platform === 'miniprogram' ? openid : null,
      wx_openid_app: platform === 'app' ? openid : null,
      wx_unionid: unionid,
      nickname: userInfo?.nickName || '微信用户',
      avatar: userInfo?.avatarUrl || ''
    })
  } else {
    // 更新已有用户的 openid（用户可能先用小程序，后用 App）
    if (platform === 'miniprogram' && !user.wx_openid_mp) {
      user.wx_openid_mp = openid
    }
    if (platform === 'app' && !user.wx_openid_app) {
      user.wx_openid_app = openid
    }
    await user.save()
  }
  
  // 生成自定义 token
  const token = generateToken(user.id)
  return { token, userInfo: user }
}
```

### 前端统一登录调用

```javascript
// utils/auth.js

/**
 * 统一微信登录
 */
async function wxLogin() {
  const platform = getPlatform()
  
  if (platform === 'miniprogram') {
    // 小程序登录
    const { code } = await wx.login()
    const res = await request({
      url: 'https://your-api.com/api/auth/wx-login',
      method: 'POST',
      data: { platform: 'miniprogram', code }
    })
    return res.data
    
  } else {
    // App 端微信登录 — 需要调用微信 SDK
    // #ifdef APP-IOS || APP-ANDROID
    const { code } = await callWxSdkLogin()
    const res = await request({
      url: 'https://your-api.com/api/auth/wx-login',
      method: 'POST',
      data: { platform: 'app', code }
    })
    return res.data
    // #endif
  }
}

/**
 * 调用微信 SDK 登录（App 端）
 */
function callWxSdkLogin() {
  return new Promise((resolve, reject) => {
    // App 端通过原生插件调用微信 SDK
    // 具体调用方式取决于多端框架的实现
    if (wx.wxLoginForApp) {
      wx.wxLoginForApp({
        success: resolve,
        fail: reject
      })
    } else {
      reject(new Error('当前环境不支持微信 SDK 登录'))
    }
  })
}

module.exports = { wxLogin }
```

> **注意事项**：
> 1. 小程序和 App 使用**不同的 AppID 和 AppSecret**，后端需要分别配置。
> 2. 必须将小程序和移动应用绑定到同一个微信开放平台账号，才能获取到相同的 UnionID。
> 3. App 端微信登录需要用户安装微信客户端。

### 微信开放平台配置

在微信开放平台的移动应用配置中填写：

| 配置项 | iOS | Android |
|--------|-----|---------|
| AppID | 微信开放平台分配 | 微信开放平台分配 |
| Bundle ID | com.yourcompany.app | - |
| Universal Links | https://your-domain.com/app/ | - |
| 包名 | - | com.yourcompany.app |
| 应用签名 | - | MD5 签名（无冒号小写） |

> **常见坑**：
> 1. Android 应用签名必须与微信开放平台填写的签名一致，否则微信登录会失败。Release 和 Debug 包的签名不同，需要分别处理。
> 2. Universal Links 的 `apple-app-site-association` 文件必须通过 HTTPS 访问，且不能有重定向。
> 3. WXEntryActivity 必须在 `(你的包名).wxapi` 包下，否则微信回调无法到达。

## 20.10 拉起微信小程序授权登录（wx.login在多端的适配）

### 多端登录适配方案

在多端开发中，`wx.login` 的行为在小程序和 App 端不同，需要统一适配：

```javascript
// utils/multi-platform-auth.js

/**
 * 多端统一登录入口
 */
async function unifiedLogin() {
  const platform = getPlatform()
  
  try {
    let authResult
    
    switch (platform) {
      case 'miniprogram':
        authResult = await miniprogramLogin()
        break
      case 'ios':
      case 'android':
        authResult = await appLogin()
        break
      default:
        throw new Error('不支持的平台')
    }
    
    // 保存登录状态
    saveAuthState(authResult)
    return authResult
  } catch (err) {
    console.error('登录失败:', err)
    throw err
  }
}

/**
 * 小程序登录
 */
async function miniprogramLogin() {
  // wx.login 获取临时 code
  const { code } = await wx.login()
  
  // 发送到后端换取 token
  const res = await request({
    url: 'https://your-api.com/api/auth/wx-login',
    method: 'POST',
    data: {
      platform: 'miniprogram',
      code: code
    }
  })
  
  return res.data
}

/**
 * App 端微信登录
 */
async function appLogin() {
  // 检查是否安装微信
  const isInstalled = await checkWxInstalled()
  if (!isInstalled) {
    throw new Error('请先安装微信客户端')
  }
  
  // 拉起微信授权
  const { code } = await pullUpWxAuth()
  
  // 发送到后端换取 token
  const res = await request({
    url: 'https://your-api.com/api/auth/wx-login',
    method: 'POST',
    data: {
      platform: 'app',
      code: code
    }
  })
  
  return res.data
}

/**
 * 检查微信是否安装（App 端）
 */
function checkWxInstalled() {
  return new Promise((resolve) => {
    // #ifdef APP-IOS || APP-ANDROID
    if (wx.canIUse('isWxAppInstalled')) {
      wx.isWxAppInstalled({
        success: res => resolve(res.installed !== false),
        fail: () => resolve(true)  // 默认假设已安装
      })
    } else {
      resolve(true)
    }
    // #endif
    
    // #ifdef MP-WEIXIN
    resolve(true)  // 小程序中不需要检查
    // #endif
  })
}

/**
 * 拉起微信授权登录（App 端）
 */
function pullUpWxAuth() {
  return new Promise((resolve, reject) => {
    // #ifdef APP-IOS || APP-ANDROID
    // 通过多端框架提供的 API 拉起微信
    if (wx.loginWithWx) {
      wx.loginWithWx({
        scope: 'snsapi_userinfo',
        success: res => resolve({ code: res.code }),
        fail: reject
      })
    } else {
      reject(new Error('当前框架不支持微信 SDK 登录'))
    }
    // #endif
  })
}

/**
 * 保存登录状态
 */
function saveAuthState(authResult) {
  wx.setStorageSync('token', authResult.token)
  wx.setStorageSync('userInfo', authResult.userInfo)
  wx.setStorageSync('loginPlatform', getPlatform())
}

/**
 * 检查登录状态
 */
function checkAuthState() {
  const token = wx.getStorageSync('token')
  if (!token) return false
  
  // 可选: 检查 token 是否过期
  const userInfo = wx.getStorageSync('userInfo')
  return !!userInfo
}

/**
 * 退出登录
 */
function logout() {
  wx.removeStorageSync('token')
  wx.removeStorageSync('userInfo')
  wx.removeStorageSync('loginPlatform')
}

module.exports = {
  unifiedLogin,
  checkAuthState,
  logout
}
```

### 后端 Token 验证中间件

```javascript
// 后端 — Token 验证中间件（Node.js Express 示例）
const jwt = require('jsonwebtoken')

function authMiddleware(req, res, next) {
  const token = req.headers.authorization?.replace('Bearer ', '')
  
  if (!token) {
    return res.status(401).json({ code: -1, message: '未登录' })
  }
  
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET)
    req.userId = decoded.userId
    req.platform = decoded.platform
    next()
  } catch (err) {
    return res.status(401).json({ code: -1, message: '登录已过期' })
  }
}

// 使用
app.get('/api/user/profile', authMiddleware, async (req, res) => {
  const user = await User.findByPk(req.userId)
  res.json({ code: 0, data: user })
})
```

> **注意事项**：
> 1. App 端微信登录的 API 名称和调用方式取决于多端框架的具体实现，以上为示意代码。
> 2. Token 应设置合理的过期时间（如 7 天），并提供刷新机制。
> 3. 登录失败时需要给用户清晰的错误提示（如"未安装微信"、"授权取消"等）。
> 4. iOS 首次拉起微信授权时会显示跳转确认弹窗，属于正常行为。

## 20.11 本章小结

本章全面介绍了小程序多端开发的核心知识，要点如下：

| 知识点 | 核心内容 |
|--------|----------|
| 多端框架概念 | 一套代码编译为小程序 + iOS/Android 原生应用 |
| 效率与成本 | 代码复用率 80%+，开发成本降低 70%+ |
| 账号体系 | UnionID 打通小程序与 App 的微信账号 |
| 微信开发者平台 | 注册移动应用、获取 AppID、关联小程序 |
| 多端项目创建 | 配置 multiPlatform、条件编译 |
| 预览与调试 | 模拟器预览、真机扫码安装 |
| 注意事项 | API 兼容性、UI 适配、平台限制 |
| SDK 对比 | Taro(React) / uni-app(Vue) / 微信多端(原生) |
| 微信登录 | 多端统一登录架构、UnionID 账号打通 |
| 授权准备 | Universal Links(iOS) / 应用签名(Android) |
| 登录适配 | wx.login 在小程序与 App 端的统一封装 |

**核心要点回顾**：

1. **多端开发的价值**：一套代码覆盖小程序 + iOS + Android，大幅降低开发和维护成本。

2. **UnionID 是账号打通的钥匙**：将小程序和移动应用绑定到同一微信开放平台账号，通过 UnionID 实现跨端用户身份统一。

3. **条件编译处理平台差异**：使用 `#ifdef` / `#ifndef` 指令针对不同平台编写差异化代码。

4. **API 兼容性是最大挑战**：云开发、微信特有 API 在 App 端不可用，需要提前规划替代方案。

5. **平台配置不可忽视**：iOS 的 Universal Links 和 Android 的应用签名是微信登录正常工作的前提。

6. **框架选择需量体裁衣**：已有小程序选微信多端（改动最小），新项目可根据团队技术栈选 Taro 或 uni-app。

7. **真机测试必不可少**：模拟器无法覆盖所有真机场景，特别是微信登录、支付等涉及第三方 SDK 的功能。

多端开发是小程序生态的重要扩展方向，掌握多端开发能力可以让小程序开发者以最低成本进入原生 App 领域，实现"一次开发，多端运行"的高效模式。
