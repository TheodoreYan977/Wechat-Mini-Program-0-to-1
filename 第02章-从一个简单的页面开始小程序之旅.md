# 第02章 从一个简单的页面开始小程序之旅

## 2.1 小程序的基本文件结构

小程序的文件结构分为**全局文件**和**页面文件**两类，它们各司其职，共同构成完整的小程序应用。

### 2.1.1 全局文件

全局文件位于项目根目录，作用于整个小程序应用：

| 文件 | 作用 | 是否必须 |
|------|------|---------|
| `app.js` | 小程序应用逻辑，定义App生命周期 | 是 |
| `app.json` | 全局配置（页面路由、窗口样式、tabBar等） | 是 |
| `app.wxss` | 全局样式表，所有页面共享 | 否 |
| `project.config.json` | 项目工具配置 | 自动生成 |
| `sitemap.json` | 搜索索引配置 | 自动生成 |

**app.js 示例：**

```javascript
// app.js
App({
  // 小程序启动时触发，全局只执行一次
  onLaunch(options) {
    console.log('小程序启动', options);
    // 可以在这里做全局初始化：获取本地缓存、检查更新等
  },

  // 小程序显示时触发（从后台切到前台）
  onShow(options) {
    console.log('小程序进入前台', options);
  },

  // 小程序隐藏时触发（从前台切到后台）
  onHide() {
    console.log('小程序进入后台');
  },

  // 全局数据，所有页面可通过 getApp().globalData 访问
  globalData: {
    userInfo: null,
    baseUrl: 'https://api.example.com'
  }
});
```

**app.json 示例：**

```json
{
  "pages": [
    "pages/index/index",
    "pages/logs/logs"
  ],
  "window": {
    "navigationBarTitleText": "我的小程序",
    "navigationBarBackgroundColor": "#ffffff",
    "navigationBarTextStyle": "black",
    "backgroundColor": "#f8f8f8",
    "enablePullDownRefresh": false
  },
  "tabBar": {
    "list": [
      {
        "pagePath": "pages/index/index",
        "text": "首页",
        "iconPath": "images/home.png",
        "selectedIconPath": "images/home-active.png"
      },
      {
        "pagePath": "pages/logs/logs",
        "text": "日志",
        "iconPath": "images/log.png",
        "selectedIconPath": "images/log-active.png"
      }
    ]
  },
  "style": "v2",
  "sitemapLocation": "sitemap.json"
}
```

> **⚠️ 注意**：`pages`数组中**第一个页面**就是小程序的首页（启动页）。新增页面只需在此数组中添加路径，开发者工具会自动创建对应文件。

### 2.1.2 页面文件

每个页面由四种文件组成，必须放在同一目录下，且文件名相同：

| 文件类型 | 作用 | 是否必须 |
|---------|------|---------|
| `.js` | 页面逻辑与数据 | 是 |
| `.json` | 页面配置（覆盖全局配置） | 否 |
| `.wxml` | 页面结构模板（类似HTML） | 是 |
| `.wxss` | 页面样式表（类似CSS） | 否 |

## 2.2 编写第一个小程序页面

让我们从零开始编写一个欢迎页面。在`pages/welcome/`目录下创建以下文件：

**welcome.wxml：**

```xml
<view class="container">
  <image class="avatar" src="/images/avatar.png"></image>
  <text class="username">Hello, 小程序</text>
  <view class="motto">开启你的小程序之旅</view>
</view>
```

**welcome.wxss：**

```css
.container {
  display: flex;
  flex-direction: column;
  align-items: center;
  padding-top: 100rpx;
}

.avatar {
  width: 200rpx;
  height: 200rpx;
  border-radius: 50%;
}

.username {
  font-size: 36rpx;
  font-weight: bold;
  margin-top: 30rpx;
}

.motto {
  font-size: 28rpx;
  color: #888;
  margin-top: 20rpx;
}
```

**welcome.js：**

```javascript
Page({
  data: {
    // 页面的初始数据
  },

  onLoad(options) {
    // 页面加载时触发
    console.log('欢迎页面加载完成');
  }
});
```

**welcome.json：**

```json
{
  "navigationBarTitleText": "欢迎"
}
```

别忘了在`app.json`的`pages`数组中注册页面：

```json
{
  "pages": [
    "pages/welcome/welcome"
  ]
}
```

## 2.3 构建页面的元素和样式

小程序的WXML提供了一套专用的组件标签，与HTML标签不同但概念相似：

| WXML组件 | 类似HTML标签 | 说明 |
|---------|-------------|------|
| `<view>` | `<div>` | 视图容器 |
| `<text>` | `<span>` | 文本（只能包含文本） |
| `<image>` | `<img>` | 图片 |
| `<button>` | `<button>` | 按钮 |
| `<input>` | `<input>` | 输入框 |
| `<scroll-view>` | 可滚动`<div>` | 可滚动视图区域 |
| `<swiper>` | 轮播图容器 | 滑块容器 |

> **⚠️ 注意**：小程序中`<text>`组件内只能放文本，不能嵌套`<view>`。长文本换行需在`<text>`中使用`\n`并设置`decode`属性。

WXSS的写法与CSS基本一致，支持类选择器、ID选择器、属性选择器等。

## 2.4 小程序所支持的CSS选择器

小程序WXSS支持以下CSS选择器：

| 选择器 | 示例 | 说明 |
|-------|------|------|
| 类选择器 | `.class-name` | ✅ 最常用 |
| ID选择器 | `#my-id` | ✅ 支持 |
| 元素选择器 | `view`、`text` | ✅ 支持 |
| 后代选择器 | `.container .item` | ✅ 支持 |
| 子选择器 | `.container > .item` | ✅ 支持 |
| 并集选择器 | `.a, .b` | ✅ 支持 |
| 伪类 | `:first-child`、`:nth-child(n)` | ✅ 部分支持 |
| 伪元素 | `::before`、`::after` | ✅ 支持（需`content`属性） |

**不支持的选择器：**
- `*` 通配符选择器
- 属性选择器中的部分高级语法
- `:hover`等鼠标交互伪类（移动端无鼠标）

```css
/* ✅ 正确示例 */
.container {
  background: #f5f5f5;
}

.container .title {
  font-size: 32rpx;
}

view:first-child {
  border-top: none;
}

/* ❌ 不支持的写法 */
* {
  margin: 0;
}

.btn:hover {
  background: red;
}
```

> **💡 注意事项**：尽量使用**类选择器**而非元素选择器，性能更好且更精确。避免深层嵌套的选择器链。

## 2.5 在WXSS文件中使用图片时的注意事项

在小程序的WXSS中使用背景图片时，有以下限制：

```css
/* ❌ 不支持：WXSS中不能直接引用本地图片作为background-image */
.bg {
  background-image: url('./images/bg.png');
}

/* ✅ 方式1：使用网络图片 */
.bg {
  background-image: url('https://example.com/images/bg.png');
}

/* ✅ 方式2：使用base64编码（适合小图标） */
.bg {
  background-image: url('data:image/png;base64,iVBORw0KGgoAAAANSUhEUg...');
}

/* ✅ 方式3：使用image组件替代（推荐） */
```

```xml
<!-- 推荐用image组件替代CSS背景图 -->
<view class="bg-wrapper">
  <image class="bg-img" src="/images/bg.png" mode="aspectFill"></image>
  <view class="bg-content">内容覆盖在图片上</view>
</view>
```

```css
.bg-wrapper {
  position: relative;
  width: 100%;
  height: 400rpx;
}
.bg-img {
  position: absolute;
  width: 100%;
  height: 100%;
}
.bg-content {
  position: relative;
  z-index: 1;
}
```

> **⚠️ 总结**：WXSS中`background-image`只支持**网络图片**和**base64**，不支持本地路径。本地图片请用`<image>`组件。

## 2.6 Flex布局

小程序对Flex布局有极好的支持，是构建页面的首选布局方式。

### Flex核心属性

```css
/* 容器属性 */
.flex-container {
  display: flex;
  flex-direction: row;        /* 主轴方向：row | row-reverse | column | column-reverse */
  justify-content: center;    /* 主轴对齐：flex-start | flex-end | center | space-between | space-around */
  align-items: center;        /* 交叉轴对齐：flex-start | flex-end | center | stretch | baseline */
  flex-wrap: wrap;            /* 换行：nowrap | wrap | wrap-reverse */
}

/* 子元素属性 */
.flex-item {
  flex: 1;                    /* flex-grow:1, flex-shrink:1, flex-basis:0% */
  align-self: flex-end;       /* 单独设置交叉轴对齐 */
  order: 2;                   /* 排列顺序 */
}
```

### 实战示例：水平垂直居中

```xml
<view class="center-box">
  <text>水平垂直居中</text>
</view>
```

```css
.center-box {
  display: flex;
  justify-content: center;   /* 水平居中 */
  align-items: center;       /* 垂直居中 */
  height: 300rpx;
  background: #eee;
}
```

### 实战示例：经典三栏布局

```xml
<view class="three-col">
  <view class="left">左侧</view>
  <view class="center">中间</view>
  <view class="right">右侧</view>
</view>
```

```css
.three-col {
  display: flex;
  flex-direction: row;
}
.left, .right {
  width: 200rpx;       /* 固定宽度 */
}
.center {
  flex: 1;             /* 自动撑满剩余空间 */
  min-width: 0;        /* 防止内容溢出，重要！ */
}
```

> **⚠️ 坑点**：flex子元素包含长文本时，需加`min-width: 0`或`overflow: hidden`，否则flex子元素不会缩小，导致布局溢出。

## 2.7 小程序自适应单位rpx简介

### rpx与px的换算关系

`rpx`（responsive pixel）是小程序独有的自适应单位，能够根据屏幕宽度自动缩放。

**换算规则**：以屏幕宽度750rpx为基准，无论屏幕实际宽度是多少，750rpx始终等于屏幕宽度。

```
设计稿宽度 = 750rpx = 屏幕实际宽度
```

以iPhone 6为例（屏幕宽度375px）：

```
750rpx = 375px
1rpx = 0.5px
1px = 2rpx
```

| 设备 | 屏幕宽度(px) | 1rpx = ?px |
|------|-------------|-----------|
| iPhone 5 | 320px | 0.42px |
| iPhone 6 | 375px | 0.5px |
| iPhone 6 Plus | 414px | 0.552px |

### 750rpx设计稿的用法

如果设计师提供的UI设计稿宽度为750px，那么直接将设计稿上的像素值`1:1`替换为`rpx`即可：

```
设计稿上标注 32px → 代码中写 32rpx
设计稿上标注 100px → 代码中写 100rpx
```

如果设计稿宽度为375px（如基于iPhone 6），则需将像素值×2：

```
设计稿上标注 16px → 代码中写 32rpx
```

> **💡 实战建议**：
> - 所有尺寸尽量使用`rpx`，保证不同屏幕自适应
> - 边框`border`通常用`1px`（物理像素，更精细）
> - 字体大小可用`rpx`也可用`px`，根据设计稿决定
> - 不要在JS中使用`rpx`，JS中只能用`px`

## 2.8 全局样式文件app.wxss

`app.wxss`是全局样式表，其中定义的样式会作用于所有页面。

```css
/* app.wxss - 全局样式 */

/* 全局重置 */
page {
  background-color: #f5f5f5;
  font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif;
  font-size: 28rpx;
  color: #333;
  line-height: 1.6;
}

/* 全局通用类 */
.flex-center {
  display: flex;
  align-items: center;
  justify-content: center;
}

.text-ellipsis {
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}

/* 卡片通用样式 */
.card {
  background: #fff;
  border-radius: 16rpx;
  padding: 24rpx;
  margin: 24rpx;
  box-shadow: 0 2rpx 12rpx rgba(0, 0, 0, 0.08);
}
```

页面级`.wxss`文件中的样式会**覆盖**同名的全局样式：

```css
/* pages/index/index.wxss - 页面样式会覆盖全局同名样式 */
.card {
  padding: 32rpx;  /* 覆盖全局的24rpx */
}
```

> **⚠️ 注意**：全局样式应保持简洁，只定义真正全局通用的样式。页面特有样式放在页面`.wxss`中。

## 2.9 小程序字体设置与动态加载字体

### 静态字体设置

在`app.wxss`中通过`page`选择器设置全局字体：

```css
page {
  font-family: -apple-system, BlinkMacSystemFont, 'Helvetica Neue',
               'PingFang SC', 'Microsoft YaHei', sans-serif;
}
```

### 动态加载网络字体

小程序支持通过`wx.loadFontFace`API动态加载网络字体：

```javascript
Page({
  onLoad() {
    // 动态加载字体
    wx.loadFontFace({
      family: 'MyCustomFont',
      source: 'url("https://example.com/fonts/myfont.woff")',
      success(res) {
        console.log('字体加载成功', res);
      },
      fail(err) {
        console.error('字体加载失败', err);
      },
      complete() {
        console.log('字体加载完成');
      }
    });
  }
});
```

在WXSS中使用加载的字体：

```css
.custom-text {
  font-family: 'MyCustomFont';
  font-size: 36rpx;
}
```

> **⚠️ 注意事项**：
> 1. 字体文件URL必须是**HTTPS**协议
> 2. 字体格式建议使用`.woff`或`.woff2`（体积小，兼容好）
> 3. 字体文件需在域名白名单中配置
> 4. 字体加载是异步的，加载完成前文字会用降级字体显示
> 5. 开发阶段可在项目设置中勾选"不校验合法域名"来跳过校验

## 2.10 页面的根元素page

每个小程序页面都有一个隐含的根元素`<page>`，类似HTML中的`<body>`。它不需要在WXML中显式声明，但可以通过WXSS对其进行样式设置。

```css
/* 设置页面背景色、字体等 */
page {
  background-color: #f0f0f0;
  font-size: 28rpx;
  color: #333;
  height: 100%;       /* page默认高度为100% */
}
```

`page`元素的常用场景：

- 设置页面整体背景色
- 设置全局字体大小和颜色
- 控制页面最小高度

```css
/* 让内容至少铺满整个屏幕 */
page {
  min-height: 100vh;
  background: #f5f5f5;
}
```

> **⚠️ 注意**：`page`是页面级别的根元素，只能在页面`.wxss`或`app.wxss`中使用。不能在WXML中显式写`<page>`标签。

## 2.11 app.json中的window配置项

`app.json`中的`window`字段用于配置所有页面的默认窗口表现：

```json
{
  "window": {
    "navigationBarTitleText": "小程序标题",
    "navigationBarBackgroundColor": "#ffffff",
    "navigationBarTextStyle": "black",
    "backgroundColor": "#f5f5f5",
    "backgroundTextStyle": "dark",
    "enablePullDownRefresh": false,
    "onReachBottomDistance": 50,
    "navigationStyle": "default"
  }
}
```

| 配置项 | 类型 | 说明 |
|-------|------|------|
| `navigationBarTitleText` | string | 导航栏标题文字 |
| `navigationBarBackgroundColor` | HexColor | 导航栏背景颜色（如`#ffffff`） |
| `navigationBarTextStyle` | string | 导航栏文字颜色：`white`或`black` |
| `backgroundColor` | HexColor | 窗口背景色（下拉时可见） |
| `backgroundTextStyle` | string | 下拉loading样式：`dark`或`light` |
| `enablePullDownRefresh` | boolean | 是否开启下拉刷新 |
| `onReachBottomDistance` | number | 上拉触底距离（单位px），默认50 |
| `navigationStyle` | string | 导航栏样式：`default`或`custom` |

> **⚠️ 注意**：`navigationBarBackgroundColor`只支持十六进制颜色，不支持`rgb()`或颜色名称。

## 2.12 取消默认顶部导航栏

某些场景下（如启动页、全屏活动页），需要取消默认导航栏以实现自定义UI。

在页面的`.json`配置中设置：

```json
{
  "navigationStyle": "custom"
}
```

取消导航栏后，页面内容会顶到状态栏下方。需要手动处理状态栏的高度：

```javascript
Page({
  data: {
    statusBarHeight: 0,
    navBarHeight: 0
  },

  onLoad() {
    // 获取系统信息
    const systemInfo = wx.getWindowInfo();
    const menuButton = wx.getMenuButtonBoundingClientRect();

    // 计算自定义导航栏高度
    const navBarHeight = (menuButton.top - systemInfo.statusBarHeight) * 2 + menuButton.height;

    this.setData({
      statusBarHeight: systemInfo.statusBarHeight,
      navBarHeight: navBarHeight
    });
  }
});
```

```xml
<view class="custom-nav" style="padding-top: {{statusBarHeight}}px; height: {{navBarHeight + statusBarHeight}}px;">
  <text class="nav-title">自定义导航栏</text>
</view>
```

```css
.custom-nav {
  display: flex;
  align-items: center;
  justify-content: center;
  background: #07c160;
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  z-index: 999;
}
.nav-title {
  color: #fff;
  font-size: 32rpx;
}
```

> **💡 实战要点**：自定义导航栏时需考虑胶囊菜单（右上角按钮）的位置，通过`wx.getMenuButtonBoundingClientRect()`获取胶囊位置来精确计算布局。

## 2.13 页面的配置文件

每个页面可以有自己的`.json`配置文件，用于**覆盖**全局`app.json`中的`window`配置：

**pages/index/index.json：**

```json
{
  "navigationBarTitleText": "首页",
  "navigationBarBackgroundColor": "#07c160",
  "navigationBarTextStyle": "white",
  "enablePullDownRefresh": true,
  "usingComponents": {}
}
```

页面配置只影响当前页面，不会影响其他页面。常见可配置项与全局`window`基本一致，额外支持：

| 配置项 | 说明 |
|-------|------|
| `usingComponents` | 引用自定义组件 |
| `disableScroll` | 是否禁止页面滚动（iOS生效） |

> **⚠️ 注意**：页面`.json`文件中不能有注释，必须是严格的JSON格式。即使为空也需要写`{}`。

## 2.14 小程序的可配置性

小程序的配置体系是分层的，体现了良好的可配置性设计：

```
app.json (全局配置)
  └── pages/xxx/xxx.json (页面配置，覆盖全局)
       └── 组件级配置 (通过组件属性和样式控制)
```

**配置优先级**：页面配置 > 全局配置 > 默认配置

这种分层设计的好处：

1. **统一规范**：全局配置设定默认行为，保证一致性
2. **灵活覆盖**：个别页面可按需覆盖，无需修改全局配置
3. **降低耦合**：每个页面的特殊需求在页面自身配置中管理

```json
// app.json - 所有页面默认白色导航栏
{
  "window": {
    "navigationBarBackgroundColor": "#ffffff",
    "navigationBarTextStyle": "black"
  }
}

// pages/detail/detail.json - 详情页单独使用绿色导航栏
{
  "navigationBarBackgroundColor": "#07c160",
  "navigationBarTextStyle": "white",
  "navigationBarTitleText": "详情"
}
```

关键要点回顾：
- `rpx`是小程序的核心自适应单位，750rpx = 屏幕宽度
- WXSS中`background-image`不支持本地图片，用`<image>`组件替代
- 页面配置覆盖全局配置，优先级：页面 > 全局
- Flex布局是小程序首选布局方式，注意`min-width: 0`防止溢出
- `page`是页面隐含根元素，用于设置全局背景和字体
