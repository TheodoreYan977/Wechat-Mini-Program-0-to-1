# 第13章 组件化思维构建电影详情页面

## 13.1 电影详情页面分析

在前面的章节中，我们已经完成了电影列表页面的开发，用户可以浏览热门电影。接下来需要实现点击列表项后跳转到电影详情页面，展示该电影的完整信息。

### 页面功能需求

电影详情页面需要展示以下内容：

- **电影海报**：大图展示，支持点击预览
- **基础信息**：电影名称、年份、导演、主演、类型
- **评分信息**：星级评分（复用 o-stars 组件）、评分人数
- **剧情简介**：电影故事梗概
- **导航栏标题**：动态设置为电影名称

### 页面间数据传递分析

从列表页跳转到详情页时，需要传递电影的唯一标识（`movieId`），详情页根据该 ID 发起网络请求获取完整数据。这是小程序中最常见的页面间通信方式——通过 URL 查询参数传递。

```
列表页 → (movieId 通过 URL) → 详情页 → (发起请求) → 服务器返回详情数据
```

### 组件化复用思路

详情页中的星级评分部分可以直接复用之前编写的 `o-stars` 组件，这正是组件化开发的核心优势——**一次编写，处处复用**。但在复用过程中，我们会遇到组件样式隔离的问题，这将在 13.6 节中重点解决。

## 13.3 编写电影详情页面的业务逻辑代码

### 获取详情数据与参数接收

`pages/detail/detail.js`：

```javascript
Page({
  data: {
    movie: {},
    movieId: ''
  },

  onLoad: function (options) {
    // 接收列表页传递过来的 movieId 参数
    const movieId = options.movieId;
    this.setData({
      movieId: movieId
    });

    // 根据 movieId 发起网络请求获取详情数据
    this.getMovieDetail(movieId);
  },

  /**
   * 获取电影详情数据
   */
  getMovieDetail: function (movieId) {
    wx.showLoading({
      title: '加载中...'
    });

    wx.request({
      url: `https://api.example.com/v2/movie/${movieId}`,
      method: 'GET',
      success: (res) => {
        if (res.statusCode === 200) {
          const data = res.data;
          // 格式化数据
          const movie = {
            title: data.title,
            year: data.year,
            postImg: data.images.large,
            directors: data.directors.map(d => d.name).join(' / '),
            casts: data.casts.map(c => c.name).join(' / '),
            genres: data.genres.join(' / '),
            score: data.rating.average,
            summary: data.summary
          };
          this.setData({
            movie: movie
          });

          // 动态设置导航栏标题
          this.setNavigationBarTitle(movie.title);
        }
      },
      fail: (err) => {
        console.error('请求失败', err);
        wx.showToast({
          title: '网络请求失败',
          icon: 'none'
        });
      },
      complete: () => {
        wx.hideLoading();
      }
    });
  },

  /**
   * 设置导航栏标题
   */
  setNavigationBarTitle: function (title) {
    wx.setNavigationBarTitle({
      title: title
    });
  },

  /**
   * 预览电影海报
   */
  onPreviewImage: function () {
    const imgUrl = this.data.movie.postImg;
    if (!imgUrl) return;

    wx.previewImage({
      urls: [imgUrl],
      current: imgUrl
    });
  }
});
```

### 列表页跳转代码

在列表页中，点击电影项时携带 `movieId` 跳转到详情页：

```javascript
// pages/list/list.js
Page({
  data: {
    movies: []
  },

  onMovieTap: function (event) {
    const movieId = event.currentTarget.dataset.movieId;
    wx.navigateTo({
      url: `/pages/detail/detail?movieId=${movieId}`
    });
  }
});
```

对应的列表项 WXML：

```html
<view class="movie-item" 
      wx:for="{{movies}}" 
      wx:key="id"
      bindtap="onMovieTap"
      data-movie-id="{{item.id}}">
  <image src="{{item.postImg}}" mode="aspectFill"></image>
  <view class="movie-info">
    <text class="title">{{item.title}}</text>
    <o-stars score="{{item.score}}"></o-stars>
  </view>
</view>
```

### 参数传递的注意事项

| 传递方式 | 适用场景 | 示例 |
|---------|---------|------|
| URL 查询参数 | 页面跳转时传递简单数据 | `url: '/pages/detail/detail?id=123'` |
| 全局变量 | 跨页面共享复杂数据 | `getApp().globalData.movie = data` |
| 缓存 Storage | 需要持久化的数据 | `wx.setStorageSync('key', data)` |
| 事件通道 | 返回上一页时传值 | `eventChannel.emit('event', data)` |

> **常见坑**：URL 参数传递时，如果数据中包含特殊字符（如 `&`、`=`），需要使用 `encodeURIComponent` 编码，接收时用 `decodeURIComponent` 解码。

## 13.4 预览电影海报（wx.previewImage）

`wx.previewImage` 是小程序提供的图片预览 API，用户点击图片后可以全屏查看，支持双指缩放、左右滑动切换等手势操作。

### 基本用法

```javascript
wx.previewImage({
  current: '',   // 当前显示图片的 http 链接，不填默认为 urls 的第一张
  urls: []       // 需要预览的图片 http 链接列表
});
```

### 在详情页中的应用

当用户点击海报时，调用预览功能：

```javascript
onPreviewImage: function () {
  const imgUrl = this.data.movie.postImg;
  if (!imgUrl) {
    wx.showToast({
      title: '图片加载中',
      icon: 'none'
    });
    return;
  }

  wx.previewImage({
    current: imgUrl,
    urls: [imgUrl]
  });
}
```

### 多图预览场景

如果电影有多张剧照，可以预览一个图片列表：

```javascript
onPreviewStill: function (event) {
  const currentUrl = event.currentTarget.dataset.src;
  const urls = this.data.movie.stills; // 剧照图片数组

  wx.previewImage({
    current: currentUrl,
    urls: urls
  });
}
```

> **注意事项**：
> 1. `urls` 数组中的链接必须是网络图片地址（http/https），不支持本地临时路径。
> 2. `current` 参数必须包含在 `urls` 数组中，否则会报错。
> 3. 图片域名需要在小程序后台配置 downloadFile 合法域名。

## 13.5 设置电影页面的导航栏标题

在静态配置中，我们在 `detail.json` 中设置了默认的导航栏标题为"电影详情"。但实际开发中，我们希望导航栏标题动态显示当前电影的名称。

### 动态设置导航栏标题

使用 `wx.setNavigationBarTitle` API：

```javascript
// 在获取数据成功后调用
getMovieDetail: function (movieId) {
  wx.request({
    url: `https://api.example.com/v2/movie/${movieId}`,
    success: (res) => {
      const movie = this.formatMovieData(res.data);
      this.setData({ movie });

      // 动态设置导航栏标题为电影名称
      wx.setNavigationBarTitle({
        title: movie.title
      });
    }
  });
}
```

### 导航栏标题设置的时机

```
onLoad → 发起请求 → 请求返回 → setData + setNavigationBarTitle
```

在数据返回之前，导航栏显示的是 `detail.json` 中配置的默认标题"电影详情"。数据返回后，标题会被替换为实际电影名称。

> **优化建议**：可以在 `onLoad` 中先从列表页传递的数据（如果有）中提前设置标题，避免用户等待。这需要在跳转时额外传递电影名称参数。

## 13.6 修复 o-stars 组件显示无效的问题

### 问题现象

在详情页中使用 `o-stars` 组件时，传入的 `starSize`、`fontSize` 等样式属性不生效，星级显示尺寸与预期不符。

### 问题原因分析

小程序组件默认启用了**样式隔离**（`styleIsolation`），组件内部的样式不会受到外部页面样式的影响，外部样式也无法直接修改组件内部元素。这种隔离机制保证了组件的样式独立性，但也带来了外部定制样式的困难。

```javascript
// o-stars 组件的默认配置
Component({
  options: {
    styleIsolation: 'isolated'  // 默认隔离模式
  }
});
```

### 解决方案：使用 externalClasses

`externalClasses` 允许组件接受外部传入的样式类名，从而实现外部样式对组件内部元素的控制。

**第一步：在组件中声明 externalClasses**

```javascript
// components/o-stars/o-stars.js
Component({
  externalClasses: ['star-class'],  // 声明外部样式类

  properties: {
    score: {
      type: Number,
      value: 0
    },
    starSize: {
      type: String,
      value: '20rpx'
    },
    fontSize: {
      type: String,
      value: '24rpx'
    }
  }
});
```

**第二步：在组件模板中使用外部样式类**

```html
<!-- components/o-stars/o-stars.wxml -->
<view class="container">
  <view class="stars-wrap star-class">
    <text class="stars" style="font-size: {{starSize}};">★★★★★</text>
    <view class="stars-cover" style="width: {{(10 - score) / 10 * 100}}%; font-size: {{starSize}};">★★★★★</view>
  </view>
  <text class="score-text" style="font-size: {{fontSize}};">{{score}}</text>
</view>
```

**第三步：在页面中传入自定义样式类**

```html
<!-- pages/detail/detail.wxml -->
<o-stars score="{{movie.score}}" 
         starSize="24rpx" 
         fontSize="28rpx"
         star-class="my-star"></o-stars>
```

**第四步：在页面样式中定义该类**

```css
/* pages/detail/detail.wxss */
.my-star {
  margin: 10rpx 0;
}

.my-star .stars {
  color: #f5a623;
}
```

### 样式隔离的三种模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| `isolated` | 完全隔离（默认） | 通用组件，不需要被外部样式影响 |
| `apply-shared` | 页面样式影响组件，组件样式不影响页面 | 需要继承页面样式的组件 |
| `shared` | 双向影响 | 高度定制化的组件 |

```javascript
Component({
  options: {
    styleIsolation: 'shared'  // 修改隔离模式
  }
});
```

> **常见坑**：`externalClasses` 传入的类名优先级低于组件内部定义的样式。如果发现外部样式不生效，检查组件内部是否有同名类覆盖了外部样式。

## 13.7 本章小结

本章通过构建电影详情页面，深入实践了组件化开发思维，主要知识点如下：

### 核心知识点回顾

1. **页面间参数传递**：通过 URL 查询参数在页面跳转时传递 `movieId`，在 `onLoad` 的 `options` 中接收。这是小程序中最基础的页面间通信方式。

2. **网络请求与数据绑定**：在 `onLoad` 中发起网络请求，获取数据后通过 `setData` 更新页面，小程序的响应式数据绑定会自动更新视图。

3. **wx.previewImage**：用于图片全屏预览，支持多图浏览和手势缩放。注意 `urls` 必须是网络图片地址，且 `current` 必须在 `urls` 列表中。

4. **动态导航栏标题**：使用 `wx.setNavigationBarTitle` 在数据加载完成后动态设置标题，提升用户体验。

5. **组件样式隔离与 externalClasses**：这是本章的重点难点。小程序组件默认样式隔离，需要通过 `externalClasses` 机制让外部样式类传入组件内部，实现样式定制。

### 组件化思维总结

```
组件化开发 = 封装 + 复用 + 定制
```

- **封装**：将星级评分功能封装为独立组件，内部逻辑对外不可见
- **复用**：列表页和详情页都能使用同一个 `o-stars` 组件
- **定制**：通过 `properties` 传参和 `externalClasses` 实现不同场景下的样式定制

### 实战要点

- 网络请求时务必添加 Loading 状态和错误处理
- URL 参数传递注意特殊字符编码
- 复用组件遇到样式问题时，优先考虑 `externalClasses` 方案
- 图片预览功能要检查域名白名单配置
- 导航栏标题的动态设置应在数据返回后及时调用
